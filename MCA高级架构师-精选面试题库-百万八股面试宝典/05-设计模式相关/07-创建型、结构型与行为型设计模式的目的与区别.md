设计模式可分为三大类：创建型（Creational）、结构型（Structural）和行为型（Behavioral），分别聚焦不同的软件设计需求：

M

#### 一、创建型设计模式

**目的**：控制对象创建过程，使系统与具体类解耦，提高灵活性、复用性与扩展性。  
具体包括：工厂方法、抽象工厂、单例、原型、建造者等模式。  
这些模式封装对象实例化，隐藏具体类和创建逻辑，客户端仅依赖抽象接口即可，无需知道实现细节。适合系统需要多样构建方式或动态实例化控制。

#### 二、结构型设计模式

**目的**：定义类或对象的组合方式，通过合理组织对象及接口关系，实现更大功能结构，提升系统模块间可组合性与灵活性。

S

#### 三、行为型设计模式

**目的**：处理对象间的交互与通信职责，通过封装算法与行为，实现松耦合、灵活协作与动态变更。  
主要包括：观察者（Observer）、策略（Strategy）、命令（Command）、职责链（Chain of Responsibility）、中介者（Mediator）、迭代器（Iterator）、状态（State）、模板方法（Template Method）等。  
这些模式定义对象间责任分配与协作流程，降低组件间直联依赖，提升沟通效率与系统可维护性。

B

### 对比

|  |  |  |  |
| --- | --- | --- | --- |
| **类别** | **关注点** | **作用** | **示例场景** |
| 创建型 | 对象创建机制 | 提供灵活、控制的实例化方式 | 工厂、构造复杂对象、控制单例 |
| 结构型 | 类与对象组合结构 | 优化模块组织、简化接口与解耦 | 适配器封装、装饰增强、代理控制访问 |
| 行为型 | 对象间交互和职责 | 解耦行为逻辑、封装通信流程 | 事件通知、策略替换、命令队列等 |

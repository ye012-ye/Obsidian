开源框架中，模板方法模式（Template Method Pattern）被广泛用于定义统一流程骨架，并允许子类通过重写部分步骤来自定义行为，同时保持整体框架结构不变。M

#### 示例一：Spring 的 `AbstractApplicationContext.refresh()` 方法

Spring 中 `AbstractApplicationContext` 类中的 `refresh()` 方法即为典型模板方法。它定义了启动应用上下文的标准流程，包括准备刷新、初始化 BeanFactory、注册处理器、初始化消息源与事件广播器、执行 Bean 后处理器、完成刷新等多个阶段。子类可通过重写诸如 `onRefresh()` 等钩子方法插入自定义逻辑，但无法改变主流程执行顺序，从而保证框架初始化步骤的一致性与可扩展性。S

#### 示例二：MyBatis 中 Executor / SqlSession 模板机制

MyBatis 中的 `BaseExecutor` 类采用模板方法模式定义数据库操作的流程（如执行更新或查询）。具体执行逻辑由子类（如 `SimpleExecutor`、`BatchExecutor`、`ReuseExecutor`）实现，而公共流程（如缓存管理、异常处理）由父类统一控制。客户端无需关心执行细节，只需调用模板方法即可完成操作。  
此外，在 MyBatis-Spring 整合中，`SqlSessionTemplate` 也负责管理事务、会话生命周期、异常转换等公共步骤，模板控制调用流程，细节由子类钩子方法或参数注入完成。B

#### 示例三：JdbcTemplate 的 JDBC 操作封装

Spring 的 `JdbcTemplate` 提供了一套 JDBC 操作流程模板：获取连接 → 创建语句 → 设置参数 → 执行回调接口中的 SQL 操作 → 规范关闭与异常翻译。用户只需提供 `StatementCallback` 或 SQL 语句，不必处理连接管理与错误处理逻辑。模板方法模式保证流程标准统一，并支持自定义 SQL 执行逻辑。

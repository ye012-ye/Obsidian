### 1. 服务熔断（Circuit Breaker）是什么？

服务熔断是一种分布式系统容错机制，用于快速隔离故障服务，防止故障蔓延。其原理是：当对下游服务的调用错误率或响应延迟超过预设阈值时，熔断器“断开”（Open 状态），后续调用立即失败或触发 fallback，而不再实际调用该服务。当熔断冷却时间结束，进入 “Half‑Open” 状态，允许少量请求探测服务是否恢复，若成功则恢复为 Closed 状态，否则继续保持 Open。

熔断器的三种主要状态如下：

1. **Closed**：正常调用，监控错误率/延迟；
2. **Open**：超过阈值，拒绝调用并快速返回失败或 fallback；
3. **Half-Open**：试探阶段，允许少数请求，如果恢复就关闭熔断器，否则继续保持 Open。

### 2. 服务降级（Fallback）是什么？

服务降级是一种在服务调用失败或超时的情况下，主动调用备用逻辑或返回默认结果的策略，以保障系统核心功能的可用性。降级通常在熔断器触发时执行，也可单独使用，如网络延迟、资源高峰等场景下提供静态页面、缓存数据或简化流程，以减少调用链对下游的压力。

降级策略包括：

- **自定义备用逻辑**：如缓存值或默认响应；
- **快速失败**：立即返回错误状态；
- **静默失败**：直接返回 null 或空值。

### 3. 熔断 vs 降级：区别与联系

|  |  |  |
| --- | --- | --- |
| **特性** | **服务熔断（Circuit Breaker）** | **服务降级（Fallback）** |
| 作用机制 | 检测调用失败或高延迟后短路调用，防止继续调用故障服务 | 返回备用响应或默认数据，保证核心功能可用 |
| 触发条件 | 错误率或延迟超出阈值 | 调用失败、超时、熔断器打开等场景 |
| 执行方式 | 自动拦截调用，停止请求传播 | 提供备用逻辑，如缓存、默认值、空响应等 |
| 恢复逻辑 | 冷却后 Half‑Open 探测，看能否恢复 | 通常无需主动恢复，等待上游熔断器关闭即可 |

熔断器和降级策略通常配合使用：当熔断触发时，由降级逻辑接管请求，从而避免用户等待或系统崩溃。

### 4. 常见实现方案

- **Spring Cloud Netflix Hystrix**：支持熔断、线程隔离、降级 fallback 和请求缓存等。但已停止维护。
- **Resilience4j**：轻量级且性能优异的熔断库，具备类似 Hystrix 的功能，并支持 Spring 集成。
- **Alibaba Sentinel**：提供流量控制、实时监控、熔断降级等功能，适用于 Spring Cloud Alibaba 生态。
- **Dubbo**：RPC 框架自带熔断与降级配置能力，无缝与服务整合。

### 1. 使用刷新令牌获取新的访问令牌

当 Access Token 到期时，客户端应使用 `grant_type=refresh_token` 将刷新令牌发送至授权服务器以请求新的 Access Token（以及可选新 Refresh Token）。此方式无需用户再次登录，提升用户体验，流程常见于后台服务或移动端应用中。

### 2. 刷新令牌安全存储与传输

- **存储**：刷新令牌属于高价值凭证，应存放在受信任环境中（如服务器端数据库或加密安全存储设备），避免在浏览器 LocalStorage 或前端存储，防止被恶意脚本窃取。
- **传输**：令牌传输仅应使用加密通道（HTTPS/TLS），确保数据安全不被中间人窃取或篡改。

​

### 3. 启用刷新令牌轮换（Rotation）与滥用检测

- **令牌轮换**：每次使用刷新令牌换 Access Token 时，授权服务器应颁发一对新的 Access + Refresh Token，并立即废弃旧的 Refresh Token，实现一次性使用机制。这种策略大幅降低被截获后继续滥用的风险。
- **滥用检测（Reuse Detection）**：服务器应检测同一令牌被重复使用的行为。一旦检测到旧 Refresh Token 被再次提交，应立刻撤销该令牌家族下所有后续令牌，并要求用户重新登录，防止潜在的攻击。

### 4. 令牌生命周期与撤销机制

- **短生命周期策略**：Access Token 建议设置为短期有效（如 15–30 分钟），Refresh Token 可以设置为较长有效期（如数天或数周），并支持闲置过期与绝对过期机制。
- **撤销机制**：当用户登出、密码修改或检测异常活动时，应调用授权服务器的撤销接口（revocation endpoint），立即使 Refresh Token 和关联 Access Token 无效，以防被盗用。

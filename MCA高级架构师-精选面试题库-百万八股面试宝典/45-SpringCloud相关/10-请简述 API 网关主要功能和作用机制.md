API 网关（API Gateway）是微服务架构中用于**集中管理客户端与后端服务通信的反向代理中间层**。它提供统一入口，并承担路由、认证、安全、监控等跨切面功能，全方位提升系统治理效率和安全性。

M

### 1. 中心入口、请求路径统一

API 网关为客户端提供唯一访问入口（单点 URL），隐藏后端服务的具体结构与地址。客户端无需持有多个微服务调用信息，即可通过网关访问系统所有接口，提升调用简洁性和解耦性。

### 2. 路由与请求聚合

网关会根据 URI、Header 或某些规则将请求转发至不同服务，并支持将多个后端服务的调用结果聚合为一个统一响应返回客户端。这减少了前端到后端的多次通信次数，提升性能和体验。

### 3. 负载均衡 & 健康检查

API 网关可将请求分发至后端服务池中的多个实例，并执行健康检查，仅把流量导向健康节点。同时支持流量控制、限速、熔断等治理能力。

S

### 4. 认证、授权与安全防护

在网关层处理客户端认证与权限控制，引入 OAuth、JWT、API Key 等机制验证访问合法性，并统一执行访问控制、安全策略与流量限速，减轻后端服务负担。

### 5. 缓存与协议转换

支持响应缓存以减少重复调用，同时可对请求/响应做协议或数据格式转换（例如 HTTP 转 gRPC、JSON 转XML），方便不同客户端或服务间异构通信。

B

### 6. 监控日志与治理

网关负责统一日志采集、请求指标统计（延迟、错误率、流量等），输出监控数据与链路信息，支持故障诊断和系统优化。

​

总结：API 网关作为客户端与微服务之间的核心中间层，为系统提供：

- 简化的客户端调用接口。
- 动态路由与结果整合。
- 统一安全与访问控制。
- 缓存、协议桥接与限流治理。
- 全局监控与性能管理。

它有效屏蔽了后端复杂性，让客户端调用更加统一、稳定与安全。

下面详细阐述 Eureka 如何通过集群化部署、客户端缓存和自我保护机制来实现高可用：

首先，Eureka 推荐以 **多实例集群** 的方式部署多个 Eureka Server 节点，使其彼此互为镜像，形成对等关系。这样当某个节点挂掉后，客户端仍可访问其他实例，服务注册信息不会中断。各实例启动时会相互同步注册表，保证各节点拥有相同的服务元数据备份。M

其次，每个服务消费者（Eureka Client）会缓存从 Eureka Server 拉取的注册表数据，即使所有 Eureka Server 突然不可用，客户端仍能基于缓存完成服务调用，具备一定的容灾能力。

再次，Eureka 引入了 **自我保护机制**（Self‑Preservation Mode）：当注册中心接收到的心跳数量突然低于预期阈值（默认 85%），Eureka Server 会进入保护模式，暂停剔除实例，即使心跳停滞也不会将服务直接下线，以避免网络波动引起的误杀已存服务实例。这样可以防止由于短期网络问题导致大规模实例下线的情况，提升整体稳定性。

除此之外，Eureka 提供 **客户端负载均衡**，通过 Spring Cloud LoadBalancer 或 Ribbon 等策略，消费者从注册中心获取实例列表后，会基于轮询、随机或加权选择最佳服务实例执行调用，从而分散负载，也增强了调用方的容错能力。S

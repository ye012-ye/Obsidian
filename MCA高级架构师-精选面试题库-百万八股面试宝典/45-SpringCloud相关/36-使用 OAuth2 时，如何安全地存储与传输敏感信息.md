### 1. 客户端不该存储用户密码

OAuth2 协议设计本身即避免客户端持有或存储用户的用户名和密码。**只有授权服务器应接收用户凭据**并验证后再颁发临时凭证（Access Token / Refresh Token）。其他服务只需使用 Token 进行认证和授权。M

### 2. 避免使用 Resource Owner Password Credentials 流程（密码模式）

密码模式要求客户端直接接收并提交用户名密码，这意味着客户端必须高度可信，且不支持多因素认证。多数现代系统不推荐此方式，除非在强控制场景中使用。

### 3. 存储凭证要采用安全机制

- **访问 Token / 刷新 Token 存储**：

- Web 应用（SPA）：推荐使用 **HttpOnly + Secure + SameSite** 的 Cookie 存储令牌，避免 XSS 攻击。
- 移动 App 或后台服务：令牌存于 **安全存储设备** 或后端服务器数据库，确保加密保存和不可轻易访问。

S

### 4. 传输敏感信息必须加密

所有交互应使用 **HTTPS / TLS 加密通道**，确保令牌和用户凭据在网络上传输时不被窃听或篡改。

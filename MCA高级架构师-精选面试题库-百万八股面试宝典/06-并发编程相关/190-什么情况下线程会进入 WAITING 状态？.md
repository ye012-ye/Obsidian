在 Java 中，线程进入 `WAITING` 状态表示它正在等待其他线程执行某个特定的操作，且这种等待是无限期的，直到被其他线程显式地唤醒。以下是常见的使线程进入 `WAITING` 状态的情况：M

### 1. 调用 `Object.wait()` 方法（无超时）

当线程调用 `Object.wait()` 方法时，它会释放当前对象的监视器锁，并进入等待状态，直到其他线程调用同一对象的 `notify()` 或 `notifyAll()` 方法来唤醒它。

### 2. 调用 `Thread.join()` 方法（无超时）

当一个线程调用另一个线程的 `join()` 方法时，调用线程会进入等待状态，直到被调用的线程执行完毕。S

### 3. 调用 `LockSupport.park()` 方法

`LockSupport.park()` 方法使当前线程进入等待状态，直到被其他线程调用 `unpark()` 方法来唤醒它。

需要注意的是，线程在进入 `WAITING` 状态时，会释放所有的锁，并且不会占用 CPU 资源。如果线程长时间处于等待状态，可能会导致 CPU 的空闲，因此需要注意避免出现不必要的等待。B

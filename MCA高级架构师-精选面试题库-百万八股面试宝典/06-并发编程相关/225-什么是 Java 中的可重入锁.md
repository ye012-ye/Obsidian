在 Java 并发编程中，**可重入锁（ReentrantLock）** 是一种允许同一线程重复获取同一个锁而不会被自身阻塞的锁机制：

首先，ReentrantLock 会跟踪持有者线程及其内部的 **锁计数（hold count）**。线程首次调用 `lock()` 时，计数设为 1；若同一线程再次调用 `lock()`，计数会递增，但不会阻塞。只有当调用 `unlock()` 的次数与获取锁的次数相等时，锁才真正释放。

这种机制的益处体现在**递归调用**或方法嵌套调用同一资源时，避免线程因自身再次获取锁而自死锁。例如：

```java
lock.lock();
try {
    methodA();
} finally {
    lock.unlock();
}

void methodA() {
    lock.lock();
    try {
        // 嵌套锁定安全
    } finally {
        lock.unlock();
    }
}
```

因为持有锁的线程再次调用加锁，不会阻塞自身 。

但是，**使用可重入锁时必须严格平衡锁的获取与释放**。如果嵌套调用中某次忘记释放，会导致锁始终被该线程占有，其他线程永远无法获取，形成假死状况。

总而言之，可重入锁简化了多层方法调用时的同步处理，并有效避免线程对自身的死锁。但在使用时必须注意正确的 `lock()`／`unlock()` 配对，以及释放次数匹配。

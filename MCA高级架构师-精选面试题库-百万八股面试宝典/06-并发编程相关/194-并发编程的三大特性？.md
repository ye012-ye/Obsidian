在并发编程中，原子性、可见性和有序性是确保多线程程序正确性和高效性的三大特性。这三者共同构成了 Java 内存模型（JMM）的基础，理解并正确使用它们是编写高效且正确的并发程序的关键。M

### ​原子性（Atomicity）

原子性指的是某个操作要么完全执行，要么完全不执行，不能被中断。在多线程环境中，原子性问题通常出现在对共享变量的操作上，尤其是复合操作（如 `i++`）容易引发竞态条件。例如，两个线程同时执行 `i++` 操作时，可能会出现读取相同值、计算后写回相同结果的情况，导致丢失更新。为确保原子性，可以使用 `synchronized` 关键字、`Lock` 接口或 `java.util.concurrent.atomic` 包下的原子类（如 `AtomicInteger`）来保护共享变量。

### ​可见性（Visibility）

可见性指的是一个线程对共享变量的修改，其他线程能够立即看到。在多核处理器和多级缓存系统中，线程对共享变量的修改可能被缓存到 CPU 的本地缓存中，导致其他线程无法立即看到这些修改，从而引发数据不一致的问题。为确保可见性，可以使用 `volatile` 关键字修饰共享变量，或者使用 `synchronized`、`Lock` 等同步机制。此外，`volatile` 关键字还可以禁止指令重排，确保操作的有序性。S

### ​有序性（Ordering）

有序性指的是程序执行的顺序按照代码的先后顺序执行。在多线程环境中，由于编译器优化、JIT 编译和 CPU 的乱序执行，可能导致代码的执行顺序与预期不一致，从而引发错误。为确保有序性，可以使用 `synchronized`、`Lock` 等同步机制，或者使用 `volatile` 关键字修饰变量，配合 `happens-before` 原则，确保操作的执行顺序。B

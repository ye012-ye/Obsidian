伪共享（False Sharing）是多线程编程中的一种性能瓶颈，尤其在多核处理器环境下尤为显著。它发生在多个线程并发访问位于同一缓存行内的不同变量时，导致频繁的缓存同步和性能下降。

**伪共享的成因：**

现代CPU采用缓存一致性协议（如MESI协议）来保持多核处理器间的缓存一致性。当一个线程修改缓存行中的某个变量时，整个缓存行会被标记为“脏”，并通知其他核心的缓存进行刷新，即使其他核心并未修改该缓存行中的数据。这种无谓的缓存同步会导致性能下降。

**伪共享的示例：**

以下是Java代码示例：

```java
public class FalseSharingExample implements Runnable {
    public static final int NUM_THREADS = 4;
    public static final long ITERATIONS = 5000000000L;
    private final int arrayIndex;
    private static ValuePadding[] values;

    public FalseSharingExample(int arrayIndex) {
        this.arrayIndex = arrayIndex;
    }

    public static void main(final String[] args) throws InterruptedException {
        Thread[] threads = new Thread[NUM_THREADS];
        values = new ValuePadding[NUM_THREADS];
        for (int i = 0; i < values.length; i++) {
            values[i] = new ValuePadding();
        }
        final long start = System.nanoTime();
        for (int i = 0; i < threads.length; i++) {
            threads[i] = new Thread(new FalseSharingExample(i));
        }
        for (Thread t : threads) {
            t.start();
        }
        for (Thread t : threads) {
            t.join();
        }
        System.out.println("Duration = " + (System.nanoTime() - start));
    }

    @Override
    public void run() {
        long i = ITERATIONS + 1;
        while (0 != --i) {
            values[arrayIndex].value = i;
        }
    }

    public static class ValuePadding {
        protected long p1, p2, p3, p4, p5, p6, p7;
        public volatile long value = 0L;
        protected long p9, p10, p11, p12, p13, p14, p15;
    }
}
```

在上述代码中，`ValuePadding`类通过在`value`字段前后添加填充字段，确保每个线程操作的`value`字段位于不同的缓存行，从而避免伪共享。

**如何避免伪共享：**

1. **使用填充字段：**  
   在共享变量前后添加填充字段，使每个线程操作的变量位于不同的缓存行，从而避免伪共享。
2. **使用**`@Contended`**注解：**  
   Java 8引入了`@Contended`注解，标记字段或类，指示JVM为其分配独立的缓存行。

```java
@sun.misc.Contended
public class ValueWithContended {
    public volatile long value = 0L;
}
```

使用`@Contended`注解时，需要在启动JVM时添加参数：`-XX:-RestrictContended`。

3. **避免共享状态：**  
   尽量避免多个线程共享状态，将数据封装在线程内部，减少共享数据的需求。

**总结：**

伪共享是多线程编程中的一种性能瓶颈，理解其成因并采取适当的措施可以有效提高程序的性能。通过使用填充字段、`@Contended`注解或避免共享状态，可以减少伪共享带来的性能影响。

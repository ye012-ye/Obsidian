在 Java 并发编程中，线程之间主要通过两种机制进行通信：共享内存方式和消息传递方式。下面分别阐述两者核心特性和适用场景：

M

### 共享内存方式

Java 中线程共享同一个堆内存区域，通过访问共享变量或数据结构来交换状态。为保证可见性和防止竞态条件，通常会使用 `volatile` 关键字或同步机制（如 `synchronized`）。在经典的生产者-消费者模型中，线程可以调用对象的 `wait()` 方法释放当前锁并进入等待队列，等待其他线程调用 `notify()` 或 `notifyAll()` 来唤醒其继续执行。这种方式效率较高，但开发者需仔细处理锁和状态变量的同步。M  
示例：`while (!condition) { lock.wait(); } … lock.notifyAll();`

S

### 消息传递方式

这种方式通过线程安全的消息队列、信号量或栅栏等机制将数据封装成“消息”发给另一线程。典型的是使用 `BlockingQueue` 在多个线程之间传递任务或数据项，或者通过 `Semaphore` 控制许可数量，从而协调并发访问。这种设计降低线程之间的耦合度，避免共享可变状态带来的复杂性。S  
例如：`queue.put(msg); otherThread takes from queue;`

B

### 总结

- **耦合度**：共享内存要求线程操作相同的对象或变量，并管理竞争状态；消息传递通过消息结构隔离线程，耦合更松。
- **复杂性**：共享方式需要同步控制、锁逻辑和异常处理；消息传递更简洁安全，常依赖标准 concurrent 工具类。
- **性能上**：共享方式一般延迟低、效率高；消息传递有序阻塞开销稍大，但易维护、更可靠。B

总结：如果希望线程高效地共享状态且能精确控制同步，就选用共享内存 + wait/notify 等机制；若偏向设计清晰、安全、易维护，尤其在生产-消费或任务分发场景下，消息传递机制（如阻塞队列和信号量）更优。

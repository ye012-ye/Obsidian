调用 `Thread.start()` 会触发 JVM 创建一个新的操作系统线程，这个线程拥有独立的执行栈，并在该线程中调用 `run()` 方法执行逻辑，从而实现真正的并发执行。  
而直接调用 `run()` 方法，只是普通的方法调用，它在当前线程（调用者所在的线程）中执行，不会创建新的线程，因此无并发性可言，所有逻辑仍在主线程中顺序执行。M

在 Java 的线程模型中，`start()` 和 `run()` 方法的差异至关重要：

- `start()` 会将线程从“新建”状态推进至“可运行（Runnable）”状态，并由线程调度器分配时间片，最终进入 “Running” 状态执行 `run()` 方法。
- 直接调用 `run()`，不会经历线程状态转换，执行仍在当前线程，没有异步或多线程效果。S

此外，还存在以下区别与注意事项：

1. **执行上下文不同**：使用 `start()` 启动时，逻辑在新线程中执行；用 `run()` 调用时，逻辑仍在调用线程中执行。
2. **线程重用限制**：一个线程对象只能调用一次 `start()`，否则会抛出 `IllegalThreadStateException`；但可以多次调用 `run()`，因为它只是普通方法。B

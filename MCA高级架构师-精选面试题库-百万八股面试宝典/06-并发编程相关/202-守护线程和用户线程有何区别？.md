Java 中线程分为两类：**用户线程**（User Thread）与 **守护线程**（Daemon Thread），二者几乎在行为逻辑上没有区别，但 **JVM 生命周期处理方式**不同。

### 一、核心区别

当所有用户线程结束后，JVM 会自动退出，并 **终止所有仍在运行的守护线程**，而不会等待它们完成。这是两者最本质的区别。也就是说，即使有守护线程在运行，只要没有用户线程存在，JVM 就立即关闭。

守护线程通常用于执行后台服务任务，如垃圾回收、性能监控、日志采集等；而用户线程用于执行核心业务逻辑，决定 JVM 是否继续运行。它们具有以下对比特点：

- **JVM 退出行为**：用户线程存在时 JVM 会等待；守护线程不会阻止 JVM 退出。
- **优先级和生命周期**：守护线程优先级较低，生命周期依赖于用户线程；用户线程优先级高且独立控制生命周期。
- **创建默认值**：默认线程（包括 main 线程）都是用户线程；新线程继承父线程的类型，需显式调用 `setDaemon(true)` 才设为守护线程。

## 示例代码

```java
public class DaemonVsUser {
    public static void main(String[] args) throws InterruptedException {
        Thread userThread = new Thread(() -> {
            try {
                Thread.sleep(3000);
                System.out.println("User thread completed");
            } catch (InterruptedException ignored) {}
        }, "user-thread");
        Thread daemonThread = new Thread(() -> {
            while (true) {
                System.out.println("Daemon thread running...");
                try { Thread.sleep(500); } catch (InterruptedException ignored) {}
            }
        }, "daemon-thread");

        daemonThread.setDaemon(true); // 必须在 start() 前设置
        daemonThread.start();
        userThread.start();

        System.out.println("Main thread exiting");
        // JVM 会在 userThread 完成后退出，不等待 daemonThread
    }
}
```

运行结果展示：daemon 线程会反复打印日志，但在 main 和 user 线程结束后 JVM 立即退出，即使 daemon 还在执行中也随之终止。

## 总结

- **守护线程** 是为用户线程提供后台支持的辅助线程，一旦没有用户线程，JVM 会自动终止所有守护线程；
- **用户线程** 是执行实际任务的主力线程，JVM 会一直等待，直到所有用户线程结束；
- 默认线程类为用户线程，若需创建守护线程，需在 `start()` 前调用 `setDaemon(true)`；

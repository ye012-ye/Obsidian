不可变对象（Immutable Object）是指其状态在创建后无法被修改的对象。在 Java 中，`String` 类、基本数据类型的包装类（如 `Integer`、`Double` 等）以及通过 `final` 修饰的类和字段通常被视为不可变对象。不可变对象的设计理念是：一旦对象被构造，其内部状态就被固定，不允许任何修改。M

### 不可变对象的并发优势

1. **天然的线程安全性**

由于不可变对象的状态在创建后无法改变，因此多个线程可以安全地共享同一个不可变对象实例，而无需额外的同步机制。这避免了由于线程间对共享可变数据的竞争而导致的同步问题，如死锁、竞态条件等。S

2. **减少同步开销**

传统的同步机制，如 `synchronized` 或 `ReentrantLock`，用于保护共享可变数据的访问，可能引入性能瓶颈。不可变对象由于其状态不可变，天然地避免了同步的需求，从而减少了上下文切换和锁竞争带来的性能开销。

3. **简化代码理解和维护**

不可变对象的状态在创建后不再变化，这使得代码的行为更加可预测。开发者无需担心其他线程在访问对象时可能引发的副作用，从而提高了代码的可读性和可维护性。B

4. **支持函数式编程风格**

不可变对象鼓励无副作用的函数式编程风格。在这种风格下，函数的输出仅依赖于输入参数，不受外部状态的影响，易于测试和推理。

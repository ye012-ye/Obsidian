在 Docker 中，为容器创建可访问的网络是实现容器间通信和服务隔离的关键步骤。Docker 提供了多种网络模式，以下是常用的两种方式：

M

1. **使用默认的桥接网络（bridge）：**

Docker 在安装时会自动创建一个名为 `bridge` 的默认网络。容器在启动时，如果未指定网络，默认会连接到此网络。

例如，启动一个容器并连接到默认桥接网络：

```bash
docker run -d --name my-container -p 8080:80 nginx
```

此命令将启动一个名为 `my-container` 的容器，使用 `nginx` 镜像，并将容器的 80 端口映射到宿主机的 8080 端口。

容器之间可以通过 IP 地址进行通信，但推荐使用容器名称进行通信，以提高可维护性。

S

2. **创建自定义桥接网络：**

为了实现更好的容器间通信和服务发现，可以创建自定义的桥接网络。

创建自定义桥接网络的命令：

```bash
docker network create --driver bridge my-custom-network
```

启动容器并连接到自定义网络：

```bash
docker run -d --name my-container --network my-custom-network nginx
```

在自定义网络中，容器可以通过名称互相通信，且具有更好的隔离性和可管理性。

**总结：**

- 默认桥接网络适用于简单的容器部署，但容器间通信仅限于 IP 地址。
- 自定义桥接网络提供了更好的容器间通信能力和服务发现机制，适用于复杂的应用场景。
- 根据实际需求选择合适的网络模式，以确保容器间的高效通信和系统的安全性。

### 一、主从复制 + Sentinel 故障自动切换

Redis 通过主从复制（master-replica）机制，将主节点的数据异步同步到一个或多个从节点。当主节点不可用时，从节点可被提升为新的主节点。

Redis Sentinel 负责监控主从节点状态，满足一定故障共识（quorum）后自动执行故障转移：将某个从节点晋升为主节点并重新配置其他副本，同时向客户端广播新主地址。Sentinel 同时提供监控、通知、配置提供者等服务功能。

**适用场景：**中等数据量（如 <200GB），无需数据分片且对一致性、可用性要求高的业务场景。Sentinel 架构相对简单、易部署，也适合单实例容灾需求。

S

### 二、Redis Cluster 分片 + 自动故障恢复

Redis Cluster 将数据分成预设的 16384 个哈希槽（hash slots），自动将不同槽的数据分布在多个主从节点上。每个主节点配有一个或多个从节点，共同形成分片集群。Cluster 可自动处理某些节点故障，完成故障恢复与槽位迁移。

**适用场景：**数据量巨大，需要横向扩展读写能力、存储容量与吞吐量时优选 Cluster。Redis Cluster 能同时处理可用性和规模扩展问题。

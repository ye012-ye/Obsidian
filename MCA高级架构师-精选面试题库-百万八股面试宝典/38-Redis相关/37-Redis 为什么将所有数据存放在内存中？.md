Redis 将所有数据放入内存，主要是为了在高并发场景下提供极致的性能和低延迟。

以下是几个关键原因与优势：

M

### ​1. 读写速度快

- **内存访问速度远高于磁盘**：相比于 SSD 或 HDD，RAM 的读写延迟通常低数十倍到上百倍，能够在纳秒或微秒级完成操作。
- **单线程 + 多路复用**：Redis 利用单线程处理数据，通过内存访问无需锁竞争，避免上下文切换，并提升 CPU 利用率。

S

### 2. 简单高效的数据结构直接内存操作

Redis 的内部数据结构（如哈希表、跳表、SDS）直接映射内存，实现操作效率最高。

B

### 3. 可选持久化与恢复机制

虽所有数据在内存中，但 Redis 可选 RDB（快照）和 AOF（追加日志）持久化方式：

- **RDB**：定期创建数据快照；通过 `fork()` 方式异步持久化，避免阻塞主线程。
- **AOF**：记录每次写操作日志，重启时回放恢复状态。

在默认配置下，如果 Redis 重启，最多只会丢失最近几秒内的数据。

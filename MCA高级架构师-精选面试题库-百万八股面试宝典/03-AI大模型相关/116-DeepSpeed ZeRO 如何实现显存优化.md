### 1. 三大阶段显存分区设计

- **ZeRO‑1**：仅分割优化器状态（如 Adam 的动量与方差）。每个 GPU 只保存其负责部分，从而优化器相关显存降低约 4 倍。
- **ZeRO‑2**：在 ZeRO‑1 的基础上继续将梯度分片至各 GPU，仅保留对应参数梯度，大约可实现整体显存压缩 8 倍左右。
- **ZeRO‑3**：最激进阶段，进一步将模型参数也分片至不同 GPU。在前向与反向计算时自动收集与拆分参数，实现显存再压缩，适合训练百亿以上模型。

M

### 2. Offload 策略与无限显存扩展

- **ZeRO-Offload**：在 ZeRO‑2 架构中支持将优化器状态和梯度放置于 CPU 或 NVMe，适用于 GPU 显存受限的场景。
- **ZeRO-Infinity**：在 ZeRO‑3 基础上进一步 offload 参数至 CPU/NVMe，实现近乎无限显存支持（理论上可训练千亿甚至万亿参数模型）。

### 3. 通信和精度优化设计要点

- ZeRO 在通信上使用 **reduce 和 all-gather 桶机制**，将大型 tensor 分块同步以降低通信次数；同时启用 **gradient accumulation** 掩盖通信延迟，提升吞吐率。
- 与 **混合精度训练（如 BF16 或 FP16）** 协同工作，可减少参数精度导致的显存消耗，并进一步提升训练速度及效率。

### 4. 总结对比

|  |  |  |  |
| --- | --- | --- | --- |
| **阶段** | **拆分对象** | **显存缩减** | **适用模型规模** |
| **ZeRO‑1** | 优化器状态 | ≈ 4× | 中小模型微调 |
| **ZeRO‑2** | 优化器状态 + 梯度 | ≈ 8× | 中等规模模型训练 |
| **ZeRO‑3** | 优化器状态 + 梯度 + 参数 | ≈ 16× 或更多 | 大模型（百亿参数及以上） |

ReAct（Reason + Act）是一种 Agent 设计范式，它将 **语言模型的推理能力**与 **外部工具操作能力**结合起来，通过反复思考、行动、观察的循环，解决复杂任务。M

**1. 推理与行动的协同机制**  
ReAct 不仅要求 LLM 生成内部思考（思考阶段），还要求根据思考结果触发具体操作（行动阶段），然后利用工具反馈（观察阶段）来更新其推理过程。每一步都包括“思考→行动→观察”，推动 Agent 在动态环境中迭代解决问题。这种方式兼顾了 Chain-of-Thought 的细致推理与 API 或工具调用的外部数据获取能力，是 CoT 与 Self‑Ask 等方法的融合集成。

**2. 动态决策循环结构**  
Agent 启动后，会按照以下模式不断循环：先内部推理确定下一行动目标，接着调用工具（如搜索引擎、数据库、API 等）获取信息，将结果作为观察反馈，再继续推理并决定下一步行动，直到满足终止条件，比如找到答案或超过最大步数。这样每个决策周期都基于最新信息，使 Agent 更可靠且更可解释。S

**3. 工程实现关键要素**  
典型实现中包括：

- **Tools 工具集**：定义可调用接口及其描述，供 LLM 选择使用。
- **Tracing 记录轨迹**：思考—行动—观察逐步存储，便于后续跟踪与调试。
- **Prompt 设计**：为 LLM 提供模板与几例演示，引导它遵循思考→行动→观察顺序生成内容。

**4. 方法优势与应用场景**  
ReAct 框架可显著减少模型幻觉产生，提高任务执行的准确性与可信度。在复杂 QA、交互式决策任务、基于工具的自动化流程中，ReAct 通常优于单纯推理或单纯工具调用的方法。此外，其轨迹可读性高，有利于可解释性与失败复盘。

B

### 总结

ReAct 是一种将语言模型**思维决策能力**与**工具调用能力**紧密结合的框架，通过反复的**思考‑行动‑观察循环**让 Agent 能够在复杂任务中动态规划、执行并调整推理路径。其优势在于任务表现优异、结果可追踪、适配多种工具和场景，是当前 LLM Agent 系统的重要基石。

**1. Top‑K 稀疏激活原理**  
Mixture‑of‑Experts (MoE) 架构通过轻量的门控网络（router），对每个输入 token 计算所有专家的得分，然后仅激活 Top‑K 个专家进行计算，其余专家不参与该 token 的处理。这种稀疏激活方式可以显著节省推理计算量和显存资源，同时保持大模型的参数容量与表达能力（Top‑1 或 Top‑2 是常用配置）。

**3. 工程实现细节**

- **门控网络设计**：轻量结构特征，例如线性层 + ReLU + Softmax，并加入噪声项以增强探索性；多数情况下设置为两层 MLP 或线性变换结构，参数量很小，经常低于 0.1M。
- **容量控制**：每个专家设置 capacity factor（通常 1.25–2 倍 batch\_size/专家 数量），限制每个专家在一个 batch 中最多处理的 token 数量，以避免某些专家成为瓶颈或丢 token 问题。
- **优化通信**：在分布式环境下，可采用异步通信（如 NCCL）、专家并行与 pipeline 并行组合方式降低同步延迟。

**4. 常见实践案例**

- 在金融风控或文本摘要等任务中，采用 Top‑2 路由可使单个 token 激活的参数量约为全模型的 1.5%–2%，计算成本显著降低；同时通过平衡损失机制，专家利用率提升到 80% 以上；推理吞吐率提升数倍，而精度通常仅下降极小（<1%）。
- Switch Transformer 应用 Top‑1 路由设计，以更简洁路由控制获得高速推理；而 GShard、GLaM 等采用 Top‑2 路由，在多个专家中动态选择以平衡性能与效率。

在部署和服务大型模型的过程中，**量化技术**是提升推理效率和减少资源消耗的重要手段。常见类型包括 FP16 / BF16（混合精度）、INT8、INT4 等，它们之间在性能、精度和资源使用方面存在明显差异。M

- **FP16 / BF16 混合精度**：是使用 16 位浮点数替代 FP32，通常用于训练或推理阶段。此类量化可将模型显存占用减半，推理速度提升约 1.5 倍，且对精度影响非常小，是最常用的入门方案。
- **INT8 量化**：将权重与部分激活精度降低至 8 位整数，模型大小和内存带宽减少约 4 倍，推理延迟通常提升 2 至 4 倍，精度损失极小（≤1%），广泛适用于 CPU/GPU 等环境。PyTorch 支持端到端动态量化，NVIDIA TensorRT 等工具也提供良好支持。
- **INT4 / GPTQ / AWQ 量化**：使用 4 位整数或更低精度，是当前最激进的量化方法。GPTQ 是一种基于近似二阶优化的 post-training quantization，可将权重量化至 3～4 位，速度提升在 FP16 基础上约 3.2×（A100 GPU）或更多，精度损失可控。微软 DeepSpeed 的 INT4 流水线对 BERT 推理延迟提升高达 8.5×（延迟优先场景）/3×（吞吐优先场景）。AWQ 则通过激活感知缩放进一步提升速度，并在多个模型中表现稳定。

### 方法对比

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **精度类型** | **显存 / 模型大小** | **推理速度提升** | **通常精度损失** | **适用场景** |
| FP16 / BF16 | ↓约 50% | ×1.5× | 极小 | 通用场景、混合精度训练/推理 |
| INT8 | ↓约 75% | ×2–4× | ≦1% | 延迟敏感部署、CPU/GPU 节省资源 |
| INT4 (GPTQ) | ↓约 90% | ×3.2–4.5×（取决硬件） | 轻微 | 移动端、大模型边缘部署 |
| INT4 (AWQ) | ↓约 90% | 比 GPTQ 快约1.45× | 极小 | 需要更快推理或更高吞吐率的任务 |

## 示例回答

我们在多个项目中通过量化优化模型推理性能。在客服问答系统中，对 BERT 模型做动态 INT8 量化，并结合 TensorRT 编译优化，使延迟从约 200ms 降至 50ms，显存占用从 1.2GB 降到 0.3GB，F1 精度损失仅约 0.8%。  
在另一个案例中，我们使用 GPTQ 方法将 LLaMA‑7B 或 Qwen 模型从 BF16 压缩至 INT4，再结合 AutoGPTQ 框架加载进行推理，推理速度提升 3.5 倍，模型大小减少约 90%，且生成质量基本一致，相当于在边缘设备上运行大型 LLM。AWQ 方法则在多模型上比 GPTQ 速度更快约 1.45 倍，并在吞吐场景中效果更佳。

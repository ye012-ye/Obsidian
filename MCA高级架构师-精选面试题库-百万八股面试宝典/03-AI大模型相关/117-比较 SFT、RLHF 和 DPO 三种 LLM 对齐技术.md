### 1. 技术机制差异

**SFT（监督微调）**  
采用输入-输出对进行有监督训练，让模型学习示例中的“正确”输出。其优点是训练稳定、数据利用效率高（少量样本即可取得效果），但无法从人类偏好中学习行为倾向，缺乏处理“拒绝有害内容”的能力。

**RLHF（人类反馈强化学习）**  
构建奖励模型（Reward Model）以估计人类偏好，然后通过 RL（如 PPO）优化策略，最大化奖励同时加入 KL 惩罚避免模型偏离 SFT 初始化。优势在于高精度对齐人类价值，能够拒绝不良或不安全内容；劣势是训练流程复杂、标注成本高、RL 不稳定。

**DPO（直接偏好优化）**  
跳过显式奖励模型训练，直接基于偏好对数据（preferred vs rejected）优化策略，等效于 RLHF 但采用分类式损失函数。训练更简洁、高效，避免 RL 的不稳定性和复杂调参；适配二元偏好数据，依赖 SFT 基础好且偏好数据质量高时效果良好；但在某些任务上略逊于精调 PPO RLHF，且对梯度平衡处理敏感。

### 2. 适用场景比较

|  |  |  |  |
| --- | --- | --- | --- |
| **技术** | **优势** | **局限** | **推荐场景** |
| **SFT** | 简单高效，标注量小 | 不对齐人类偏好，不能拒绝有害内容 | 基础问答、小样本领域 |
| **RLHF** | 人类价值高对齐，安全性强 | 标注成本高、训练复杂、不稳定 | 医疗、金融等高风险场景 |
| **DPO** | 实施简洁、高效稳定、数据要求低 | 依赖偏好数据质量，有时效果略低于 PPO | 快速迭代、成本敏感业务落地 |

### 3. 总结

- **技术演进脉络**：SFT 提供基础准确性，RLHF 强化对齐与安全性，DPO 提供兼顾效率与对齐效果的替代方案；
- 在医疗/金融等高安全需求场景，RLHF 仍是首选；在快速迭代、资源受限或小规模偏好数据环境，可优先采用 DPO；多数普通业务场景，SFT+DPO 的组合即可达到性能与成本的平衡。

### 1. 全参数微调（Full Fine‑Tuning）

**定义**：对预训练大模型的所有参数（通常为数十亿至数百亿）进行训练以适配下游任务。

**资源需求**

- 需加载并更新全部模型参数，显存占用极高（例如7B模型需约28 GB GPU显存）。
- 训练时间长，成本高，容易产生灾难性遗忘。

**性能表现**

- 性能上线高，适合复杂任务场景，但容易过拟合，稳定性较差。
- 微调结果可能偏离预训练分布，影响泛化表现。

### 2. LoRA（Low-Rank Adaptation）

**原理**：冻结基础模型，向每层注意力矩阵中注入两个低秩矩阵（A 和 B）学习任务差异，通过 ΔW = ABᵀ 实现参数调整。

**资源消耗**

- 仅训练新增低秩矩阵，通常仅占模型参数的 0.1%–1%，显存显著降低（≈原模型显存的33%）。

**训练与性能**

- 更快训练速度，避免加载整个模型；推理阶段无额外延迟。
- 在大多数下游任务上性能可接近全参数微调，几乎无精度损失。
- LoRA 更新引入“intruder dimensions”，改变参数空间结构，对预训练分布的保持力不如全调。

### 3. QLoRA（Quantized LoRA）

**原理**：在 LoRA 的基础上对预训练模型权重进行 4-bit 量化（如 NF4），训练时保留高精度计算，LoRA adapter 保持高精度格式。

**资源消耗**

- 显存需求进一步降低（可节省到 LoRA 的 25%），支持巨型模型在单卡上微调，例如 65B 模型。

**训练与性能**

- 比 LoRA 多一点量化开销，训练稍慢，表现略低但可通过适配器微调补偿精度。
- 在多个 benchmark 上性能达 LoRA 的 95%+，在 instruction tuning 中表现接近 full fine‑tuning。

### 4. 对比

|  |  |  |  |  |  |
| --- | --- | --- | --- | --- | --- |
| **方法** | **可训练参数** | **显存需求** | **训练速度** | **性能表现** | **应用场景** |
| 全参数微调 | ≈100% | 极高 | 最慢 | 理论最优，但易过拟合 | 资源充足、对性能极端要求的任务 |
| LoRA | 0.1%–1% | 显卡显著下降 | 快速 | 接近全调，稳定性好 | 中大型模型任务，资源有限且需多任务适配 |
| QLoRA | LoRA + 4-bit base | 极低（4×更省） | 稍慢于 LoRA | 微降（可微调补偿） | 超大模型（如 ≥33B/65B）、显存受限环境适配 |

模型蒸馏常见的培训方案主要有 **离线蒸馏（Offline）**、**在线蒸馏（Online）** 和 **自蒸馏（Self‑Distillation）**，并扩展到 **多教师蒸馏 (Multi‑Teacher)**、**知识迁移蒸馏** 等变种。下面分别说明各自特点、适用情况及优劣。

M

### 1. 离线蒸馏（Offline Distillation）

- 教师模型**先训练并固定**，学生模型基于教师输出 logits 或特征进行训练。是最早提出的方法，易于实现且稳定可靠。
- **优点**：实现简单，可利用丰富预训练模型；**缺点**：教师固定，无法应对实时业务变化，对教师模型依赖强。
- **典型适用场景**：数据充足、模型预训练允许的场景，如图像分类或批量 NLP 模型压缩。

### 2. 在线蒸馏（Online Distillation）

- 教师与学生模型**同训练同步更新**，通常以互学习（mutual learning）的方式训练，使多网络协作提升能力。
- **优点**：教师可随数据更新自调整，适应动态场景；**缺点**：训练复杂，需同步协调多个模型，且易引入噪声。
- **典型适用场景**：智能客服、对话系统等在线变化明显的业务场景，需要快速响应业务变化。

S

### 3. 自蒸馏（Self‑Distillation）

- 使用同一模型的不同快照、层次或早期输出作为“教师”，对自身模型进行监督。典型形式包括知识从深层传给浅层，或用训练中早期模型指导后期模型训练。
- **优点**：无需额外教师模型，适合数据不足或标注困难场景；**缺点**：容易陷入自反馈的局部最优，需合理温度调节。
- **典型场景**：医学影像、小样本任务等标注稀缺、资源受限环境。

### 4. 多教师蒸馏（Multi‑Teacher Distillation）

- 学生模型同时从多个教师模型学习，例如不同语言或任务的教师联合指导。
- **优点**：丰富知识来源，提高泛化能力；**缺点**：教师选择与权重分配复杂，训练计算资源高。
- **典型场景**：跨语言翻译、多任务学习场景。

B

### 5. 知识源类型：Logit、特征、关系对齐

- **Logit-based**：最基础形式，用 KL 散度拟合教师 logits 概率分布。
- **Feature-based**：中间层特征对齐，例如 FitNet、AT、NST 等，适合 capacity gap 大者传递细粒度知识。
- **Relation-based**：对样本结构、通道或实例间相似性进行模仿，如高阶语义关系。

## 策略对比

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **蒸馏类型** | **核心机制** | **适用场景** | **优点** | **限制与挑战** |
| 离线蒸馏 | 教师先训练，学生模仿 | 数据充足、系统稳定业务场景 | 稳定、便于调试 | 无实时更新能力、教师依赖强 |
| 在线蒸馏 | 教师与学生同步训练 | 实时业务、模型快速迭代场景 | 可适应更新快、泛化强 | 训练复杂、噪声风险高 |
| 自蒸馏 | 模型自身层次或快照知识自监督 | 数据稀缺、轻量边缘部署场景 | 无需额外教师、成本低 | 易陷局部最优 |
| 多教师蒸馏 | 学生融合多个教师知识 | 跨领域、跨语言问题 | 泛化与覆盖全面 | 计算成本高、权重难调 |
| 知识源对齐方式 | logits / feature / relation | 各类蒸馏策略 | 灵活传递不同层次知识 | 对齐策略复杂、需结构匹配 |

###### 基于额外参数的方法

**Adapter Tuning**

在预训练模型的特定层（如 Transformer 的 FFN 层后）插入轻量级的适配模块（Adapter），仅训练适配模块参数，其余参数冻结。适配模块通常由降维和升维的 MLP 组成，通过低秩分解减少参数量，适合多任务场景。

**优点**：模块化设计，易于扩展；**缺点**：增加推理延迟。

**Prefix-Tuning**

在输入序列前添加可学习的连续前缀（Prefix），通过 MLP 重参数化后注入到模型的每一层，仅训练前缀参数。适用于生成任务，如文本摘要。

**优点**：任务适配灵活；**缺点**：序列长度增加，计算开销略高。

**Prompt-Tuning**

仅在输入层添加可学习的连续提示向量（Soft Prompts），通过反向传播优化提示嵌入。随着模型规模增大，效果逼近全量微调。

**优点**：参数极少；**缺点**：小模型效果有限，需长提示词优化。

###### 基于重参数化的方法

**LoRA（低秩适配）**

通过低秩矩阵分解模拟全量微调的权重更新，在原始权重旁添加旁路矩阵（A 和 B），仅训练低秩参数。推理时合并矩阵，不增加计算量。

**优点**：参数效率高（比全量微调少 10-50 倍），支持多任务适配；**缺点**：需选择合适秩（`r`）。

**变体**：

- **QLoRA**：结合模型量化（如 NF4 精度），进一步降低显存占用。
- **AdaLoRA**：动态调整低秩矩阵的秩，优化参数分配。

###### P-Tuning 系列

- **P-Tuning v1**：将离散提示转化为连续嵌入，通过 LSTM+MLP 编码提示向量，增强提示间的关联性。
- **P-Tuning v2**：在每层添加连续提示，移除重参数化编码器，支持多任务学习，显著提升小模型效果。

###### 基于部分参数更新的方法

**BitFit**

仅微调模型中的偏置项（Bias），参数占比不到 2%，适合极低资源场景。

**优点**：实现简单；**缺点**：效果略逊于其他方法。

###### 混合方法

**MAM Adapter**

结合 Adapter 和 LoRA 的优势，通过门控机制动态选择适配方式，平衡参数效率与性能。

**UniPELT**

集成多种 PEFT 方法（如 Adapter、LoRA、Prefix-Tuning），根据任务自动分配适配策略，提升泛化性。

###### 其他创新方法

**ReFT（表征微调）**

通过干预模型隐藏表示而非修改权重，实现参数高效微调。例如 LoReFT（低秩线性子空间 ReFT）训练参数比 LoRA 少 10-50 倍，性能接近全量微调。

**优点**：参数效率极高，可解释性强；**缺点**：需额外计算干预步骤。

###### 方法对比与选择建议

|  |  |  |  |
| --- | --- | --- | --- |
| **方法** | **参数效率** | **适用场景** | **典型应用** |
| LoRA | 高 | 通用任务 | 文本生成、分类 |
| QLoRA | 极高 | 显存受限环境 | 大规模模型微调 |
| Prefix-Tuning | 中 | 生成任务 | 摘要、对话系统 |
| BitFit | 极低 | 极低资源微调 | 快速原型验证 |
| ReFT | 极高 | 高可解释性需求 | 语义理解、常识推理 |

**实践建议**：

- **显存不足**：优先选择 QLoRA 或 BitFit。
- **多任务适配**：使用 LoRA 或 MAM Adapter。
- **生成任务**：Prefix-Tuning 或 P-Tuning v2。

Transformer 的编码器–解码器架构广泛用于机器翻译、摘要生成、对话系统等任务，其核心优势在于完全基于自注意力机制，支持并行和长距离依赖建模。

M

编码器接收输入序列，将其嵌入为高维向量并加入位置编码，然后依次经过多个相同层级的处理，每层包括多头自注意力、前馈网络（Feed‑Forward）模块，以及残差连接和层归一化，目的是生成上下文敏感的 token 表示序列。

解码器在生成阶段结合两种注意力机制：首先执行 **掩码（causal）自注意力**，确保预测下一个 token 时只能访问当前及之前生成的 token；其次执行 **编码器–解码器交叉注意力**，将编码器输出作为 key/value，以 decoder 的 query 获取输入上下文信息；接着通过前馈网络生成输出 token 分布。该机制确保模型在生成时既参考上下文又保持自回归属性。

S

## 示例流程解析（翻译任务为例）

假如输入为英文文本 **“The cat sat on the mat.”**：

- **编码器部分**  
  首先将每个 token 转换为 embedding 加入位置编码，然后每层都进行并行计算注意力，捕捉句子中所有 token 之间的依赖关系，并通过前馈网络提炼信息，最终输出一组上下文表示。
- **解码器部分**  
  初始化解码器输入为特殊起始标记 `<s>`，进行掩码自注意力，防止看到未来 token；接着交叉注意力从编码器提取与上下文相关的信息；最后通过前馈层处理并输出下一个 token 概率分布。

随着每一步生成输出，新的 token 被添加进 decoder 输入序列，迭代进行直到生成结束符 `</s>`。

B

## 示例面试回答

“Transformer 的编码器–解码器结构通过两个堆叠模块协同工作。编码器将输入文本转化为上下文向量表征；解码器则在生成过程中结合历史生成 token（掩码自注意力）和编码器输出（交叉注意力）决定下一 token。

自注意力机制使模型能够捕捉整个序列的依赖关系；残差连接加 LayerNorm 保证了训练的稳定性；并行计算则大幅提升效率。该结构已广泛应用于机器翻译、文档摘要和多轮对话生成等任务。"

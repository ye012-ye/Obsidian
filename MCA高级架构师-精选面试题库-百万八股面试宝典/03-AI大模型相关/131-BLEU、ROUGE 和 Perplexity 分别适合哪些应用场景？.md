在大模型的任务评估中，不同指标适用于不同场景。BLEU 和 ROUGE 倾向于评估生成文本与参考文本的词汇或 n‑gram 匹配程度，而 Perplexity 用于衡量语言模型本身的概率预测能力。M

BLEU（Bilingual Evaluation Understudy）主要衡量机译输出与参考翻译之间的 n‑gram 精确率（precision），并引入 brevity penalty 来惩罚过短输出。它在机器翻译任务中有较高的相关性，但对语义流畅性和长句翻译的语义适应性较弱。

ROUGE（Recall-Oriented Understudy for Gisting Evaluation）则强调召回率（recall），尤其 ROUGE‑N 和 ROUGE‑L 可衡量生成摘要中与参考摘要重叠的 n‑gram 和最长公共子序列。它在摘要、报告自动生成等任务中表现更好，但同样对同义替换或语义重述不敏感。

Perplexity 衡量语言模型对下一个 token 的预测不确定性，一般定义为交叉熵的指数形式。Perplexity 越低表示模型越能准确预测下一个词，它适用于预训练阶段对模型内在预测能力的评估，而非直接评估生成质量 。

### 适用场景对比

|  |  |  |  |
| --- | --- | --- | --- |
| **指标** | **最佳应用场景** | **主要衡量维度** | **局限性与注意点** |
| BLEU | 机器翻译、严格公式化生成任务 | n‑gram 精确率 | 对长句翻译或同义替换敏感性弱；与人工评分相关度低 |
| ROUGE | 文本摘要、报告生成类任务 | 召回覆盖度 | 无法体现语义同义和上下文一致；忽略术语完整性 |
| Perplexity | 语言模型训练与调优阶段 | 模型预测概率准确度 | 对生成文本质量指标表现差；不反映事实性或语义准确性 |

## 示例面试回答

“BLEU 由于强调生成与参考翻译在词汇和顺序上的精确匹配，非常适合用于机器翻译任务中衡量输出质量。但它对文化差异、同义词表达和句式变通的描述支持不足。ROUGE 更注重召回，因此在摘要生成任务中能评估关键信息覆盖度，但仍无法识别语义重述或术语准确性问题。至于 Perplexity，它用于评估语言模型自身的流畅性和预测能力，在预训练或微调阶段非常重要，但并不适合作为生成任务质量评价指标。”

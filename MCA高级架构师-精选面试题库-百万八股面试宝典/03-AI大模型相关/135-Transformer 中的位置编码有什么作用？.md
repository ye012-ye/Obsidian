在 Transformer 模型中，自注意力机制本身并不考虑 token 的顺序，这意味着模型无法区分“猫吃鱼”与“鱼吃猫”。**位置编码 (Positional Encoding)** 的引入正是为了解决这一问题：它将位置信息注入每个 token 的表示中，使模型具备顺序意识，从而理解语序依赖关系。M

绝对位置编码通过为每个位置生成一个唯一的编码向量（通常采用正弦余弦函数或可学习参数），并将其与 token embedding 相加。这种方式实现简单，能稳定提供绝对位置信息，并且在训练长度范围内表现良好。但缺点在于，其外推能力受限于预设最大长度，当遇到超长输入时效果明显下降。

相对位置编码则关注 token 间的距离关系而非具体位置。例如 Shaw 等人提出的加入相对偏置项的 attention 机制，或 RoPE（Rotary Positional Embedding）通过旋转 query 和 key 向量，使注意力权重只依赖相对距离。这类方法不仅能处理超出训练长度的序列，还能捕捉距离衰减和结构关系，适用于长文本生成和推理任务。S

此外，使用 RoPE 的模型可以自然演化出类似“多分辨率处理”的特性，akin 于小波变换，在不同层次上自动捕捉文本结构信息，这种 emergent wavelet-like 行为进一步提升了长上下文的表达能力。

### 示例面试回答：

Transformer 的注意力结构本身不考虑序列顺序，因此位置编码负责注入位置信息。经典方式是将正弦／余弦生成的绝对编码添加到词嵌入中，这在短文本任务中效果稳定且计算开销小。但这一方式在处理超出训练长度的文本时表现欠佳。

为此，相对位置编码（尤其是 RoPE）被引入：它以旋转矩阵形式编码 query 和 key，使得 attention 点积仅与 token 的相对距离有关，因而不受绝对位置限制。这种设计不仅提高了长序列的泛化能力，还自然演化出类似波形小波处理的特性，使模型更适应长上下文及跨长度推理场景。B

因此，在短文本固定任务中使用绝对编码足矣；而在需要处理长上下文、生成式或跨长度推理时，更推荐采用 RoPE 或其他相对编码方法。

RAG（检索增强生成）是一种结合外部知识检索与语言模型生成的技术架构，能显著提升大模型回答的准确性和时效性。其流程可分为以下几个关键阶段：

M

### 一、离线索引（Indexing）

首先，对待检索的数据源进行预处理：加载原始文档后按一定粒度拆分为文本 Chunk；然后使用嵌入模型（如 Sentence‑BERT、OpenAI Embedding）将每个 chunk 转换为向量，并存入向量数据库（如 FAISS、Milvus、Pinecone），完成索引创建供应检索使用。

### 二、用户查询处理

接收到用户查询后，能进行意图解析、关键词与实体识别，必要时还可扩展查询（如同义词、补充上下文）。然后使用相同的嵌入模型将查询向量化，准备进行检索。

### 三、检索阶段

以查询向量为输入，在向量库中搜索最相关的 chunk（如 top‑k 检索）。可以结合稀疏检索（BM25）和密集检索（向量搜索）进行混合召回。结果通常会排序并筛选去重。

S

### 四、增强与整合（Augmentation）

将检索得到的相关内容与原始查询拼接为增强 Prompt 输入给生成模型。此阶段可能包括对多个文本片段加权、重新排序、截断以满足长度限制，以及加入上下文元信息如来源或时间戳。

### 五、生成输出（Generation）

增强后的 Prompt 输入到 LLM（如 GPT‑4、Llama、Claude），由语言模型综合使用外部知识与其拟文能力生成回答。可以通过 prompt 工程技巧控制输出风格、增加事实性声明或控制温度等参数。

### 六、后处理（Post‑processing）

生成内容后可执行文本润色、去冗余、语法校正等，进行事实核验（如引用验证、NLI 比对）和格式统一处理。最后可能附上检索来源，用以增强可信度。

B

### ​流程脉络概览：

|  |  |
| --- | --- |
| **阶段** | **内容描述** |
| 索引构建 | 文档拆分 → 嵌入向量 → 存入向量数据库 |
| 查询处理 | 意图与关键词分析 → 查询向量化 |
| 检索与排序 | 向量搜索 + 排序过滤 |
| 增强提示（Prompt） | 拼接查询和相关文档信息 |
| 语言模型生成 | LLM 综合上下文生成最终答案 |
| 后处理与验证 | 文本优化、准确性核查与引用打标 |

### 技术挑战与优化方向

- 检索精度和召回率平衡：混合方法（稀疏+密集）能提升召回性能，并减少遗漏关键内容。
- 检索与生成协同优化：联合训练检索器与生成器（如 Meta 的原始 RAG），提升端到端效果。
- 实时性与效率优化：使用近似最近邻搜索、限制检索量或启用缓存机制以减少响应延迟。

### 示例场景

用户提问：“如何解决深度学习过拟合？” 系统流程如下：

1. 抽取查询关键词“深度学习”“过拟合”并向量化。
2. 检索知识库中相关文档（如“正则化方法”“Dropout 技术”“数据增强”）。
3. 将检索文本拼接到 Prompt 中，引导生成模型输出结构化答案。
4. 最终结果包括多种方法建议，并标注参考来源。

### 推荐答案示例

“RAG 是一种将外部知识动态注入大语言模型的架构。其流程包括：离线索引阶段将文档嵌入并存入向量数据库；运行时用户查询经过解析并向量化；通过检索召回相关片段并排序；将检索结果与查询拼接形成 Prompt 输入给 LLM；模型生成回答后执行事实验证与格式处理。该结构显著降低 LLM 幻觉风险，提升回答专业性和最新性，同时无需重新训练模型，适用于企业知识问答和智能客服等场景。”

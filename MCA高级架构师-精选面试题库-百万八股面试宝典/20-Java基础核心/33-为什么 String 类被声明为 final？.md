在 Java 中，`String` 类使用 `final` 修饰，意味着它不能被继承。这一设计对性能、安全性和正确性具有多重保障：

### 1. 保持不可变性

`String` 是不可变对象，一旦创建，其内部字符内容不能改变。将其声明为 `final` 可防止子类覆写行为或通过内部机制修改字符串，从而保障内容的一致性。这对于字符串实例间共享、缓存和并发访问至关重要。M

### ​2. 支持字符串常量池

字符串池机制依赖于字面量共享相同实例。若 `String` 可以被继承且子类可以修改其行为，对象池共享将失去意义。使用 `final` 保证池中实例的行为稳定，提升内存效率。

### ​3. 安全性和类加载保障

Java 常用 `String` 表示关键路径，例如数据库连接字符串、类名等。若字符串可变或行为可被子类篡改，将带来严重安全隐患。而 `final` 保证字符串标准行为不被篡改，从根本上维护可靠性和安全性。S

### ​4. 可用于高效缓存与多线程操作

不可变且 `final` 的 `String` 可以作为 `HashMap` 的键，缓存其 `hashCode`，提高运行效率。同时，因其状态不可变，也天然支持多线程共享，无需外部同步。B

Java 的 `HashMap` 初始化时默认容量为 **16（即 2^4）**，并且容量始终为 **2 的幂次方**。这种设计基于以下几个关键原因：

#### 1. 高效的索引计算

`HashMap` 使用位运算 `index = hash & (table.length - 1)` 来确定 key 所在桶的位置。如果 `table.length` 是 2 的幂次方，则 `length - 1` 的二进制全为 1，配合 `&` 运算即可等价于 `hash % length`，但性能更优，省去了昂贵的取模操作。M

#### 2. 均匀分布，降低冲突概率

当容量是 2 的幂次时，`length - 1` 能确保下层的位参与到索引计算中，使得哈希值的低位也能混合输入桶索引。若容量非幂，可能导致卫位无法参与，增加冲突概率 。S

#### 3. 扩容与迁移优化

每次扩容 `HashMap` 将容量翻倍（仍保持幂次）。由于高效的位映射机制，迁移节点时只需判断 `e.hash & oldCap == 0` 来决定节点是否“落桶”或“偏移 oldCap”，无需重新计算整个 hash，避免重复 hash 操作，提高性能。B

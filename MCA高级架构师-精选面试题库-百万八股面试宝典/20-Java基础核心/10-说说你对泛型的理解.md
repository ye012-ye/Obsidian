Java 泛型（Generics）是一种在编译时期进行参数化类型检查的机制，使得类、接口或方法可以适应不同的数据类型，确保类型安全并减少运行时类型错误。

## **核心理解：**

泛型通过使用类型参数（如 `<T>`、`<E>`）让开发者编写可重用的组件，比如 `List<String>` 和 `List<Integer>` 它们本质为同一类但行为安全([turn0search0])。泛型避免了运行时 `ClassCastException`，并删除了对类型转换的需求。

Java 泛型在运行时通过**类型擦除（type erasure）**实现，即编译后泛型信息被移除，所有泛型类型均转换为原始类型，例如 `Object` 或带边界的类型。这样既保证兼容性，又不影响性能，但也导致运行时无法获取泛型实际类型。

泛型还支持**通配符**，如 `?`、`? extends T`、`? super T`，用于表达协变与逆变关系，满足灵活性与类型安全的平衡，例如使用 `List<? extends Number>` 可获取 Number 或其子类型元素。

## **简洁示例：**

```java
// 泛型类
class Box<T> {
    private T value;
    public void set(T v) { value = v; }
    public T get() { return value; }
}

// 泛型方法
public static <U> void copy(U[] src, List<U> dest) {
    for (U item : src) dest.add(item);
}
```

使用 `Box<String>` 时，无需类型转换，编译器会自动推断类型。

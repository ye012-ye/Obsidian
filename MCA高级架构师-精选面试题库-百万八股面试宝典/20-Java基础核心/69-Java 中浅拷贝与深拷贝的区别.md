在 Java 中，对象复制主要有两种方式：浅拷贝和深拷贝，它们在对引用类型字段处理上有本质差异。

**浅拷贝** 即复制对象自身的字段，但不复制其引用类型字段所指向的对象。换言之，新旧对象共享相同的子对象。如果你通过 `Object.clone()` 获得一个对象副本，但其内部字段若是引用类型，则拷贝的只是那份引用。对该字段操作，会影响原对象—这是因为它们指向同一个对象实例。M

**深拷贝** 则是递归地复制整个对象图，包括所有引用字段所指向的对象。这样得到的新对象在逻辑和内存上都与原对象独立。对原对象修改不会影响副本，反之亦然。

### 详细解读

- **复制层级**  
  浅拷贝仅复制一层结构（父对象），引用类型字段仍指向原对象。深拷贝会递归创建所有被引用对象的拷贝直到最内层。S
- **内存开销与性能**  
  浅拷贝复制速度快，占用内存少；深拷贝需要额外创建多级对象，性能开销较大。
- **实现方式**  
  浅拷贝可直接使用 `super.clone()` 或拷贝构造器实现。深拷贝可以通过手动重写 clone 方法，对每个引用字段调用 clone，再赋值；也可使用序列化/反序列化，或借助如 Apache Commons Lang 的 `SerializationUtils.clone()` 方法。B

### 总结建议

当对象仅包含基本类型或不可变类型时，浅拷贝通常足够。而当对象引用了可变子对象，并希望复制后的实例完全独立，应优先采用深拷贝策略。确保 clone() 方法实现与设计意图一致，避免共享状态带来的隐患。

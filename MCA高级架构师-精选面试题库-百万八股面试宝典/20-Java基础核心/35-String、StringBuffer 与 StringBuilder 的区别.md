1. **不可变 vs 可变**

- `String` 是不可变类，每次修改都会创建新对象，适用于少量变动且需安全共享的场景。
- `StringBuffer` 和 `StringBuilder` 都是可变类，通过内部字符数组变更内容，无需不断创建新对象，因此更高效。

2. **线程安全性**

- `StringBuffer` 的方法使用 `synchronized` 进行同步，线程安全，但性能较低。
- `StringBuilder` 是非同步的，非线程安全，但在单线程场景下性能最佳。

3. **性能表现**

- `StringBuilder` 性能最优，`StringBuffer` 次之，`String` 最差，尤其在大量拼接场景中差异明显。

​

总结：

|  |  |  |
| --- | --- | --- |
| **类型** | **线程安全** | **使用场景** |
| **String** | 安全 | 值不会改变，或需要共享常量、不频繁拼接时 |
| **StringBuffer** | 安全 | 多线程环境下需要安全地构建字符串时 |
| **StringBuilder** | 不安全 | 单线程环境下频繁拼接字符串时 |

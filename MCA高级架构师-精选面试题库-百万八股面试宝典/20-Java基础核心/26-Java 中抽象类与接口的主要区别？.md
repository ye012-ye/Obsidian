在 Java 的面向对象设计中，抽象类和接口都用于定义通用行为，但在结构和用途上存在关键差异：M

#### 1. **继承关系与粒度**

- 抽象类（abstract class）用于共享相近类之间的实现逻辑和状态，只能单继承，适用于“是一个”（is-a）关系。
- 接口（interface）定义行为契约，可以多实现，适合描述“能做什么”（can-do）角色，不具备状态，体现松耦合设计。

#### 2. **方法与字段支持**

- 抽象类可以包含抽象方法、具体实现、成员变量、构造函数，也可包含初始化逻辑。
- 接口方法默认是抽象的，Java 8+ 可定义 `default` 和 `static` 方法，但不能维护状态，变量隐式为 `public static final`。

#### 3. **设计意图与扩展性**

- 抽象类更关注代码复用与版本演进，可在子类继承链中加入模板模式等设计。
- 接口用于定义应用模块间的协议与角色契约，为实现多态与解耦提供灵活支持。

懒汉模式是在首次调用 `getInstance()` 时才会创建实例，这样可以延迟资源消耗，但在多线程环境中如果不加锁可能会创建多个实例。  
而饿汉模式则在类加载时即初始化实例，由 JVM 类加载机制保证线程安全；这种方式简单高效，但会提前占用资源，即使不被使用，也已创建。

M

懒汉模式在单线程或对启动资源有限制的场景适用，但如果用于多线程需要配合 `synchronized`、双重检查锁（DCL）或静态内部类等方式确保线程安全，例如：

```java
// 双重检查锁
public static Singleton getInstance() {
    if (instance == null) {
        synchronized (Singleton.class) {
            if (instance == null) {
                instance = new Singleton();
            }
        }
    }
    return instance;
}
```

静态内部类方式更简洁，且线程安全与延迟加载兼顾 。

S

饿汉模式代码更简洁，使用如下方式：

```java
private static final Singleton INSTANCE = new Singleton();
public static Singleton getInstance() { return INSTANCE; }
```

适用于实例创建成本低且肯定会用到的场景，避免懒加载的同步复杂度 。

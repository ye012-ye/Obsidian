在大多数情况下，这两种遍历方式差距微乎其微，但在特定场景下性能表现有所不同。M

## **数组遍历**

对于数组，Java 编译器会将 `for-each` 循环转换为类似于常规 `for` 的索引访问，因此两者在运行时几乎没有性能差异。S

## **集合遍历**

- 对于 `ArrayList`（支持随机访问）：

- 常规 `for (int i=0; i<list.size(); i++)` 直接通过索引取值，通常略优于使用 `Iterator` 的 `for-each`，尤其在 Android 和老旧 JVM 上，这种差异可能在某些基准中看到约 3 倍速度提升。
- 但现代 JIT 优化后，两者差异已变得可忽略。

- 对于 `LinkedList`（不支持随机访问）：

- 使用带索引的常规 `for` 每次都要从头遍历链表，造成 **O(n²)** 时间复杂度。
- 而 `for-each` 使用一次 `Iterator`，每次访问下一个节点，仅为 **O(n)**，效率更高。B

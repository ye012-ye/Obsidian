这两种模式都是创建型设计，但它们关注的粒度不同：M

#### 1. 创建对象的数量与关联程度

- **工厂方法（Factory Method）**：专注于创建单个产品。它通过抽象方法或接口，由子类实现具体对象实例化。
- **抽象工厂（Abstract Factory）**：用于创建一组相关或相互依赖的对象（产品族），如同时得到多个产品接口的实例组合。

#### 2. 实现机制

- **工厂方法**：依赖**继承**，子类实现工厂方法来生产产品。S
- **抽象工厂**：使用**组合**方式，客户端持有引用，调用具体工厂创建多个产品。

#### 3. 抽象层次

- 工厂方法为**单产品提供单一入口**，用于动态创建不同类型实例。
- 抽象工厂则是**工厂的工厂**，把多个工厂封装成一组，通过接口统一处理整个产品族。B

### 对比

|  |  |  |
| --- | --- | --- |
| 模式 | 适用场景 | 示例 |
| 工厂方法 | 仅需基于条件创建一个产品类 | 接收 `VehicleFactory` 参数，根据配置返回 `Car` 或 `Bike` 实例。 |
| 抽象工厂 | 需要同时生成同体系的多个产品实例 | GUI 程序，根据操作系统选择 `WinFactory` 或 `MacFactory`，统一创建按钮、菜单等组件。 |

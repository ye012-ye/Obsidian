Java 中重载和重写都是多态的体现，但作用、时机、使用条件各不相同：M

#### 1. **定义 & 绑定时机**

- **重载（Overloading）**：在同一个类中，方法名相同但参数列表不同（数量、类型或顺序）。发生在编译阶段，是**编译期多态 / 静态绑定**，由编译器根据参数确定调用哪个方法。S
- **重写（Overriding）**：子类中对继承自父类的方法提供相同签名、相同（或协变）返回类型的新实现。它基于运行时对象的实际类型进行**运行时多态 / 动态绑定** 。

#### 2. **应用条件**

|  |  |  |
| --- | --- | --- |
| **条件** | **重载** | **重写** |
| 类关系 | 同一个类中 | 需有继承关系（父子类） |
| 参数签名 | 必须不同 | 必须完全相同（签名一致） |
| 返回类型 | 可相同可不同 | 必须相同或协变 |
| 访问权限 | 不影响 | 不可降低访问级别（子类方法必须 ≥ 父类） |
| 异常抛出 | 无特殊限制 | 只能抛父类方法声明的异常或其子类 |

#### 3. **典型使用场景**

- **重载**：用于提供多个功能变体，如工具类中 `print(int)`, `print(String)` 等，增强接口易用性。
- **重写**：实现子类特定行为，例如 `Animal` 定义 `eat()` 方法，`Dog` 重写以输出“Dog is eating”。实现面向接口、设计模式和代码扩展的关键机制。B

Java 在装箱 `Integer` 时，会对某些常用值进行缓存，提高性能并节省内存。默认缓存范围是 **–128 到 127**。当你执行 `Integer.valueOf(100)` 或直接写 `Integer x = 100;` 时，编译器会调用 `valueOf()`，这时如果值在缓存范围内，将直接返回缓存中已存在的对象，而不是新建一个实例。M

这种机制带来两个主要影响：S

- **性能提升和内存优化**：避免频繁创建相同值的 `Integer` 对象，特别是在循环或常数使用场景中，能减少垃圾回收压力 。
- **注意** `==` **比较问题**：缓存范围内，两个装箱值相等的 `Integer` 对象引用相同，`==` 返回 `true`；范围外则不会，可能返回 `false`。因此应统一使用 `equals()` 比较值。

开发者可以通过 JVM 参数 `-XX:AutoBoxCacheMax=<size>` 扩大缓存的最大值，但合理限制是为了避免缓存过多对象，导致内存压力增加。B

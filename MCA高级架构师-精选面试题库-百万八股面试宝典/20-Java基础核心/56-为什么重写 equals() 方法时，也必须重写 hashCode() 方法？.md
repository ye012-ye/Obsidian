根据 Java 规范，如果两个对象通过 `equals()` 被认为相等，那么它们的 `hashCode()` 也必须返回相同值；这是 `hashCode()`**与**`equals()` 之间的契约关系。M

#### 为什么需要这样？

在基于哈希的数据结构（如 `HashMap`、`HashSet`）中，查找元素分两步进行：S

1. **通过** `hashCode()` **定位桶**；
2. 在该桶内部，通过 `equals()` 判断是否存在相等对象。

如果你重写了 `equals()`，但忘记重写 `hashCode()`，那么即使两个对象内容相同，`hashCode()` 仍由 `Object` 默认实现（通常与对象地址相关），返回不同结果，会被分到不同的桶。这样，即便 `equals()` 返回 `true`，集合也可能找不到该对象，导致如 `get()` 返回 `null` 或 `add()` 插入重复数据的情况。

反之，仅重写 `hashCode()` 而不修改 `equals()`，可能会让不相等的对象拥有相同 hash 值，虽然查找定位到了相应桶，但最终因为 `equals()` 判断不同，集合仍可正常工作。但这种情况会降低性能，且不是完整修复方式 。B

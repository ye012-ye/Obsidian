避免 NPE 的思路可以从以下几个角度展开：

### 1. 使用 Optional 链式处理

采用 `Optional.ofNullable(...)` + `map()` 拆链方式，减少层层嵌套判断，同时保持函数式风格：

```java
public String getUserCity(User user) {
    return Optional.ofNullable(user)
    .map(User::getAddress)
    .map(Address::getCity)
    .map(String::trim)
    .orElse("Unknown");
}
```

此方法直观清晰，避免复杂嵌套，同时可设置默认值。M

### 2. 先置默认对象，防患于未然

在构造时直接用非 null 默认值替代 null，能让后续调用更稳健：

```java
public User() {
this.address = new Address("", "", "");
}
```

这样一来，`user.getAddress().getCity().trim()` 不会抛空异常。

### 3. 用工具类简化空检

借助 Apache Commons / Spring Utils：

```java
if (StringUtils.isNotBlank(user.getAddress().getCity())) {
    // 安全使用
}
```

也可在集合操作中用 `CollectionUtils.isEmpty(...)` 检查。S

### 4. 关键点前置校验（Fail-Fast）

对关键参数及状态进行断言或抛错，保证方法内逻辑调用可用：

```java
public void setUserAddress(User user) {
    Assert.notNull(user, "user cannot be null");
    Assert.notNull(user.getAddress(), "address cannot be null");
    // 然后安全调用
}
```

### 5. 开启 JDK17+ 的“Helpful NPE”

在 JDK 14+ 可通过 `-XX:+ShowCodeDetailsInExceptionMessages` 启用，出错时 JVM 会指出哪一步 null，例如：  
“Cannot invoke “String.trim()” because the return value of getCity() is null”B

### 6. 字符串比较等场景建议用字面量 equals

避免空调用：

```java
if ("OK".equals(response.getStatus())) { … }
```

字面量不会为 null，可避免 NPE。

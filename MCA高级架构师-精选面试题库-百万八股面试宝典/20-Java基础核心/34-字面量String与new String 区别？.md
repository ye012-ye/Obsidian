Java 中两种创建字符串的方式实质上都生成 `String` 对象，但底层机制和性能差异明显：M

### 1. **字面量创建**

- JVM 会将 `"literal"` 放入 **字符串常量池（String Pool）** 中。
- 若池中已存在相同内容的字符串，则变量指向该已存在实例；否则就存入池并引用。
- 因此，`String a = "hello"; String b = "hello"; a == b` 为 `true`，内存利用高效。
- 常量池中的字符串可能随着 JVM 生命周期持续存在（但 Java 7+ 会 GC 非活跃常量池项）。

### 2. **new String(...) 创建**

- 每次执行都会在堆上生成一个新的 `String` 对象，，与常量池的内容无关。S
- 即使内容一致，不同调用也产生不同实例，`new String("x") == new String("x")` 返回 `false`。
- 通常性能更差，内存消耗更大，因为总是分配新对象。

### 3. **intern() 方法**

- `String.intern()` 可以将堆中的字符串加入常量池，返回池中的实例。
- 可用来确保引用池中的唯一对象，例如：

```java
String s1 = new String("A").intern();
String s2 = "A";
// s1 == s2 是 true
```

### 比较

|  |  |  |  |
| --- | --- | --- | --- |
| **方式** | **内存使用** | **相等比较 (==)** | **推荐场景** |
| 字面量 `"..."` | 池中共享，最节省 | 相等 | 默认使用字符串常量 |
| `new String(...)` | 堆上每次新建，资源浪费 | 不相等 | 很少需要，避免使用 |
| `intern()` | 初次放池，后续共享 | 相等（与池中） | 特定场景：大量重复字符串优化时 |

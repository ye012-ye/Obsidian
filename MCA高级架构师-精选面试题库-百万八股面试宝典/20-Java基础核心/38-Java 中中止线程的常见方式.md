Java 中中止线程主要有三种方式，不同方法各有特点：

### 1. 使用退出标志（flag）配合 `volatile`

**原理**：线程运行时循环检查一个 `volatile boolean` 标志，外部将该值置为 `true`，线程退出循环并完成 `run()` 方法终止。  
**优点**：协作式结束，线程可自行清理资源，避免突然停止导致的不一致情况 。  
**缺点**：循环体中必须定期检查标志，如果循环长或阻塞，响应会有延迟。M

### 2. 使用 `Thread.interrupt()` 中断

**原理**：给线程发送中断信号，设置中断标记。线程在阻塞（如 `sleep()`、`wait()`、某些 I/O）会抛出 `InterruptedException`；否则可显式检查 `isInterrupted()` 并安全停止 。  
**优点**：适合处理阻塞状态，配合标志一起使用能更友好地终止线程。S  
**注意点**：非阻塞操作不自动响应中断，需要手动检查；捕获 `InterruptedException` 后应调用 `Thread.currentThread().interrupt()` 恢复状态 。

### 3. 使用 `Thread.stop(`（不推荐）

**原理**：立即抛出 `ThreadDeath` 异常，强制终止线程 。  
**缺点**：

- 释放所有锁，可能导致对象处于不一致状态；B
- 无法保证资源清理，如锁、I/O 等；
- 已被弃用，不应在新代码中使用。

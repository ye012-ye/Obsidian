Java 中的 `char` 是 16 位的基本类型，用于表示一个 UTF-16 编码单元。

**对于大多数常用汉字（在 Unicode 基本多语言平面 BMP 内的字符）**，`char` 类型可以存储。因为这些字符的 Unicode 码点小于 U+FFFF，只需一个 16 位的 `char` 即可。如 `char c = '\u4E2D';` 可以成功表示 “中” 字。

但如果是 **超出 BMP 范围的字符**（如某些扩展汉字或 Emoji），它们的码点大于 U+FFFF，则在 UTF-16 中要用两个 `char`来表示，单个 `char` 不够存储完整字符。

备注：

BMP，全称 **Basic Multilingual Plane**，是 Unicode 码位体系中的第一个平面（Plane 0），涵盖了从 **U+0000 到 U+FFFF** 共 65,536 个码点。

这个平面包含了几乎所有常用书写系统的字符，包括拉丁语、希腊语、西里尔文、汉字、日语、韩语等基本字符，以及通用标点符号和少量图形符号。

Java 的 `HashMap` 采用了 **数组 + 链表/红黑树** 的组合结构，具体如下：

首先，`HashMap` 维护一个 Node 数组（也称桶或 bin）。每次 `put()` 操作时，通过 key 的 `hashCode()` 再经扰动计算得到数组索引，将 key-value 存入对应的桶中。

在 **JDK 1.7 及更早版本**，每个桶内部采用**链表**结构连接所有发生 hash 冲突的元素。如果多个 key 映射到同一个桶，就在链表末端添加新节点。

从 **JDK 1.8 开始**，为了提升性能，当链表长度超过 **8** 且当前总容量超过 **64** 时，该桶将被树化为**红黑树**，从原先的 O(n) 查找性能提升至 O(log n)。

在 Java 中，**完全可能存在两个不相等的对象拥有相同的** `hashCode()`，这被称为**哈希冲突**。这是由于 `hashCode()` 返回的是一个 32 位的 `int`，其值域远小于可能的对象数量，符合鸽巢原理(即使两个不同对象映射到相同整数是不可避免的)。M

在实践中，仅需几万至十几万个 `Object` 实例，就有较高概率出现冲突。S

#### 为什么冲突并不异常

哈希码设计的目标是加快对对象的初步分组。对于真正比较两个对象是否相等，Java 会在检测到相同 `hashCode()` 后，再调用 `equals()` 方法。若两个对象 `equals()` 返回 `false`，即使 `hashCode()` 相同，也会被视为不同对象。在 `HashMap`、`HashSet` 中，这种冲突通过链表或红黑树等结构进行处理，保证正确性和最终一致性。B

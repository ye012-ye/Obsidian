在 Java 中，**静态嵌套类**和**非静态内部类**都是定义于另一个类中的类，但它们在实例关联、访问权限和使用场景上存在显著区别。

**1. 实例化方式与作用域**  
静态嵌套类属于外部类本身，不依赖外部类的实例，可以通过 `Outer.StaticNested nested = new Outer.StaticNested()` 直接实例化。它与外部类实例的生命周期相独立。相比之下，非静态内部类（又称 inner class）必须依附于外部类实例，通过 `Outer.Inner inner = outer.new Inner()` 创建，存在隐式引用指向外部对象。M

**2. 对成员的访问**  
静态嵌套类无法访问外部类的实例成员，只能操作静态字段与方法，类似顶级类但在语法上归属于外部类作用域。非静态内部类则可以自由访问外部类的所有成员，包括私有成员 。S

**3. 内存引用与资源占用**  
静态嵌套类没有隐式的外部类引用，占用更少内存且更易被垃圾回收。非静态内部类会持有一个外部类引用，如果不谨慎处理，可能导致外部对象无法释放。B

**4. 典型使用场景**

- 当你的嵌套类只是逻辑上属于外部类、但不需要访问其实例成员时，推荐使用静态嵌套类。例如实现辅助工具类、表达式构建器等用途。
- 若嵌套类需要操作外部类的非静态字段或方法，比如迭代集合、处理外部对象状态，则应使用非静态内部类，这样可以直接引用外部实例 。

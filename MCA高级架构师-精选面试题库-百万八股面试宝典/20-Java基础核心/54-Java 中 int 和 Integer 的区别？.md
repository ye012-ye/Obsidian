在 Java 中，`int` 是**基本数据类型**，而 `Integer` 是其对应的**包装类对象**，二者在多个层面存在区别。M

​

`int` 直接在内存里存储数值，本身不是对象，无方法可调用。它占据 **4 字节**，访问速度快，适合高性能场景（如大量数学计算或循环）。而 `Integer` 是 **引用类型**，存在于堆内存中，包含对象元数据及缓存机制，占用空间更大，同时访问时涉及装箱/拆箱开销。S

​

此外，`Integer` 提供多种便捷方法，如 `parseInt()`、进制转换 (`toBinaryString()`)、位操作 (`rotateLeft()`, `reverse()` 等)，这些是 `int` 所不具备的。包装类还可为 `null`，在处理可选值或数据库字段时更具灵活性；而 `int` 不可为 null，只能取默认值 0。

​

对于集合类型（如 `List<Integer>`），由于 Java 泛型只支持引用类型，也必须使用 `Integer`。在其他场景下，优先使用原始类型 `int`，以避免性能下降与内存负担。B

Java 中不允许一个类同时继承多个父类，这是为了保持语言设计的简洁与安全，同时规避多继承带来的模糊和复杂问题。M

1. **避免二义性及钻石问题**  
   如果多个父类定义了同名方法（可能实现不同逻辑），Java 无法判断到底应继承哪一个，导致行为不确定。多继承会引发这种编译时或运行时的不明确情况。
2. **降低语言复杂度**  
   为防止因类层级过深或关系复杂而带来的解析、维护难题，Java 设计者选择了单继承，以确保语言更加稳定、安全。
3. **接口实现的替代方案**  
   Java 提供接口机制：一个类可以实现多个接口，从而获得多种行为契约。接口直到 Java 8 才允许定义默认方法（default），但其依然是由实现类来选择重写或调用，避免了多继承的二义性。

- 实现类必须显式覆盖冲突的 default 方法，否则编译报错，确保执行逻辑清晰。S

4. **推荐组合优于继承**  
   通过“Has‑a”的方式（组合）重用功能比“Is‑a”的继承更灵活，结构松耦合，易于测试与替换。

### 示例对比

```java
interface A {
    default void foo() { System.out.println("A.foo"); }
}
interface B {
    default void foo() { System.out.println("B.foo"); }
}
class C implements A, B {
    @Override
    public void foo() {
        // 必须显示解决冲突
        A.super.foo();
    }
}
```

通过上面设计，`C` 显式指定使用 `A` 的 `foo`，而非语言自动选择，实现清晰而安全的多继承效果。B

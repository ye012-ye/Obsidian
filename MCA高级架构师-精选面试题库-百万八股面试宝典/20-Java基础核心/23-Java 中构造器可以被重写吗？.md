在 Java 中，**构造器（Constructor）不能被重写（override）**，因为它们并非普通方法，仅用于初始化新对象。以下几点阐明其设计与用法：M

1. **构造器不是继承来的方法**

- Java 中重写要求子类继承父类方法并重新定义签名相同的方法，但构造器不参与继承，因此无法实现重写。
- 构造器只是用于创建当前类实例，子类拥有自己的构造器，与父类构造器名称一致但并非覆盖关系。

2. **构造器重载是允许的**

- 同一个类内部可以定义多个构造器，只要它们参数列表不同，即构造器重载。例如提供无参和含参构造器不同初始化策略 。S
- 这是“同名不同参”机制，而非子类重写。

3. **子类可以复用父类构造逻辑**

- 在子类构造器中使用 `super(...)` 显式调用父类对应构造器，以确保父类成员被正确初始化。
- 如果子类未定义任何构造器，编译器会生成默认构造器并自动调用 `super()`，否则开发者需要自行调用相应父构造器。B

在缓存淘汰策略中，LRU 和 LFU 是两种常见的算法，它们在设计目标、实现复杂度和适用场景上存在显著差异：

M

#### 1. **淘汰原则**

- **LRU（Least Recently Used）** 移除最近最久未使用的数据，适用于临时热点集中、访问时间相对较新的场景。
- **LFU（Least Frequently Used）** 删除访问频率最低的数据，对长期热点具有更强保留能力，但对突发热点反应较慢。

S

#### 2. **实现复杂度**

- **LRU** 通过哈希表＋双向链表可实现 O(1) 的增删操作，代码简洁，性能稳定。
- **LFU** 需维护频率计数、多个链表及最小频率指针，逻辑复杂，但可通过 Hash＋频率链表结构实现 O(1) 平均操作。

B

#### 3. **适用场景对比**

|  |  |  |
| --- | --- | --- |
| **特性** | **LRU** | **LFU** |
| 响应热点变化 | 快速适应突发热点 | 适合长期热度数据 |
| 实现难度 | 简单，适用于大部分场景 | 复杂，需要特别性能优化 |
| 内存使用 | 低，结构简单 | 高，维护复杂结构 |

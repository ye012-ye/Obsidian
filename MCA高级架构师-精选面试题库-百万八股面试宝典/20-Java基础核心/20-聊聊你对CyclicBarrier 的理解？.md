`CyclicBarrier` 是 Java 并发包中的同步工具，允许一组固定数量线程在每个阶段（phase）执行到一半时**等待彼此，然后统一继续执行**。它内置“屏障点”，并支持**重用**与“屏障动作”。

以下按主题逐段说明：

M

#### 阶段协同机制

- 创建时指定参与线程数（`parties`），例如 `new CyclicBarrier(3)`；每个线程在完成当前阶段工作后调用 `barrier.await()`，会在此阻塞直到所有线程都调用了`await()`。
- 当最后一个线程到达时，屏障触发并自动重置，允许线程进入下一阶段。参与线程可按循环多次重用该屏障。

S

#### 可选屏障动作

- 可在构造时传入 `Runnable barrierAction`，该动作在最后一个线程到达并且所有线程放行前执行。通常用于阶段合并、统计或日志。
- 屏障动作仅执行一次，由最后一个到达的线程触发。

B

#### 异常与超时处理

- `await()` 方法可能抛出 `InterruptedException` 或 `BrokenBarrierException`。
- 也支持带超时的 `await(long, TimeUnit)`，超时或中断会**破坏屏障**（broken state），所有等待中的线程会收到异常。
- 在 catch 中应合理处理，比如重试下一阶段或退出。

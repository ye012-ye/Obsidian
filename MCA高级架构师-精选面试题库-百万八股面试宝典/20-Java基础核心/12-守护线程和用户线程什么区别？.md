- **守护线程（Daemon Thread）**：后台支持性线程，JVM 在只剩守护线程时会自动退出，不等待它们完成。
- **用户线程（User Thread）**：执行主业务逻辑，JVM 会等待所有用户线程结束后才终止。

### 核心区别总结

|  |  |  |
| --- | --- | --- |
| 特性 | 用户线程 | 守护线程 |
| JVM 等待 | 会等待结束 | 不会等待，一旦用户线程结束即终止 |
| 优先级 | 默认较高 | 默认较低，被视为后台任务 |
| 用途 | 执行核心业务，例如处理数据、请求等 | 执行背景支持任务，如垃圾回收、日志等 |
| 生命周期 | 可独立存活，手动或自然结束 | 生命周期依赖用户线程，父线程是守护则子线程自动继承 |

在 Java 里，变量和常量在定义、可变性和内存表现上有明显差异。

**1. 可变性**

- **变量**：声明后其值可以随时修改。
- **常量**：使用 `final` 修饰后，仅能赋值一次，值保持不变。S

**2. 声明与初始化**

- 变量可以先声明后赋值，也可声明时直接初始化；例：

```java
int x;
x = 5;
```

- 常量必须在声明时或在构造函数/静态块中初始化，标记为 `static final`（类级常量）或 `final`（实例常量）；例如：

```java
public static final double PI = 3.1415926;
```

**3. 内存与编译处理**

- 变量在运行时分配内存，值存储在该内存中，并以变量形式引用。
- 常量若是 `static final` 且为编译时常量，编译器会在编译期间直接替换调用处的字面值，从而不占类实例内存，并能提升运行效率。B

**4. 使用场景**

- 变量适合存储运行时会改变的数据，如循环计数、临时状态等。
- 常量适合表示固定值或配置，如数学常数、系统配置项等，确保值不可被篡改。

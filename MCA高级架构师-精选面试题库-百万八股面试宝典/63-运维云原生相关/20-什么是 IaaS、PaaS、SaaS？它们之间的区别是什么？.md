云计算有三种主要服务模型：IaaS、PaaS 和 SaaS。它们体现了用户与服务提供商在管理责任上的不同分工，也代表了不同的服务抽象层级。

M

### ​IaaS（Infrastructure as a Service，基础设施即服务）

IaaS 提供虚拟化的计算、存储、网络以及硬件资源，用户负责安装和管理操作系统、运行时、中间件和应用程序。供应商则管理底层的物理资源与虚拟化平台。

- **用户控制层级最高**：完全管理 OS、应用、数据等。
- **复杂性和灵活性并存**：适合需要高度控制的场景，如大数据、AI、可定制化应用。
- **代表厂商**：AWS EC2、Azure VMs、Google Compute Engine 等。

### ​PaaS（Platform as a Service，平台即服务）

PaaS 提供完整的开发与运行环境，包括操作系统、数据库、执行环境和 Web 服务器，用户只需专注于应用程序和数据的构建。底层基础设施和运行环境则由服务商负责管理与维护。

- **适合开发者使用**：能专注开发而无需管理服务器或中间件。
- **灵活但更受限于平台约束**：适合快速迭代和新应用部署，但定制性低于 IaaS。
- **代表厂商**：Heroku、Google App Engine、AWS Elastic Beanstalk、Azure App Service 等。

S

### ​SaaS（Software as a Service，软件即服务）

SaaS 是完整的软件应用服务，用户通过互联网访问，无需安装或管理底层任何基础设施。所有更新、维护、安全及其后端均由服务商负责。

- **对用户而言最简便**：提供即用即付的软件服务。
- **控制最少**：仅可进行软件配置，不能修改平台或基础设施。
- **典型示例**：Google Workspace、Microsoft 365、Salesforce、Dropbox、Slack 等。

### 模型对比

|  |  |  |  |
| --- | --- | --- | --- |
| **服务模型** | **用户负责内容** | **服务商负责内容** | **典型场景及优势** |
| **IaaS** | 操作系统、中间件、应用程序、数据 | 物理资源、虚拟化层 | 最大控制权、灵活性高，适合复杂应用 |
| **PaaS** | 应用程序与数据 | 操作系统、平台、数据库 | 简化运维流程，开发效率高 |
| **SaaS** | 软件配置与使用 | 完整运维、安全和升级 | 最轻量级，部署快速、适用于终端使用者 |

防火墙是一种关键的网络安全设备或软件系统，主要用于监控和控制网络流量，根据预设安全规则允许、拒绝或限制数据包通过网络边界，起到隔离“可信”和“不可信”网络、阻止未经授权访问与攻击的作用。它可以部署于网络边界、主机端或云环境中，包括基础的包过滤、防火墙、代理、防火墙以及下一代防火墙（NGFW）等多种形式。

### 常见配置策略

#### 1. 基于 IP 地址的策略（IP Filtering）

设定允许或拒绝特定源 IP 或目标 IP 的流量。例如，只允许内部网络 IP 段访问数据库服务器，屏蔽外部访问请求。此策略简单高效，适合初级隔离及基本访问控制。

#### 2. 基于端口与协议的过滤

结合端口号和协议类型进行控制，例如仅开放 TCP 80（HTTP）、443（HTTPS）服务端口，拒绝非必要服务及高风险协议；若需允许 ICMP，则可采用精细控制，只允许 ping 请求同时限制洪泛型 ICMP 类型。该策略有助于最小化开放面并防止利用协议漏洞攻击。

#### 3. 基于状态检测的过滤（Stateful Inspection / CBAC）

采用状态跟踪技术（如 CBAC 或传统 Stateful Firewall），不仅检查包头信息，还追踪会话状态，允许响应从内部发起的连接数据返回，同时封堵未授权的外部连接建立。这比静态包过滤安全性更高，但对性能有一定影响。

#### 4. 基于应用层与用户身份的过滤（NGFW 与 User‑ID）

下一代防火墙（NGFW）通过深度包检测（Deep Packet Inspection，DPI）识别应用或服务类型，并可结合用户身份（User‑ID）策略制定更细粒度访问规则，例如限制某用户组访问社交媒体服务，而非依赖 IP 地址区分。

#### 5. 基于时间段或网络区域的访问控制

允许管理员设定规则生效时间，比如在办公时段允许访问特定业务系统，非工作时间关闭部分服务；或根据区域划分不同防护策略，如 LAN、WAN、DMZ 区域间采用不同默认策略（LAN→WAN 通行，WAN→LAN 默认阻止）。

#### 6. 基于端口安全或 MAC 地址白名单

有些防火墙或交换设备支持 MAC 地址过滤或端口绑定，例如允许指定交换机端口只接入设定过的 MAC，并限制接入数量。此方式适合防止物理设备外部接入或 MAC 欺骗场景。

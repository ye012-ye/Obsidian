Zabbix 是一款企业级、开源的监控系统，适用于监控数万台设备和海量指标。其架构包含多个核心组件，协同工作以实现数据采集、存储、触发告警、可视化分析及系统管理。轴心思想是集中管理且可扩展，适合云原生与分布式环境。

M

### 架构组成与作用

- **Zabbix Server**：核心模块，负责数据采集、触发器运算、事件生成与告警通知，是整个监控系统的指挥中心。它定期执行轮询、触发 SNMP、IPMI、SSH 等方式采集数据，并分析指标是否触发预定义条件。
- **Zabbix Agent**：部署在被监控主机上的守护程序（daemon），以高性能 C 语言实现，可采集主机本地资源使用情况（如 CPU、内存、磁盘、进程等），并上报给 Zabbix Server。也提供 agent2 (Go 语言实现，可插件扩展)。
- **Zabbix Proxy**（可选）：用于分布式、多站点部署场景。Proxy 在远端或隔离网络中收集监控数据，缓存在本地并定期推送至主服务器，减轻主服务器压力，同时可在网络不通时缓存数据、实现离线监控。
- **数据库后端**：Zabbix 使用关系型数据库（如 MySQL、PostgreSQL、Oracle、SQLite、IBM DB2 等）存储配置、历史数据、趋势数据和告警记录。这是整个平台的数据支撑层。
- **Web 前端界面**：用于可视化展示监控数据、配置监控项与触发器、管理主机与用户权限。该界面通常部署于 PHP 运行环境，可与 Server 和数据库通信操作监控数据与配置。

S

### 工作流程原理

1. **数据采集**：Zabbix Server 主动轮询（polling）或接收 Agent、Proxy 推送的数据，也可通过 SNMP、ICMP、IPMI、JMX、SSH 等方式进行无代理监控。
2. **指标处理与触发器判断**：Server 定期从数据库缓存配置项，分析采集数据是否满足触发器的条件。当阈值被触发时，会生成事件和问题记录，并启动报警机制（如邮件、短信、脚本等）。
3. **数据存储**：采集结果首先存储在 history 表中，趋势数据则通过归档方式写入 trend 表，用以长期分析和容量规划。
4. **可视化与告警管理**：通过 Web 前端，用户可查看仪表盘、图表、主机状态、告警历史、网络拓扑地图等，并配置监控模板、触发器、用户权限及自动发现策略。

B

**总结：**

Zabbix 架构中Server 负责控制与告警，Agent 和 Proxy 负责数据收集，数据库存储持久数据，Web 前端负责监控管理与可视化。支持多协议采集、模板配置、触发器报警、自动发现、分布式 Proxy 模式与高可用部署。其组件由高性能 C/PHP/Go 实现，适合大规模云原生与企业环境监控。

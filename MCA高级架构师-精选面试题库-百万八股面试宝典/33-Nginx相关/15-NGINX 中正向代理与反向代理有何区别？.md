在 NGINX 中，正向代理（forward proxy）与反向代理（reverse proxy）虽然都扮演流量中转角色，但它们服务的对象和作用方向截然不同：M

### ​正向代理（面向客户端）

- **位置与对象**  
  位于客户端与互联网之间，客户端明确请求通过这个代理访问资源，服务多个客户端。服务器端只知道请求来自代理而非真实客户端 。
- **应用场景**

- 消除地理或内容访问限制
- 隐匿真实客户端 IP
- Client-side 缓存与过滤控制
- 常用于企业、校园等网络环境 。

S

### ​反向代理（面向服务端）

- **位置与对象**  
  处于互联网与后端服务器之间，所有到达服务器的请求都由代理接收和分发。客户端并不清楚后端服务的存在 。
- **用途**

- 进行请求分发与负载均衡
- 缓存静态资源或进行 SSL 终止
- 提升安全性（隐藏后端服务器）
- 可加入访问控制、请求压缩等功能 。

B

## 对比

|  |  |  |
| --- | --- | --- |
| **特性** | **正向代理** | **反向代理** |
| 用户 | 面向客户端用户 | 面向后端服务 |
| 客户端行为 | 配置代理后才能访问外部网络 | 无需配置，客户端直接访问代理 |
| 隐匿对象 | 隐匿客户端 IP | 隐匿后端服务器 IP |
| 典型场景 | 翻墙、网络过滤、隐私保护 | 负载均衡、缓存、SSL 终端、Web 加速 |

在 Go 语言中，单元测试是通过标准库 `testing` 包结合 `go test` 工具来实现的。这种方式简单高效，非常适合在持续集成中使用。

**1. 基本结构**

测试文件命名需以 `_test.go` 结尾，测试函数必须以 `Test` 开头并接收 `*testing.T` 参数：

```go
func TestAdd(t *testing.T) {
    got := Add(1, 2)
    want := 3
    if got != want {
        t.Errorf("Add(1,2)= %d; want %d", got, want)
    }
}
```

`go test` 会自动识别并执行这类函数；`t.Errorf` 用于报告失败。

**2. 表格驱动测试（Table‑Driven Tests）**

表格驱动是 Go 测试推荐的模式之一，通过定义多个 case 并循环测试，实现测试逻辑复用与清晰覆盖：

```go

func TestAddTable(t *testing.T) {
    cases := []struct { name string; a, b, want int }{
        {"1+2",1,2,3},
        {"-1+1", -1,1,0},
    }
    for _, c := range cases {
        t.Run(c.name, func(t *testing.T) {
            if got := Add(c.a, c.b); got != c.want {
                t.Errorf("got %d; want %d", got, c.want)
            }
        })
    }
}
```

这种方式让每个测试子项都清晰命名，便于定位问题 。

**3. 基准测试与示例测试**

- **基准测试**：以 `func BenchmarkXxx(b *testing.B)` 格式定义，执行 `b.N` 次操作，用于性能测量：

```go
func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Add(1, 2)
    }
}
```

- **示例测试**：以 `ExampleXxx` 命名，引擎会执行代码并验证输出注释，例如：

```go
func ExampleAdd() {
    fmt.Println(Add(2, 3))
    // Output: 5
}
```

这种方式既测试又生成文档 。

**4. Mock 与依赖注入**

Go 常通过接口（interface）实现依赖注入，将外部依赖（如数据库、HTTP 客户端）抽象为接口。在测试中可以替换为 mock 实现，以避免对真实外部系统的依赖。

```go
type DB interface {
    Get(key string) (string, error)
}
```

通过注入 mock，实现纯粹、可控的单元测试。

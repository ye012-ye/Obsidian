Go 并不使用传统类继承机制，而是通过**结构体（struct）和接口（interface）结合组合（composition）**来实现面向对象的设计理念。

**1. 结构体表示对象：**  
结构体用于定义数据属性，类似于其他语言中的类成员。它是值类型，但通常配合指针使用以便方法能修改其状态。M

```go
type Person struct {  
    Name string  
}

func (p *Person) Greet() string {  
    return "Hello, " + p.Name  
}
```

以上示例中，`Person` 结构体拥有属性 `Name`，并通过接收者方法实现行为。

**2. 接口描述行为契约，多态的核心：**  
接口定义一组方法签名，任何实现这些方法的类型都隐式满足该接口，无需显式声明。S

```go
type Greeter interface {
    Greet() string
}
```

任何具有 `Greet()` 方法的类型都可被视为 `Greeter`，实现了运行时多态。

**3. 结构体组合支持代码复用：**  
通过在结构体中嵌入其他结构体，实现字段与方法继承（更准确地说是复用）。Embedders 可以直接使用被嵌入类型的方法。B

```go
type Employee struct {
    Person  // 嵌入 Person
    Role    string
}

// Employee 自动拥有 Greet 方法
```

**4. 多态示例：**

```go
func SayHello(g Greeter) {
    fmt.Println(g.Greet())
}

func main() {
    bob := &Employee{Person: Person{"Bob"}, Role: "Engineer"}
    SayHello(bob)  // 输出："Hello, Bob"
}
```

`SayHello` 接受任何实现了 `Greet()` 的类型，体现接口多态。

**5. 设计风格：组合优先于继承**  
Go 鼓励用接口解耦组件逻辑，用组合替代继承树，从而避免复杂层级结构，并提高代码可组合性与测试性。若需扩展行为，只需添加新的类型实现接口即可，无需修改调用代码，符合开放-封闭原则。

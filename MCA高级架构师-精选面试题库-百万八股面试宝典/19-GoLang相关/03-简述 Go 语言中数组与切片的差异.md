在 Go 语言中，数组和切片虽相关但用途差异明显。M

**1. 固定 vs 动态**  
数组（如 `[3]int`）大小固定，声明时即确定，内存是一块连续区域；切片则是基于数组的动态视图，可以通过 `append` 动态扩展，也能缩小长度。S

**2. 值类型 vs 引用类型**  
数组作为值类型，复制或传参时，会拷贝整个内存；切片是引用类型，包含指向底层数组的指针、长度 `len` 和容量 `cap`，复制切片只复制这三项，操作仍作用于同一数据。B

**3. 底层结构**  
切片内部结构体大体如下：

```go
type _slice struct {
    ptr *T      // 指向底层数组
    len int     // 当前长度
    cap int     // 容量
}
```

它指向一个数组、记录长度和容量，这三个字段赋值时会复制，但底层数组不会被复制。

**4. 扩容机制**  
当 `len < cap` 时，`append` 会直接往原数组写入，`len` 增加；若 `len == cap` 则会触发扩容，通常分配新数组，容量翻倍（具体依情况而定），并复制旧数据。

**5. 切片共享与注意事项**  
多个切片可以共享同一个底层数组，通过 `slice1 := arr[1:4]` 会创建新的切片结构但指向相同数组，修改任何一个切片内元素都会影响其他切片，使用时要注意副作用。

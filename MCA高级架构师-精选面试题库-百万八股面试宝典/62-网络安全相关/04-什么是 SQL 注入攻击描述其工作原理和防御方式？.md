SQL 注入是一种攻击方式，攻击者通过在用户输入的位置注入恶意 SQL 代码，利用应用程序未对输入进行严格处理的漏洞，成功触发并执行数据库操作。S

首先，从运行机制讲，攻击者在表单或 URL 参数中嵌入 SQL 语句片段（例如 `'; DROP TABLE users;--`），打断原本的查询结构，使这些恶意代码被数据库引擎当作合法语句执行，可能导致数据泄露、篡改甚至删除。这种攻击依赖于代码中拼接 SQL 字符串的方式未将用户输入当作数据隔离开来。M

典型的后果包括：一是敏感数据泄露，如用户账户、信用卡信息；二是非法修改数据，甚至控制数据库；三是导致服务挂起或拒绝服务；四是在某些环境中形成对系统权限的提升。B

防御策略建议采用 **参数化查询（prepared statements）**。这是业界公认的最佳实践：通过使用占位符和绑定变量，确保用户输入只作为数据处理，无法改变 SQL 语句结构，从根本上避免注入风险。例如：

```plsql
cursor.execute(
  "SELECT * FROM users WHERE email = ?", 
  (email,)
)
```

再配合**最小权限原则**，即数据库账户仅具备执行必要操作的权限，可进一步降低潜在风险。

Scala中抽象类和Trait区别如下：

|  |  |  |
| --- | --- | --- |
| **特性** | **抽象类** | **Trait** |
| **关键字** | 使用 abstract class 定义 | 使用 trait 定义 |
| **构造参数** | 支持，可以定义构造参数，并且子类需要调用父类构造器 | 不支持 |
| **继承关系** | 只能继承一个抽象类，但可以实现多个 Trait | 可以通过 extends 和 with 实现多继承 |
| **动态混入** | 不支持动态混入 | 支持动态混入，可以在实例化对象时添加 Trait |
| **抽象成员支持** | 可以包含抽象方法和抽象字段，也可以有具体实现 | 可以包含抽象方法和抽象字段，也可以有具体实现 |
| **使用场景** | 定义一个具有通用状态或行为的父类，适用于描述“是什么”的场景，例如动物类、几何图形类等。 | 用于定义行为模块（例如日志记录、监听等）并在多个类中复用，给类动态扩展功能。 |

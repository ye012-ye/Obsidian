Java 对象的大小由 **对象头**、**实例数据** 和 **对齐填充** 三部分构成。首先，对象头包含 Mark Word（锁状态、GC 信息、hash 等）和 Klass Word（类指针）。在 64 位 JVM 开启压缩指针时，对象头为 12 字节；若禁用压缩，是 16 字节左右。M

接着，加上实例字段的大小。基本类型字段依其类型占 1 至 8 字节不等，引用类型字段在开启压缩指针时占 4 字节，否则占 8 字节。在多继承或字段顺序复杂时 JVM 会按类型分组并对齐。

最后，对象整体大小必须是 8 字节的整数倍，因此在末尾可能需要填充若干字节确保对齐。这种对齐有助于提升 CPU 访问效率并减少 GC 处理碎片的复杂度。S

例如，类 `class E { int x; long y; byte z; }` 在 64 位 JVM 启用压缩指针时：对象头 12 字节 + 字段数据（4+8+1 =13 字节）+ 填充 3 字节，最终总计 28 字节。

#### **回答注意点：**

- 工具如 JOL 可用于精准测量浅层和深层对象大小。
- 数组对象在头部额外包含长度字段（4 字节），同样对齐。
- 在新 JVM 版本中，对象头可能进一步简化，例如未来试验性“Compact Headers”功能将头部减至 8 字节，从而改变计算方式。B

一个 Java 进程在操作系统看来占用了多块关键内存区域，主要分为堆（Heap）和非堆（Non‑Heap）两大类。

堆内存是存放对象实例和数组的地方，分为新生代与老年代，由 GC 管理，申请 `-Xms/-Xmx` 配置。非堆内存则包含多个子区域：  
方法区（包括元空间 Metaspace）存储类元信息、常量池、静态变量和 JIT 编译后的代码；每个线程还独享栈空间用于局部变量、调用栈帧及越界检查；本地方法栈为 JNI 或本地代码服务；程序计数器追踪当前执行指令的位置；此外还包括直接内存、线程本地结构、代码缓存、JVM 内部数据结构等，这些区域不在 GC 管理范围内。

因此，一个 Java 进程真实占用内存远大于仅 Heap 的配置大小，还包括栈空间、元空间、直内存、代码缓存、本地方法栈和 JVM 本身的 native 内存，全面理解这些有助于调优性能与排查 OOM。

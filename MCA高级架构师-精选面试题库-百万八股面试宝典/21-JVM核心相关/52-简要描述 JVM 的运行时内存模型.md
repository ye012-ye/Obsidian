JVM 的运行时内存模型由多个逻辑区域组成，各区域职责清晰、界限分明，协同保证 Java 程序的执行效率和线程安全。

#### 1. 程序计数器（PC Register）

- 每个线程拥有独立的 PC 寄存器，用于记录当前正在执行的字节码指令地址。
- 仅在本地方法（native）调用时不可用，栈溢出也不会影响该区。M

#### 2. Java 虚拟机栈（Stack）

- 每个线程拥有独自的栈结构，由一个或多个栈帧组成，每帧包括局部变量表、操作数栈、动态链接和返回地址。S
- 方法调用时创建新帧，执行完毕后弹出，对线程安全没有影响。

#### 3. 本地方法栈（Native Method Stack）

- 专用于 JNI 调用 native 方法。结构和 JVM 栈类似。
- 当 native 方法执行失败时，也会抛出类似栈溢出的异常。

#### 4. 堆（Heap）

- JVM 中所有线程共享的区域，负责存储 Java 对象实例与数组。
- 分代模型（新生代 Eden、Survivor，老年代 Tenured）配合垃圾回收机制（Minor/Full GC）进行内存管理。

#### 5. 方法区（Method Area / Metaspace）

- 存储类信息：运行时常量池、字段和方法数据、JIT 编译后的代码等。B
- 从 Java 8 开始，方法区实现为 Metaspace，位于本地内存，不再是堆内存的一部分。

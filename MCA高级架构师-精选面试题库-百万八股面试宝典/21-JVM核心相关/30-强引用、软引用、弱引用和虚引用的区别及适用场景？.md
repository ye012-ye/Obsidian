在 Java 中，**强引用**是最常见的引用类型，例如 `mashibing Foo obj = new Foo();`。只要对象被强引用持有，垃圾回收器绝不会回收它，除非引用被显式置为 `null` 或超出作用域。这保证了对象的正常使用，但如果不及时释放，也可能导致内存一直被占用。M

**软引用**通过 `SoftReference<Foo> sf = new SoftReference<>(obj);` 创建。当对象仅由软引用可达时，JVM 会在内存紧张时才回收它，这使它非常适合做缓存等需要在内存允许时保留对象的场景。软引用的对象通常会在发生 `OutOfMemoryError` 之前被清理，以帮助减缓内存压力。

相比之下，**弱引用**用 `WeakReference<Foo> wf = new WeakReference<>(obj);` 创建，GC 一旦发现对象只由弱引用可达，就会立刻回收它。弱引用适合像 `WeakHashMap` 这样的场景，其中缓存条目不应妨碍垃圾回收。S

最弱的是 **虚引用**，通常配合 `ReferenceQueue` 使用，例如：

```java
ReferenceQueue<Foo> rq = new ReferenceQueue<>();
PhantomReference<Foo> pr = new PhantomReference<>(obj, rq);
obj = null;
```

虚引用不会阻止对象被回收，其 `get()` 方法总是返回 `null`，主要用于在对象被回收前做清理或释放非堆资源，或实现比 `finalize()` 更可靠的回收机制。

总体来看，这四种引用关系如下：**强引用 > 软引用 > 弱引用 > 虚引用**。选择何种引用类型，应结合具体需求：如果你希望对象只在空闲内存允许的情况下存在，用软引用；如果不想影响回收，用弱引用；若需在回收前执行清理逻辑，则使用虚引用。B

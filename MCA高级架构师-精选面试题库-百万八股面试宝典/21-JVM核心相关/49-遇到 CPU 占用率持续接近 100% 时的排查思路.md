当系统 CPU 长时间高达 100%，会严重影响性能。有效的排查应当遵循 : "定位 → 分析 → 优化"流程 , 步骤如下：

#### 1. 定位热点进程

- 在 Linux 上使用 `top`（或更交互友好的 `htop`）查看占用最高的进程，关注 `%CPU`、`load average` 等关键指标。
- 在 Windows 环境下，打开任务管理器（Task Manager），切换到 Performance 或 Processes 选项卡，按照 CPU 排序查找问题进程 。  
  M

#### 2. 进程内部分析

- Linux 可结合 `mpstat`、`pidstat`、`perf` 等工具，查看系统与用户态的 CPU 使用分布，确认是否为系统调用或用户态计算密集型过程 。
- Windows 可启动 Resource Monitor 查看线程级 CPU 分布，辅助判断每个线程的执行情况。

#### 3. 检查系统或驱动原因

- 若高 CPU 属于系统进程（如 System 或 kernel task），可能是驱动异常或硬件中断过多。Linux 可追踪 `dmesg` 输出，Windows 可用 ETW/xperf 工具深度采样。
- 若 CPU 占用在 I/O 等待中的比例异常升高，说明可能存在磁盘 / 网络瓶颈。

#### 4. 应用层问题排查

- 许多情况是应用中存在死循环、频繁 GC、锁争用或慢 SQL 导致。S
- Java 等可用 `jstack`、`jmap`、Arthas、VisualVM 检查热点线程或 GC 日志；数据库慢查询则检查 DB 日志进行优化。

#### 5. 杀毒、系统清理与更新

- CPU 高利用也可能是因恶意软件在后台运行，建议执行杀毒、关闭不必要的后台服务，对 Windows 推荐关闭 SysMain、Windows Search 等高频访问型服务。
- 同时确保操作系统、驱动、BIOS/固件保持最新，以避免已知 bug 引发的持续性高耗。

#### 6. 重启 & 压力测试

- 重启可作为临时缓解手段，但如问题重复出现，则需要进行压力测试、流量仿真、定位瓶颈。
- 对 CPU 发热或供电异常的硬件问题，通过压力测试（stress tools）验证，并进行清理散热或更换硬件 。B

## 1.简述 java 垃圾回收机制?

"Java的垃圾回收（GC）是JVM自动管理内存的核心机制，它通过‌分代收集‌策略高效处理内存回收。简单来说，堆内存分为新生代和老年代——新创建的对象优先在新生代分配，存活一定周期后晋升到老年代。GC会定期扫描对象，基于‌可达性分析‌（从GC Roots出发追踪引用链）判断哪些对象不再被使用，然后自动回收它们的内存。

新生代通常用‌标记-复制算法‌（快速但占用额外空间），老年代则用‌标记-整理‌或‌标记-清除‌（避免碎片但耗时较长）。开发者可以通过选择不同的GC器（如G1、ZGC）来平衡吞吐量和延迟。

重点：分代收集 + 可达性分析 + 算法特点

## 一、什么是GC？

参考回答：

GC（Generational Collection）是一种分代收集算法，用于释放JVM中那些不再使用的对象所占用的内存。一般常称之为垃圾回收机制。

## 三、解释一下Java中垃圾收集的主要流程

参考回答：

垃圾收集主要有两种形式：

（1）自动：自动会不定期进行回收，以释放无用的空间。

（2）手工：手工调用的是System类中的gc()方法，此方法实际上调用的是Runtime类中的gc()方法，当一个对象被回收之前将调用类中的finlalize()方法，此方法为 Object类所提供，表示对象回收前的收尾工作。即使出现了异常，也不影响程序的执行，而且此方法抛出的是Throwable，表示可能是异常也可能是错误。

## 六、垃圾回收机制的特点?

参考回答：

Java垃圾回收机制的特点是：

（1）垃圾回收机制自动运行

系统提供一个低优先级的线程来跟踪内存的分配情况。如果发现某个内存单元不再使用，就清除它。

（2）何时清除垃圾是不可预期的

当垃圾搜集器运行时，其目的是查找和删除不能被访问的对象。

垃圾收集器运行时，当它发现一个对象不能被任何活线程访问时，它将认为该对象符合删除条件，它可能在某时删除该对象。 垃圾收集器不能保证有足够的内存，它只能保证可以使用的内存将尽可能被有效的管理。

使对象符合垃圾搜集器搜索条件的情况有：

1）出现空引用

2）重新为引用变量赋值

3）隔离引用

4）强制执行垃圾搜集

## 八、GC的触发条件

参考回答：

（1）执行System.gc()的时候：建议执行Full GC，但是JVM并不保证一定会执行

（2）新生代空间不足

由于对象大都在Eden区分配内存，如果某个时刻JVM需要给某一个对象在Eden区上分配一块内存，但是此时Eden区剩余的连续内存小于该对象需要的内存，Eden区空间不足会触发minor GC。

触发minor GC前会检查之前每次Minor GC时晋升到老年代的平均对象大小是否大于老年代剩余空间大小，如果大于，则直接触发Full GC；

否则，查看HandlePromotionFailure参数的值，如果为false，则直接触发Full GC；

如果为true（默认为true，表示允许担保失败，虽然剩余空间大于之前晋升到老年代的平均大小，但是依旧可能担保失败），则仅触发Minor GC；

如果期间发生老年代不足以容纳新生代存活的对象，此时会触发Full GC 。

（3）老年代空间不足

大对象直接进入老年代，经常出现大对象容易导致内存还有不少空间时就提前触发垃圾收集以获取足够多的连续空间；

长期存活的对象将进入老年代；

如果survivor空间中相同年龄所有对象大小的总和大于survivor空间的一半，年龄大于等于该年龄的对象就可以直接进入老年代；

CMS GC在出现promotion failure和concurrent mode failure的时候

上面这几种情况会导致“老年代“满”，会触发full GC。

## 九、你能保证 GC 执行吗?

参考回答：

不能保证 Java 的 垃圾回收 （GC）执行。虽然可以通过调用`System.gc()`或`Runtime.getRuntime().gc()`建议JVM执行垃圾回收，但这些操作只是建议而非强制命令，JVM可能根据当前内存使用情况选择忽略该请求

原因解析：

1. ‌**自动管理机制**‌

2. ‌**强制执行的局限性**‌  
   `System.gc()`，JVM也可能因性能优化或内存充足等原因延迟或忽略回收请求，且频繁强制执行可能影响程序性能

## 十、为什么不能显示直接调用finalize方法？

参考回答：

finalize方法在垃圾回收时一定会被执行，而如果在此之前显示执行的话，也就是说finalize方法会被执行两次以上，而在第一次资源已经被释放，那么在第二次释放资源时系统一定会报错，因此一般finalize方法的访问权限和父类保持一致，为protected。

## 十一、4大垃圾回收思想是什么？

参考回答：

（1）分代回收（根据对象生命周期划分新生代（短期存活）和老年代（长期存活），分别采用不同算法。新生代用复制算法，老年代用标记整理或标记清除）

（2）复制（将存活对象复制到新内存区域，原区域释放。适用于存活对象少的情况（如新生代），需额外空间）

（3）标记清除 （通过标记所有从根节点可达的对象，未被标记的对象即为垃圾。适用于存活对象较多时，但会产生内存碎片）

（4）标记整理 （在标记清除基础上，将存活对象压缩到一端，避免碎片。适用于老年代等存活对象较多时）

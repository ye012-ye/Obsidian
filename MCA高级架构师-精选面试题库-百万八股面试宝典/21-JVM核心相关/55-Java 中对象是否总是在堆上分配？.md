在 Java 中，并非所有对象都必须分配在堆上。现代 JVM 借助即时编译器和高级优化技术，可以根据对象使用情况，动态选择更高效的分配策略。

首先，**逃逸分析（Escape Analysis）** 是一种静态分析技术，用于判断对象是否会“逃逸”出当前方法或线程。如果分析结果显示对象只在方法内部被使用，且没有被返回、存储到成员变量或共享线程访问，JVM 可以将其视为非逃逸对象，从而选择 **栈上分配**。这减少了 GC 负担，也提升了内存访问效率。M

此外，当对象结构简单，属性可独立访问时，JVM 会执行 **标量替换（Scalar Replacement）**，将对象拆解成多个基本类型变量直接存于局部变量甚至寄存器里，无需在栈或堆上形成完整对象。S

在某些极端优化下，这些“对象”可能完全驻留于 CPU **寄存器** 中，不会占用任何内存空间。B

以下是核心关键点总结：

- **逃逸分析**：判断对象是否仅在局部使用；
- **栈分配**：适用于非逃逸对象；
- **标量替换**：将对象拆解为原始变量；
- **寄存器分配**：优化后的最优空间使用。

**示例说明：**

```java
public void foo() {
    Point p = new Point(1, 2);
    System.out.println(p.x + p.y);
}
```

在 `foo()` 方法中，若 `Point` 对象未被外部引用，即可由 JVM 逃逸分析判定为非逃逸对象，然后将其安全分配在栈上，进一步进行标量替换，直接将 x 和 y 存为局部变量，无需完整对象分配。

**总结：**  
现代 JVM 利用逃逸分析、标量替换与寄存器分配优化，能够让部分对象避免堆分配，在栈或寄存器中高效管理，从而提升性能、降低 GC 开销。

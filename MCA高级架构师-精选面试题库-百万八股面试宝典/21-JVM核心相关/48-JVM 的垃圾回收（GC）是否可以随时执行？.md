JVM 的垃圾回收（GC）**不能随时随地**执行，它只能在程序运行到所谓的 **安全点（Safepoint）** 时暂停线程并触发 GC 操作。Safepoint 是 JVM 在字节码执行的特定位置（如方法调用/返回点、循环边界、异常抛出点或执行 JNI/native 代码期间）插入的检查机制。线程在这些点“自愿”停下来，确保堆和栈的状态是一致、可追踪的，从而让 GC 或其他 VM 操作安全执行。

当 JVM 需要执行 GC 或类重新定义、线程转储、deoptimization 等操作时，会设置一个全局 safepoint 标志。每个线程在下一个安全点检测到该标志后进入等待状态，一旦所有线程均已停下，所需操作即可执行，完成后清楚标志并恢复线程。这种机制避免在任意处暂停可能导致状态不一致或栈损坏，保护 Java 应用的稳定性与可预测性。

简单来说，GC 只能在程序执行到安全点时才会触发，以保证线程状态清晰、安全、不影响正在进行的操作。

Java 的字符串常量池（String Constant Pool）是 JVM 为优化字符串重复分配而设计的一种特殊缓存机制。尽管在源码层面未以 Java 类实现，而是由底层 JVM（例如 HotSpot）的 C++ 代码管理字符池，整体逻辑可归纳如下：

### ​1. 池的结构与存放方式

- 池本质上是一个基于哈希表的结构，存储于 JVM 堆空间（JDK7 及以上）的专用区域中，用于存放字符串字面量的唯一实例。
- JVM 加载类时，会将字面量字符串加入该池；运行时调用 `String.intern()` 也会触发该操作。

### 2. `intern()` 的行为

- 原生方法（native）`String.intern()` 会检查常量池中是否已存在相同内容的字符串：

- 若存在，返回池中的引用；
- 若不存在，将堆中对应的字符串实例放入池中并返回它。

- 这机制使相同内容的字符串不会重复存储，提升内存利用率。

### 3. 字符串字面量 vs `new String(...)`

- 使用双引号 `"..."` 时，JVM 编译期直接将其放入常量池，下次引用直接复用，因此 `"hello" == "hello"` 返回 `true`。
- `new String("hello")` 会在堆上额外创建一个对象，即使字面量已在池中，比较引用时仍为不同对象。
- 调用 `new String("hello").intern()` 后，返回值为池中的实例，从而实现统一引用。

### 4. 池的垃圾回收

- JDK 7 之前，常量池位于 PermGen 区，无法被垃圾回收；
- JDK 7+ 常量池迁移至堆，允许 JVM 对无引用的常量回收机制参与清理，有助于释放内存。

### 5. 性能权衡

- 池结构减少系统中的重复字符串，占用较低，符合 Flyweight 模式；
- 查找与插入需要走哈希过程，稍有计算耗费，尤其是在池持续膨胀时 。

### 示例演示

```java
String a = "World";
String b = new String("World");
String c = b.intern(); 

System.out.println(a == b); // false，b 是新对象
System.out.println(a == c); // true，c 指向池中的实例
```

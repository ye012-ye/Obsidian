Java 判断对象是否可回收，采用的是 **可达性分析（Reachability Analysis）** 方法。当 JVM 触发 GC 时，它会从一组称为 **GC Roots** 的根元素出发，沿引用链遍历所有引用对象，凡是无法被任何 GC Roots 直接或间接访问到的对象，就会被认定为**不可达**，从而可安全回收。

GC Roots 包括：

- 线程栈中的局部变量与方法参数；
- 正在运行的线程本身；
- 类中的静态变量和常量；
- JNI（本地方法）所持有的 Java 对象引用；
- JVM 内部引用，比如监视器或系统类加载器等。

这套机制保证仅回收真正无用的对象，避免误删仍被程序引用的实例，并为垃圾回收提供高效且正确的判断依据。

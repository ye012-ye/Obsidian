在 Java 中，**内存泄漏**是指程序中不再使用的对象仍被引用，因此无法被垃圾回收回收，导致堆内存随着时间持续增大但并未立即触发崩溃。这种问题慢慢累积，可能导致应用性能变差甚至最终出现 `OutOfMemoryError`。M

相比之下，**内存溢出（OutOfMemoryError）** 是程序尝试分配新对象时，JVM 无足够可用内存，导致抛出异常并通常中断执行。这种情况可以是因为内存泄漏引起，也可能只是一次性申请过大的数据结构（如巨型数组），或 JVM 堆配置过小。

两者关系是：内存泄漏是一种持续性、隐匿的“气球泄气式”增长行为；而内存溢出则是一种瞬发性、明确的“爆炸式”错误。泄漏不会马上停程序，但会逐渐把内存 “占满”；溢出则直接发生在申请新内存失败的那一刻。S

如果应用表现为**内存使用缓慢上涨但依然正常运行**，则更可能是内存泄漏问题。这种状况可通过 heap dump、MAT 等进行排查，并找到持有无用对象的引用路径。

而若是程序**突然抛出 OOM 并终止**，应立即检查是否存在一次性大对象分配或 JVM 配置不足，此时应分析 OOM 堆栈信息，如 `Java heap space`、`Metaspace`、或 `Requested array size exceeds VM limit`，并采取对应措施。B

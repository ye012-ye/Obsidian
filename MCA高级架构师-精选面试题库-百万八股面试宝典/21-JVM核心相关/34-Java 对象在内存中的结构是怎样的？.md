Java 对象在堆上由三部分依次组成：**对象头 + 实例数据 + 对齐填充**，这种结构确保了运行效率与灵活性。

首先，对象头包括两项关键内容：一个 Mark Word 和一个 Klass Word（类型指针）。Mark Word 用于存放运行时信息，如锁状态、GC 年龄、hashCode 等；Klass Word 指向对象的类元数据，支持动态类型判断和方法分发。在开启压缩指针（Compressed Oops）模式下，64 位 JVM 的对象头通常为 12 字节（8 字节 Mark + 4 字节 Klass），禁用压缩则为 16 字节。

紧接着是**实例数据区**，保存类声明的字段（包括继承自父类的字段）。JVM 根据字段类型将其按从大到小整理存放（long/double → int/float → short/char → byte/boolean → 引用类型），以减少内部碎片，优化访问效率。

最后，**对齐填充**确保整个对象大小是 8 字节（或平台指定边界）的整数倍。如果头部与数据总和不是对齐目标的整数倍，会添加少量填充字节保证对齐。

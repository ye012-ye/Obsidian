JVM 使用两种常量池机制，以支持编译期信息保存和运行期优化，保障字节码加载与执行的有效性M。

#### 1. Class 常量池

- 定义为 `.class` 文件中的静态数据结构，由编译器生成，位于每个类/接口文件中。它包含字面量（如字符串、整数）、符号引用（如类名、字段名、方法名与描述符）等信息。
- 特点：编译阶段固定，随 `.class` 文件分发；是 JVM 解析链接类时的依据。

#### 2. 运行时常量池

- 是 JVM 在类加载阶段将 Class 常量池内容转换并存储于方法区（或 Java 8+ 的 Metaspace）中的数据结构，用于解析与链接。
- 动态性：不仅加载编译常量，还支持运行时添加常量（如 `String.intern()`），特别是在字符串池操作时。
- 角色：维持符号引用解析，用于支持指令如 `ldc`、`invokevirtual`、`getfield` 等的运行时执行。S

​

#### 三者关系对比：

|  |  |  |  |
| --- | --- | --- | --- |
| 特性 | Class 常量池 | 运行时常量池 | 字符串池 |
| 所在阶段 | 编译时，存 `.class`  文件 | 类加载后，驻方法区/Metaspace | 堆中（String 池，Java 7+） |
| 内容 | 符号引用、字面量等 | 编译内容+运行新增（如 intern） | 字符串对象的缓存 |
| 用途 | 链接阶段引用解析 | 支持运行指令、优化性能 | 减少重复字符串、共享实例 |

#### 举例说明：

当加载如下代码时：

```java
String s = "Hello";
int n = 100;
```

`"Hello"` 和 `100` 首先记录在 Class 常量池中。加载时进入 运行时常量池：

- `"Hello"` 会被创建为 `String` 对象并加入字符串池；
- `100` 被加载为 Integer 常量；B
- 在执行 `ldc "Hello"` 及相关指令时，JVM 从运行时常量池取得对应内容，为执行提供基础结构。

Java 中的类加载机制由多个加载器组成，形成一个分层结构，保障类的统一性、安全性与可扩展性。M

#### 1. 三类内建加载器

- **Bootstrap（引导类加载器）**：由 JVM 本身实现，负责加载核心 Java 库（如 `rt.jar`），通常用 native 代码实现，父加载器为空。
- **Platform / Extension（扩展类加载器）**：加载 Java 的扩展类库，如 `JAVA_HOME/lib/ext` 中的内容，由引导加载器的子类负责。
- **Application / System（系统类加载器）**：加载用户类路径（classpath）下的应用类，是开发中最常用的类加载器，也是扩展加载器的子类。

此外，用户可自定义加载器继承 `ClassLoader`，通过重写 `findClass(...)` 方法从特定来源（如网络、JAR、数据库）加载类。

#### 2. 双亲委派模型（Parent Delegation）

采用“请求 → 委派 → 本地加载”流程：

1. 调用 `loadClass(name)` 时，当前加载器先调用 `findLoadedClass(name)` 检查缓存。
2. 缓存未命中，则把加载请求委派给父加载器，由它去加载。S
3. 若父 loader 无法加载，再由当前加载器调用 `findClass(name)` 从自身路径加载。

这种模型确保核心类可由引导加载器统一加载，避免了核心类被恶意或重复加载，从而提高安全性与稳定性。

#### 3. 模型优势

- **统一性**：相同全限定名的类只会由同一个加载器加载，避免类冲突。
- **安全性**：核心类先由 Bootstrap 加载，且子加载器无法覆盖；防止用户定义类篡改标准库。
- **扩展友好**：自定义加载器方便插件机制、动态任务或隔离框架，如应用服务器中的 WAR/EAR 隔离加载。

#### 4. 加载生命周期（简述）

加载完成后，类还需经历三步链接和初始化：

- **验证（Verify）**：校验字节码安全性；B
- **准备（Prepare）**：为 `static` 变量分配并设初始值；
- **解析（Resolve）**：将符号引用解析为直接引用；
- **初始化（Initialize）**：执行类构造器 `<clinit>` 等静态初始化逻辑。

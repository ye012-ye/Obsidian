JVM 会在以下几种场景下退出：

1. **正常结束**：`main()` 方法执行完成，且所有非守护线程（用户线程）结束后，JVM 会自动退出。守护线程不会阻止 JVM 退出。M
2. **显式调用退出**：若程序调用 `System.exit(int status)`，JVM 会启动关闭流程，包括调用已注册的 shutdown hook，再以指定状态码终止运行。
3. **调用 Runtime.halt()**：这是最强制的终止方式，不触发 shutdown hook，也不等待线程结束，直接停止虚拟机。S
4. **未捕获的异常**：

- 如果 `main()` 或其他用户线程抛出未捕获异常，会导致该线程终止。如果是非守护线程，且没有 catch 全局处理，可能引发 JVM 退出。
- 守护线程的异常只终止自身，不影响 JVM，如果还有其他用户线程仍在运行，JVM 不会退出。

5. **JVM 崩溃**：因本地代码出错、内存访问违规、原生库异常（如 JNI）等原因引起 JVM 崩溃、进程终止。
6. **外部杀死进程**：使用系统命令（如 Unix 下的 `kill`）或任务管理器强制终止 Java 进程。B

哈希表是一种通过哈希函数将键映射到数组索引的结构，旨在实现插入、查找和删除的平均 O(1) 时间性能。其核心在于将键“哈希”为一个整数，再根据表长度取模得到桶位置，从而直接定位操作目标。同时构建良好哈希表，需要充分考虑冲突处理、负载因子和扩容机制。

M

#### 1. 哈希函数与基础机制

哈希函数接收输入键，输出范围在 `0…m-1` 的整数，通过 `% m` 得到数组索引。理想情况下，同一哈希函数对不同键的映射分布要尽可能均匀，以避免冲突。

#### 2. 冲突处理策略

由于不同键可能映射到相同位置，需有效解决冲突：

- **链地址法（Separate Chaining）**  
  每个数组位置维护一个链表或桶结构，冲突键将附加到链表中。它实现简单，负载因子可超过 1，扩容机制对性能影响较小。但当链过长时，查找将退化至 O(n)。
- **开放地址法（Open Addressing）**  
  冲突时，在数组中按探查策略寻找下一个可用桶，插入、查找均在表内完成。常见方式包括线性探测、二次探测和双重哈希：

- **线性探测**：探查步长固定为 1，当连续存储存在冲突时可能形成集群，但缓存表现较好。
- **二次探测**：步长随探查次数平方增长，能减轻聚集问题，但实现复杂度略增。
- **双重哈希**：使用第二哈希函数决定步长，不同键产生不同探查序列，能最大限度减少聚集，但需额外计算第二哈希。

开放地址法适合装载率不高的场景。装载率（n/m）接近 1 时，性能显著下降，通常建议控制在 0.6–0.75 左右。

S

#### 3. 负载因子与扩容策略

负载因子决定性能表现，过高则冲突增多。链地址法负载因子可适度大于 1，而开放地址法需控制在容量的 60–75% 以下。多数实现采用插入触发扩容（如扩容为原来两倍），并重新哈希所有元素，以维持平均 O(1) 时间复杂度。

B

#### 4. 各策略优缺点对比

|  |  |  |
| --- | --- | --- |
| **方案** | **优点** | **缺点** |
| 链地址法 | 实现简单，扩容灵活，贴合多元素冲突场景 | 链表访问存在指针跳转，缓存性能稍差 |
| 开放地址（线性） | 内存紧凑、缓存利用好 | 易产生聚集，装载高于阈值时性能急剧下降 |
| 开放地址（二次/双重） | 可缓解聚集问题，搜索路径多样 | 实现复杂度略增，需配合良好哈希函数 |

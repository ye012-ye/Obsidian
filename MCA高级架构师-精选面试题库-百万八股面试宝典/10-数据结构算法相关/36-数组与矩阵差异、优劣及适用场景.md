#### 1. 数组（Array）

数组是一维线性表，所有元素类型相同并连续存储，因此可以通过索引在 *O(1)* 时间内访问任意元素。这种结构具有良好的缓存局部性，能高效实现数据访问与批量处理，适合频繁读取和遍历的场景。缺点是无法动态扩展插入中间元素开销较大，需要移动后续元素，插入/删除操作时间复杂度为 *O(n)*。当开发者需要固定大小存储、快速随机访问，且修改操作不频繁时，数组是最佳选择。

#### 2. 矩阵（Matrix）

矩阵实际上是二维数组，按行列组织元素，也在内存中连续存储（采用行优先或列优先布局）。在矩阵中访问一个元素同样是 *O(1)*，并支持高效的行列操作、数学计算（如矩阵乘法）。但它同样不适用于频繁变更尺寸的场景，因为扩容和插入成本也很高。矩阵特别适合用于图像处理、图算法、线性代数及科学计算等需要二维数据布局的应用场景。

#### 3. 结构对比

|  |  |  |
| --- | --- | --- |
| **特性** | **数组（1D）** | **矩阵（2D 数组）** |
| 维度 | 一维 | 二维（行×列） |
| 存储方式 | 连续内存 | 连续内存，按行/列主顺序布局 |
| 访问时间 | O(1) | O(1)，通过行列索引定位 |
| 插入/删除性能 | 中间操作成本高（O(n)） | 中间操作成本高，扩展成本大 |
| 适用场景 | 数据缓存、高速读取 | 图像处理、矩阵计算、网格数据表示 |

#### 4. 使用建议

- **优选数组**：当场景只需一维，有固定大小或扩展机制时，如动态数组、List 实现、哈希表底层等。
- **选择矩阵**：处理二维结构时更直观高效，如棋盘、像素图、图连接矩阵、表格运算等。

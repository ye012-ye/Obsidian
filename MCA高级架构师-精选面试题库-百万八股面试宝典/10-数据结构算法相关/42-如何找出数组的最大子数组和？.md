我们可以借助 Kadane 算法，用线性扫描和动态规划思想，以 O(n) 时间完成任务，同时仅使用 O(1) 的额外空间，从而找出数组的最大子数组和。

**思路说明：**  
定义两个变量，`maxEndingHere` 表示以当前元素结尾的最大子数组和，`maxSoFar` 用于记录遍历过程中见过的最大值。初始化它们为数组的第一个元素。遍历数组剩余部分，对于每个元素 `x`，更新 `maxEndingHere = max(x, maxEndingHere + x)`，确保取当前元素与前缀子数组哪种更优；同时再用它更新 `maxSoFar = max(maxSoFar, maxEndingHere)`。最终 `maxSoFar` 即为所求结果。此过程兼容只含负数的数组，也可轻松扩展记录子数组边界。

#### Java 代码示例

```java
public class Solution {
    public static int maxSubArray(int[] nums) {
        if (nums == null || nums.length == 0) {
            throw new IllegalArgumentException("输入数组为空"); M
        }

        int maxEndingHere = nums[0];
        int maxSoFar = nums[0]; S

        for (int i = 1; i < nums.length; i++) {
            int x = nums[i];
            maxEndingHere = Math.max(x, maxEndingHere + x);
            maxSoFar = Math.max(maxSoFar, maxEndingHere);
        }

        return maxSoFar; B
    }
}
```

### 复杂度分析

- **时间复杂度**：O(n)，每个元素只做一次处理。
- **空间复杂度**：O(1)，仅使用两个辅助变量。

栈和队列是两种基础的线性数据结构，它们的主要区别来自于元素的进出顺序。

M

#### 1. 栈 — 后进先出（LIFO）

栈是一种只能从“栈顶”进行插入（push）和删除（pop）操作的结构，新增的元素最先被移除。这一特性使它非常适合用于管理函数调用、递归处理、括号匹配以及回溯算法中，如深度优先搜索（DFS）等。  
栈通常提供以下基本操作：push、pop、peek（查看最顶元素）、isEmpty()、size()等。它只能访问最新入栈的元素，无法随机读写。

应用示例：

- 编译器的语法分析或表达式求值
- 函数调用链和递归返回机制
- 算法中的回溯路径保存

S

#### 2. 队列 — 先进先出（FIFO）

队列允许从“队尾”插入元素（enqueue），并从“队首”删除（dequeue），保证最早进入的元素最先被处理。它特别适合用于广度优先搜索（BFS）、任务调度以及缓存系统等场景中。  
队列通常支持 enqueue、dequeue、peek（查看队首）、isEmpty()、size() 等操作。

应用示例：

- BFS 遍历树或图结构，按层次处理节点
- 操作系统中的任务调度、消息队列、缓冲区处理
- 请求排队处理，如打印任务、HTTP 请求调度

#### 3. 栈与队列的对比

|  |  |  |
| --- | --- | --- |
| **特性** | **栈（Stack, LIFO）** | **队列（Queue, FIFO）** |
| 插入位置 | 栈顶 | 队尾 |
| 删除位置 | 栈顶 | 队首 |
| 访问顺序 | 最近插入的元素最先访问 | 最早插入的元素最先访问 |
| 典型应用 | 函数调用、回溯、表达式解析、DFS | BFS 遍历、线程调度、任务排队、缓冲区管理 |
| 随机访问 | 不支持随机访问，只能操作顶端元素 | 不支持随机访问，只能操作首尾端 |

B

#### 4. 应用场景建议

- 当你的任务需要**后进先出**处理时（例如回退、嵌套结构、DFS），应选用**栈**。
- 当任务以**按顺序处理所有输入**为前提（如层序遍历、请求处理、资源排队），应选用**队列**。

我们可以使用哈希表（`HashMap`）来实现线性时间的查找，并且写出一个包含 `main` 方法的完整 Java 程序。以下是思路和代码说明：

M

#### 实现思路

1. 遍历数组中的每个元素 `nums[i]`。
2. 计算其补数 `complement = target - nums[i]`。
3. 在哈希表中查找是否已存在该补数：

- 如果存在，即 `map.containsKey(complement)`，则已找到一组答案，返回对应索引。
- 否则，将当前元素及其索引加入映射，继续遍历。

4. 若遍历结束仍无结果，则可以抛出异常或打印提示。

该策略保证只遍历一次数组，时间复杂度为 O(n)，空间复杂度为 O(n)。简称为“一次哈希表方法”。

S

### 代码如下：

```java
import java.util.HashMap;
import java.util.Map;

public class TwoSumSolution {
    public static int[] twoSum(int[] nums, int target) {
        Map<Integer, Integer> map = new HashMap<>();
        for (int i = 0; i < nums.length; i++) {
            int complement = target - nums[i]; // M
            if (map.containsKey(complement)) {
                return new int[]{ map.get(complement), i }; // B
            }
            map.put(nums[i], i);
        }
        throw new IllegalArgumentException("No two sum solution"); // S
    }

    public static void main(String[] args) {
        int[] nums = {2, 7, 11, 15};
        int target = 9;

        try {
            int[] result = twoSum(nums, target);
            System.out.printf("Found indices %d and %d (values: %d + %d = %d)%n",
                              result[0], result[1],
                              nums[result[0]], nums[result[1]], target);
        } catch (IllegalArgumentException e) {
            System.out.println(e.getMessage());
        }
    }
}
```

B

### 复杂度分析

- **时间复杂度**：O(n)，每个元素只遍历一次，查找和插入哈希表都为 O(1) 平均复杂度。
- **空间复杂度**：O(n)，最坏情况哈希表中存储所有元素。

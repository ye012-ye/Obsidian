消息队列是用于系统间异步通信的一种机制，其核心在于将消息暂存、缓冲、传递，解耦发送方和接收方。具体要点包括：M

- 消息是程序间交换的数据单元，可能简单（文本）或复杂（结构体）。
- 发送者发布消息到队列后即返回，系统负责可靠投递；接收者从队列中取出消息并消费，彼此无需直接感知对方存在。

**Kafka 与典型 MQ（如 RabbitMQ、ActiveMQ）对比：**

1. **架构模式差异**

- 传统 MQ：采用 push 模式，broker 控制消费投递与 client 状态。
- Kafka：基于分布式日志的 pull 模式，消费者主动拉取，高吞吐并允许多消费者并行处理同一个 topic 中的数据。

2. **持久化与顺序保障**

- 传统 MQ 主动清除已消费消息。
- Kafka 将消息追加写入磁盘，按 offset 保留一段时间，可实现顺序消费与消息重放。

3. **可扩展性与吞吐量**

- Kafka 使用 partition 强分区，topic 可水平扩展，每个 partition 支撑独立写读流，broker 集群协作吞吐巨大。
- 相比之下，传统 MQ 在扩展上要依赖复杂集群部署、复制和路由机制。

4. **故障恢复与消费控制**

- Kafka 由消费者自行提交 offset，支持 at‑least‑once、exactly‑once 模式，可精准控制失败后的重试或跳过。
- 传统 MQ 更多在 broker 端实现可见度超时与确认机制，控制较集中但不够灵活。B

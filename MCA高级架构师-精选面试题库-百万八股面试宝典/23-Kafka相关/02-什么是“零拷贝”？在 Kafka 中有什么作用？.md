Kafka 高效数据传输的关键优化之一就是**零拷贝（Zero Copy）**，它依赖操作系统提供的机制，极大减少了数据在内核态与用户态之间的冗余拷贝。

M

### 1. 零拷贝原理

- 传统 I/O 流程会经历④次数据复制＋④次上下文切换：磁盘 → 内核缓存 → 用户缓存 → Socket 缓存 → 网络设备，开销高昂。
- 零拷贝机制将其精简为只需②次复制（磁盘 → 内核缓存 → NIC 缓冲区）＋②次上下文切换，大幅降低 CPU 与内存开销。

S

### 2. Kafka 如何实现零拷贝

Kafka 结合了两种 Java NIO 技术：

- **mmap（内存映射）**  
  将日志 segment 文件映射至进程虚拟地址空间，通过操作页缓存直接访问数据，无需用户态复制，适用于小规模和随机访问场景。
- **sendfile 系统调用**  
  Kafka 调用 `FileChannel.transferTo()`，利用 Linux `sendfile` 功能直接将 page cache 中数据发送至 network socket。适合大批量、顺序消费场景。

B

### 3. Kafka 中零拷贝的优势

- **吞吐提高**：减少大量冗余操作，数据可达网络传输速度上线。
- **CPU 利用率降低**：无需频繁用户态扫描/拷贝，解放 CPU 资源。
- **I/O 效率提升**：相同数据被多个消费者复用，不重复触发磁盘读。

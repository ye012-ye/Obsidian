Spring 是一个轻量、非侵入式的 Java 企业级应用开发框架 ，具备如下核心功能：

### 1. IoC 容器 & DI（控制反转与依赖注入）

Spring 核心是其 IoC 容器，通过配置（XML、注解或 Java 配置），容器负责创建 Bean、管理生命周期、注入依赖，达到高内聚、低耦合的目标。它用 BeanFactory 和 ApplicationContext 实现这一机制，非常适合大型可维护架构设计。

M

### 2. AOP（面向切面编程）

Spring 提供基于代理的 AOP 支持，允许我们将事务、安全、日志、监控等横切关注点从核心业务逻辑中分离出来，实现模块化和重用。采用 `@AspectJ` 注解或 XML 配置，能在运行时动态织入这些拦截行为。

S

### 3. 丰富的生态模块

除了核心容器、AOP，Spring 还提供数据访问（JDBC、ORM）、事务管理、Web MVC、消息、测试等模块。开发者可以根据业务需要，按需组合使用，构建完整的企业级系统 。

B

### 4. 轻量级与非侵入式

Spring 不要求业务类继承特定类或实现接口，只要用注解或配置即可被容器管理，保证业务代码纯净、易于测试和维护，符合现代开发最佳实践。

在 Spring MVC 中，`forward:` 和 `redirect:` 前缀用于控制请求的进一步处理方式，各自原理和应用场景如下：

M

### ​Forward（服务器内部转发）

- 使用方式：在 controller 中返回 `"forward:/目标路径"`；如 `return "forward:/home";`
- **内部机制**：Spring 的 `UrlBasedViewResolver` 识别 `forward:` 前缀，并生成一个 `InternalResourceView`，接着调用 `RequestDispatcher.forward()`，将在服务器端将请求转向另一个资源处理。该操作在服务器内部完成，对客户端透明，**URL 不改变**。
- **优点**：无需客户端参与，避免一次新的 HTTP 请求，适合在一个请求内共享数据（如设置请求属性）；
- **缺点**：请求共享状态不清晰，容易产生重复提交问题，例如刷新可能导致重复操作。

S

### ​Redirect（客户端重定向）

- 使用方式：返回 `"redirect:/目标路径"`；例如 `return "redirect:/home";`
- **实现机制**：`UrlBasedViewResolver` 识别 `redirect:` 前缀，并最终调用 `HttpServletResponse.sendRedirect()`（默认返回 302 或视配置返回 303 状态码），告知浏览器发送新的请求到指定 URL。
- **特点**：

- 浏览器发起一次全新的请求，浏览器地址栏变化；
- Session、FlashAttributes、URL 参数或 query 参数用于跨请求传递数据；
- 可避免 POST 请求的重复提交，符合 PRG（Post-Redirect-Get）模式。

### 比较

|  |  |  |
| --- | --- | --- |
| **特性** | **Forward** | **Redirect** |
| 请求类型 | 单个请求 → 同一次服务器内部处理 | 两次请求 → 客户端主导 |
| 地址栏 URL | 不变 | 改变 |
| 数据传递方式 | 可通过 request attribute | 通过 query 参数或 FlashAttribute |
| 场景推荐 | 适用于同一请求内共享数据或视图切换 | 避免重复提交、跳转至新资源 |
| 刷新行为 | 可能导致重复操作 | 安全，刷新仅重复 GET 请求 |

Spring 事务 **不支持跨线程共享**，这是因为它的事务上下文是线程绑定的，存储于 `ThreadLocal` 中，无法传递给其他线程。

### 为什么不能跨线程共享事务？

- Spring 的事务状态保存在 `TransactionSynchronizationManager` 内部的多个 `ThreadLocal` 中，只对当前线程生效。
- 一旦启用了新线程，该线程对这些 `ThreadLocal` 的值是空的，因此事务上下文不在子线程中存在。
- 如果你在事务方法中启动一个子线程并执行 DB 操作，子线程会开启独立新事务，且与主线程事务互不影响。甚至会在提交或回滚时报错，因为事务上下文不存在。

M

### 举例说明

```java
@Transactional
public void parentTx() {
    performMainOp();

    new Thread(() -> {
        // 虽在 parentTx 内，但这个操作在子线程中不共享事务
        performSubOp();
    }).start();

    // 子线程异常不会影响父线程事务
}
```

在上述代码中，`performSubOp()` 会运行在全新的事务环境中，与主事务无关联。

S

### 如何处理跨线程事务需求？

1. **避免在线程内直接开启事务**

- 如果不要求真正的事务一致性，尽量统一在单线程内处理逻辑。

2. **如果一定要多线程并发，主线程控制边界+同步等待**

- 使用 `TransactionTemplate`，主线程开启事务，分发任务到线程池，让子任务返回结果，主线程汇总后决定提交或回滚。但子线程的事务仍是独立的。

3. **跨资源或跨线程的一致性需求？用分布式事务（JTA/XA）**

- 使用 `JtaTransactionManager` 或 XA，实现 2PC 协调器共享事务上下文，可跨多个参与者，包括线程、服务、数据库。

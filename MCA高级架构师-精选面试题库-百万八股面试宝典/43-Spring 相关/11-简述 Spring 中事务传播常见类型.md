以下是七种传播类型的说明及其典型应用场景 ：

### 1. **REQUIRED**（默认）

当前已有事务时加入该事务，否则开启新事务。适用于大多数业务调用，确保在同一事务中执行。M

**示例用法：**

```java
@Transactional(propagation = Propagation.REQUIRED)
public void doBusiness() { … }
```

### 2. **REQUIRES\_NEW**

无论当前是否已有事务，都挂起现有事务，执行全新的事务。S 适用于需要独立提交或回滚的操作，如日志记录、补偿机制等，可防止外层事务失败影响内层事务。

### 3. **SUPPORTS**

当前有事务则加入，无事务则以无事务方式执行。适用于仅需支持事务但不强制要求的辅助方法。B

### 4. **NOT\_SUPPORTED**

当前有事务时挂起事务，以非事务方式执行。适用于不希望在事务中执行且避开锁、延长事务的读操作等场景。

### 5. **MANDATORY**

必须在已有事务环境中执行，若无事务则抛出异常。适用于强依赖事务环境的方法。

### 6. **NEVER**

禁止在事务中执行，若存在事务则直接抛异常。常用于明确要求无事务边界的场景。

### 7. **NESTED**

若有事务，则在当前物理连接中创建保存点；无事务则新建事务。允许内层回滚到保存点，而不影响外层。适合复杂业务中局部失败但不想整体回滚的场景。S

```java
@Transactional(propagation = Propagation.NESTED)
public void nestedPart() { … }
```

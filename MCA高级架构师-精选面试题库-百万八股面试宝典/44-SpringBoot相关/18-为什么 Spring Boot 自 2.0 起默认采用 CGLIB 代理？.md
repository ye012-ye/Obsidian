Spring Boot 在 AOP 自动配置中，会通过 `spring.aop.proxy-target-class` 属性决定代理方式。自 2.0 版本起，默认设置为 `true`，即优先使用 CGLIB 代理 M 而不是 JDK 代理。这样的设计有以下几个核心原因：M

1. **无需接口依赖**  
   CGLIB 通过生成目标类的子类进行代理，因此即使业务类未实现任何接口，也可以正常代理。而 JDK 动态代理只能对实现了接口的类生效 S。这让 Spring Boot 在多样化场景中更具适应性。
2. **一致的方法调用与注解支持**  
   如果使用 JDK 动态代理且切面注解只加在实现类上，代理基于接口的调用可能无法识别这些注解。CGLIB 直接代理类，实现注解对所有方法的一致识别，避免遗漏。
3. **更接近真实类行为**  
   CGLIB 生成的代理是目标类的子类，能够代理本类内部方法（包括继承的非接口方法），增强了功能透明度；方法调用路径也更直接，性能上略胜一筹。S
4. **依赖易用性**  
   Spring Boot 内嵌了 CGLIB 的运行时实现，开发者无需额外引入依赖库，降低了使用门槛。

综合来说，采用 CGLIB 作为默认代理机制，使 Spring Boot 在兼容性、功能性与性能上更稳定，也减少了开发者的配置负担。若仍需使用 JDK 代理，只需显式将 `spring.aop.proxy-target-class=false` 添加至配置中即可。B

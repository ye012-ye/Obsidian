雪花算法最初由 Twitter 于 2010 年开源，用于生成分布式唯一且按时间顺序递增的 64 位 ID。该 ID 通常由四部分组成：一位保留符号位（始终为 0），41 位时间戳（自定义纪元以来的毫秒数），10 位机器标识（包括数据中心 ID 与节点 ID），以及 12 位序列号，用于在同一毫秒内生成多个 ID。M

这种结构带来了多个优势。首先，它生成的 ID 在分布式环境中具有全局唯一性和高并发支持；其次，由于时间戳位于高位，ID 本身天然具备趋势递增性质，可用于高效排序；此外，生成过程完全在本地节点完成，不依赖中心服务，从而减少网络瓶颈。S

不过雪花算法对时钟同步与节点 ID 管理有较高要求。时钟倒退会导致 ID 重复或暂停服务，在扛住高并发的同时序列号位数也可能耗尽（如单节点每毫秒最多生成 4096 个 ID），需要等待下一毫秒才能继续生成 。同时，机器 ID 分配也必须避免冲突，否则可能产生重复 ID。B

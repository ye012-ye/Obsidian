在 Linux 运维中，实时掌握系统资源状态对性能监控与故障诊断至关重要。以下说明常用命令及其功能：

#### **top / htop**

`top` 是 Linux 内建的实时交互式监控工具，展示当前系统中进程的 CPU 占用率、内存使用、load average 及任务状态等信息，默认按 CPU 使用排序，便于迅速定位高资源进程。其高级变体 `htop` 提供更丰富的可视化界面、彩色条形图和进程树结构，更适合交互操作和快速筛选。S

#### **free / vmstat**

`free -h` 快速显示系统总内存、已用、可用、缓存与 swap 情况，让你立刻判断内存压力与换页行为；而 `vmstat 1 5` 可以定期输出进程、内存、swap、I/O 与 CPU 的统计数据，是分析系统整体负载趋势的轻量级工具。M

#### **iostat**

来自 sysstat 套件，`iostat -x 1` 会每秒输出磁盘设备的读写吞吐、IOPS、平均响应时间及利用率 (%util)。长时间高于 80–90% 表明可能存在 I/O 瓶颈，需要调整存储结构或分散负载。B

#### **netstat / iftop**

`netstat -tulnp` 可查看当前所有监听端口、连接状态及对应进程，适用于排查网络配置与端口冲突；`netstat -s` 显示网络协议统计信息（如丢包与错误）；而 `iftop` 可监控实时带宽使用，按连接排序显示，帮助迅速发现网络流量热点。

#### **iotop / mpstat**

`sudo iotop` 实时展示进程级的磁盘 I/O 使用情况，便于识别是哪个程序占用了大量读写资源，而 `mpstat -P ALL` 则统计所有 CPU 核心的使用率情况（包括 user / system / idle / iowait 等），帮助判断是否存在单核瓶颈或并行调度问题。

#### **ps / sar 等补充工具**

`ps -eo pid,ppid,cmd,%cpu,%mem --sort=-%cpu` 可记录某一时刻所有进程的 CPU 和内存使用情况，适合非交互式环境分析。`sar`（同样来自 sysstat）则可生成历史资源趋势报表，用于回溯分析特定时间节点的性能状态。

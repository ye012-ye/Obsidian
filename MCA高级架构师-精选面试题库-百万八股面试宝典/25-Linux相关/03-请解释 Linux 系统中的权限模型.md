Linux 文件权限系统将访问控制分为三类角色：**所属用户（user/owner）**、**所属组（group）和其他用户（others）**。每类角色都有三种基本权限标志：`r`（读）、`w`（写）、`x`（执行或进入目录）。当一个用户尝试访问文件时，系统会首先判断其是否为所属用户，若否则判断是否属于所属组，最后才考虑是否归为“others”类，权限验证按此顺序进行。

在权限位表示中，会看到类似 `-rwxr-xr--` 的串。第一个字符是文件类型（`-` 普通文件，`d` 表示目录），接着三组分别对应所属用户、所属组与其他用户的 `rwx` 权限标志。例如 `rwx`代表拥有者可读、写、执行；`r-x`代表组用户可读与执行但不能写；`r--`代表其他用户只能读取。

除了符号形式外，还可以用八进制数字表示权限：`r=4`、`w=2`、`x=1`。如权限 `754` 表示所属用户有 `7（4+2+1）` 即 `rwx` 权限，所属组有 `5（4+1）` 即 `r-x`，其他用户为 `4（4+0+0）` 即 `r--`。

#### 示例：

- **设置权限**  
  使用 `chmod` 命令。例如将 `file1.txt` 设置为所有者 `rwx`，所属组 `r-x`，其他用户仅 `r--`：

```bash
chmod 754 file1.txt
```

或使用符号模式：

```bash
chmod u=rwx,g=rx,o=r file1.txt
```

- **修改所属用户与组**  
  使用 `chown` 更改文件所有者与组。

```bash
sudo chown newuser:newgroup file1.txt
```

若只改所有者，可省略冒号后部分；仅改组则写 `:newgroup`。

- **设置特殊位**（可选项）  
  可使用 `setuid`、`setgid` 或 `sticky` 位实现特殊权限控制。例如设置目录 `/tmp/shared` 的 sticky 位：

```bash
chmod 1777 /tmp/shared
```

此时只有文件所属者、目录所属者或 `root` 才能删除或重命名其中内容，增强共享目录安全性。

Java 的传统 I/O（如 `java.io.InputStream` 与 `OutputStream`）是基于字节流的一次性、阻塞式读取或写入。线程发起读取或写入请求后会被挂起，直到操作完成，处理完才能继续下一步。M

而 Java NIO（New I/O）则引入了更加灵活高效的机制，区别主要体现在三个方向：

**1. 面向 Buffer 而非 Stream**  
传统 I/O 一次操作少量字节，并逐字节处理，不支持随机访问。而 NIO 使用 `Buffer` 作为数据载体，先一次性将数据读入 `Buffer`，再在内存块中对数据进行随机访问或重读，使处理更高效。

**2. 支持非阻塞 IO**  
NIO 的通道（`Channel`）可设置为非阻塞模式。线程向通道发起读写请求后可立即返回，无需等待数据准备，就可以处理其他逻辑。这意味着单线程可以管理多个未准备好的通道，提升并发性能。

**3. 引入 Selector，可管理多个通道**  
`Selector` 是 NIO 的核心，它允许一个线程监听多个通道的事件，比如数据是否准备好读取或通道是否可以写入。相比创建多个线程去等待 I/O 就绪，使用 `Selector` 能显著降低线程开销并提高扩展性。

### NIO 的核心组件

- **Buffer**：用于临时存放数据的容器，程序从通道读入内容或写出内容需先将数据放入 `Buffer`，提供诸如 `flip()`、`clear()` 等方法支持随机读取与写入。
- **Channel**：与 data source／sink（如文件、网络连接）连接的通道，支持读取至 Buffer 或从 Buffer 写入。代表类包括 `FileChannel`、`SocketChannel`、`ServerSocketChannel` 等。
- **Selector**：事件多路复用器，能监听多个通道，并在某个通道准备就绪后通知应用程序，可用来替代多线程轮询方式管理大量连接。
- **SelectionKey**：注册通道到 `Selector` 后会生成对应的 `SelectionKey`，它记录兴趣事件集合以及关联的通道。
- **Charset 编解码支持**：NIO 提供 Charset 编码器/解码器，方便在字符与字节之间转换，减少编码转换的繁琐。

此外，NIO.2（Java 7 引入）进一步增强了文件处理能力，新增 `java.nio.file` 包支持更强大的文件路径、文件属性、锁、记忆映射及异步文件 I/O。

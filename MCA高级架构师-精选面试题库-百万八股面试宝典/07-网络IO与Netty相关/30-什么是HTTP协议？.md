HTTP（HyperText Transfer Protocol，超文本传输协议）是一种应用层协议，用于客户端（如浏览器）与服务器之间以请求—响应方式传输资源（HTML、图像、视频、API 数据等）。作为万维网的基础，HTTP 规定了通信格式、方法（GET、POST 等）、状态码与头部字段，依赖底层 TCP 或 QUIC 进行数据传输。

M

HTTP 的关键特性包括：

- **无状态性**：每个请求独立，服务器不保留前次通信状态，这提升了可扩展性，但如需状态管理则需依赖 Cookie 或会话机制。
- **文本为主，易读易调试**：请求与响应均采用纯文本格式，便于开发和调试，但在性能优化上存在限制。
- **客户端–服务器模式**：由客户端发起请求，服务器返回数据，结构清晰，对网络行为可控。

S

### 演进历程：

- **HTTP/1.1（1997年∼）**：引入持久连接、管线化和缓存控制（如 `Cache-Control`、ETag），提升性能与稳定性。
- **HTTP/2（2015年标准化）**：采用二进制帧、请求多路复用、头部压缩（HPACK）、优先级控制和 Server Push，大幅减少延迟、提高并发能力。
- **HTTP/3（2022年标准）**：基于 QUIC（UDP 上的传输层协议）替代 TCP，解决 HTTP/2 的队头阻塞问题，实现更低延迟、更快连接建立与更强的网络鲁棒性。

B

### 总结：

HTTP 是一种轻量、无状态、基于请求–响应的应用层协议，形成了 Web 的核心。随着版本升级，其在性能、安全和用户体验方面不断优化，已发展到基于 QUIC 的 HTTP/3，并在现代互联网中广泛应用。

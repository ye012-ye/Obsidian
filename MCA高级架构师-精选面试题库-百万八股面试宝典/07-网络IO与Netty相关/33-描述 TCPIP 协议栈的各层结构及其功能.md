TCP/IP 协议栈由 **五层结构** 构成（也常合并为四层），每层承担特定职责，协同实现网络通信功能。

M

#### 1. **网络访问层（Network Access / Link Layer）**

- 负责设备之间在同一局域网（LAN）上的实际数据传输，涵盖物理层和数据链路层功能。
- 包括硬件介质（如网线、光纤、Wi‑Fi）、交换机、网卡等；通过 MAC 地址形成帧结构，检测和纠正物理层错误，决定介质访问控制方式（如 CSMA/CD）。

#### 2. **互联网层（Internet Layer）**

- 基于 IP 协议，实现不同网络之间的数据传递与路由选择；处理地址分配、分片与重组。
- 该层核心协议为 IPv4/IPv6，支持 ARP、ICMP 等辅助功能，确保数据包按照地址寻路到目标主机。

S

#### 3. **传输层（Transport Layer）**

- 面向端到端的数据传输，提供可靠或不可靠传输方式：

- **TCP**：连接导向，提供三次握手、重传、序列号、流量控制和拥塞控制，适用于网页加载、文件传输等高可靠性场景。
- **UDP**：无连接，轻量高效，适合实时视频、语音、DNS 查询等场景。

B

#### 4. **应用层（Application Layer）**

- 最贴近用户的一层，提供各种应用协议，例如 HTTP、FTP、SMTP、DNS、SSH 等。
- 该层可能融合传统 OSI 模型的会话、表示、应用三层，负责具体业务数据的格式化、加密与展示。

### 四层 vs. 五层对比：

|  |  |
| --- | --- |
| **模型版本** | **层次划分** |
| **五层模型** | 应用 → 传输 → 网络 → 数据链路 → 物理 |
| **简化四层** | 应用 → 传输 → 互联网（网络） → 网络访问（含物理+链路） |

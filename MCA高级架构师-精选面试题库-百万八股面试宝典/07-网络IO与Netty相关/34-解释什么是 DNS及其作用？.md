**DNS（Domain Name System）** 是互联网的分布式“电话簿”，它将人类可读的域名（如 `www.example.com`）转换为计算机使用的 IP 地址（如 `192.168.1.1`），以便设备能定位目标服务器并进行通信 。

M

#### 一、DNS 解析的核心流程：

1. **DNS 查询发起**  
   当用户在浏览器输入域名时，系统首先会检查本地缓存；若未命中，则向本地递归解析器（通常是 ISP 提供）发起请求。
2. **递归解析与缓存**  
   解析器若未缓存该域名，便向根服务器、TLD（顶级域名）服务器、再到权威服务器逐层查询，直至获取最终 IP，再回传给用户同时缓存。
3. **分层解析结构**

- 根服务器负责顶级域名 (.com/.org 等) 的定位；
- 接着 TLD 服务器指向特定域名的权威名称服务器；
- 最终权威服务器返回实际 IP 地址。

4. **DNS 缓存机制**  
   递归解析器和客户端都会缓存解析结果以减少查询次数、提升性能，缓存内容根据 TTL 生效期动态更新。

S

#### 二、DNS 的主要作用：

- **域名解析**：将易记的域名转换为数字 IP，便于访问和通信 。
- **分布式与高可用**：采用全局分布式结构和层级解析机制，保障解析效率与可靠性 。
- **反向解析与多记录支持**：支持 PTR / MX / CNAME / TXT 等多种记录类型，实现邮件路由、别名管理、安全验证等 。
- **智能负载与就近优化**：通过为不同用户返回不同 IP 地址，为 CDN 提供地域定位支撑，提升访问速度 。

B

### 总结：DNS 是互联网通信的基础设施，其核心功能是将域名解析为 IP 地址，确保用户无需记忆复杂数字即可访问网络资源。它通过分布式、层级化结构和缓存机制，实现了高效、可靠、灵活的域名解析服务，是现代网络运行不可或缺的关键组件。

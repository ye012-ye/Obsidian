零拷贝是一种通过消除或减少数据在传输过程中的多余内存复制来提高系统性能的技术。在传统模型中，数据从源设备（磁盘或网络）到目标设备（应用或另一网络）通常要经过多次复制，消耗大量 CPU 和内存带宽。M

通过零拷贝，这些重复复制可被省略或替代为更高效的方法，从而显著降低延迟和 CPU 占用。以下是关键手段：

1. **DMA（直接存储器访问）**  
   使用 DMA 控制器将设备上的数据直接传送到物理内存中，绕过 CPU 干预。这样，在磁盘读或网卡收数据时，减少了 CPU 参与，仅在传输完成时才触发中断，节省资源。S
2. **内存映射（**`mmap()`**）**  
   内存映射允许应用进程将文件或设备缓冲区映射到自己的虚拟地址空间，使用户空间直接读写共享的物理页，无需先复制到用户缓冲区。这样至少可消除一次复制。
3. **系统调用级别优化（如** `sendfile()`**、**`splice()`**、**`vmsplice()`**、**`sendmsg()` **+** `MSG_ZEROCOPY`**）**  
   这些接口允许内核直接在源和目标缓冲区之间进行数据传递，省略用户态到内核态的拷贝。例如，`sendfile()` 可以在内核中把磁盘缓存内容直接传输给网络套接字：不仅减少了复制，还减少了用户/内核切换次数。
4. **内核态/用户态零拷贝接口**  
   如零拷贝 TCP（Linux 上 `MSG_ZEROCOPY`）允许用户空间的数据缓冲直接被 NIC 使用 DMA 提取，省去内核中转缓冲。这样将数据从用户缓冲直接传入网卡。

#### 作用与收益

- **CPU 使用率降低**：消除了多余的数据复制路径，将 CPU 从繁琐的拷贝任务中解放；S如 Wikipedia 所说，零拷贝能“最小化 CPU 使用和内存带宽使用” 。
- **上下文切换减少**：省略了拷贝，加速了跨用户／内核态切换，显著减少上下文切换次数。
- **延迟降低、吞吐提升**：特别在高速网络（10+ GbE）和大文件传输场景，零拷贝避免复制瓶颈，从而提升整体系统吞吐率。 B
- **零拷贝依赖硬件支撑**：DMA、MMU、支持零拷贝的 syscalls 以及 RDMA/NIC offload 等硬件基础是其前提，若这些缺失则无法实现全面零拷贝。

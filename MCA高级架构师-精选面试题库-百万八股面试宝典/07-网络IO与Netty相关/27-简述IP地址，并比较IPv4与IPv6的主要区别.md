IP 地址是网络上设备的唯一标识，相当于邮寄系统中的“门牌号”，用于支持数据在发送端与接收端之间进行正确路由和定位。M

**1. IPv4 的定义与特点**

IPv4 是 Internet Protocol version 4 的缩写，它使用 32 位二进制地址格式，通常以四段十进制数字（范围为 0–255）表示，如 `192.168.0.1`。这一结构允许约 43 亿个唯一地址。IPv4 支持广播与子网划分机制（CIDR），但地址数量有限，随着互联网普及，地址资源逐渐枯竭。B

**2. IPv6 的定义与改进**

IPv6 引入于 1995 年，是对 IPv4 的重构。它使用 128 位地址，表示方式为八段十六进制数，用冒号分隔，例如 `2001:0db8::8a2e:0370:7334`。庞大的地址空间（约 3.4×10³⁸ 个地址）基本消除了地址耗尽的问题，便于大规模部署物联网与云计算场景。

**3. IPv4 与 IPv6 的关键区别对比**

- **地址长度**：IPv4 为 32 位，IPv6 为 128 位，后者地址空间约为前者的几千万亿倍。
- **表示格式**：IPv4 使用点分十进制（例如 `192.168.1.1`），IPv6 使用冒号分隔的十六进制格式（例如 `2001:db8::1`）。
- **地址资源量**：IPv4 最多约 4.29×10⁹ 个地址；IPv6 拥有约 3.4×10³⁸ 个地址，几乎不可能耗尽。
- **内置功能**：IPv6 将安全性机制（如 IPsec）和自动配置（Stateless Address Autoconfiguration）标准化，而 IPv4 需手动配置或通过 DHCP 支持相应功能。
- **头部结构优化**：IPv6 的报文头部结构更精简、固定长度（40 字节），去除部分可选字段，便于路由器快速处理。
- **互操作性与迁移路径**：由于格式差异，IPv4 与 IPv6 不能直接互通。部署时常采用双栈技术或者翻译机制（如 NAT64/DNS64）实现兼容。

**1. URI（Uniform Resource Identifier）**  
URI 是一个通用的资源标识符，用于在网络或系统中唯一标识任何资源。它可以标识资源的名称（URN）、位置（URL），或者两者兼具。

**2. URL（Uniform Resource Locator）**  
URL 是 URI 的一个子集，指明资源具体位置及访问方式。它由协议（如 HTTP/HTTPS）、主机、端口、路径、查询字符串等部分组成，例如 `https://example.com/index.html`。其作用是“告诉你去哪拿资源”。

**3. URN（Uniform Resource Name）**  
URN 也是 URI 的子集，用于为资源提供持久、全局唯一的命名，它不包含访问路径。例如 `urn:isbn:0451450523`，它命名了某本书，但并不指示如何获取这本书。

#### ​核心区别对照：

|  |  |  |
| --- | --- | --- |
| 术语 | 所属关系 | 功能 |
| URI | 最上层 | 标识资源，不涉及获取方式 |
| URL | URI 的子集 | 标识资源并指明其位置与访问方式 |
| URN | URI 的子集 | 为资源提供一个永久唯一的名称，不指示位置 |

- **URL 是 URI，却不是所有 URI 都是 URL**。
- URN 不用于定位资源，仅用于命名，例如图书 ISBN、数字对象标识符等。

#### ​总结

- 遇到如 `http://…`、`ftp://…` 的字符串，它既识别又指明访问方式，它是 **URL（也是 URI）**。
- 遇到如 `urn:isbn:…` 这样的字符串，它只有标识意义，它是 **URN（也是 URI）**。
- 当你听到“URI”时，把它看作一个“资源标识”的总体概念：它可以是 URL，也可以是 URN，甚至两者兼具。

CORS 全称是 **跨域资源共享**（Cross-Origin Resource Sharing），它是一种基于 HTTP 头的机制，用于允许浏览器从不同域（协议、域名、端口三者组合）安全地请求资源。浏览器出于安全会默认拒绝这种跨域请求，因此必须通过 CORS 来明确授权，以便访问资源。

M

在实际 HTTP 交互中，跨域过程如下：前端 JavaScript 会自动带上 `Origin` 请求头；服务器检查该来源是否在允许名单里，并通过在响应头中加入 `Access-Control-Allow-Origin`（比如具体域名或 `*`），以及其他控制，如共享方法、头部或凭据等，来告知浏览器是否可以允许请求。

若请求属于复杂类型（如使用 PUT、DELETE、上传 JSON、自定义头等），浏览器还会先发起一个 OPTION 预检请求，确认服务器支持后才继续发起实际请求，这进一步提升了安全性。

S

CORS 对 Web 开发格外重要，原因包括：

在前后端分离或微服务架构中，前端应用常运行于一个域，调用后端 API 则在另一个域。若无 CORS，会被浏览器阻止，从而影响功能实现。CORS 是支持现代 Web 架构的基础设施。

它帮助加载来自 CDN、第三方服务（例如字体、图片、脚本）的资源，同时保持安全性。恰当配置可提升资源性能，而错误配置可能导致安全漏洞或失败。

CORS 是浏览器层面阻止潜在跨站脚本攻击（例如 CSRF）的关键安全屏障。仅授权可信源访问敏感资源，可避免恶意站点盗用登录凭证或执行脚本。

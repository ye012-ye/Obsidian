HTTPS（HyperText Transfer Protocol Secure）是在 HTTP 的基础上，增加了加密与身份验证机制的安全协议。它通过 SSL/TLS 层，实现客户端和服务器之间数据传输的私密性与完整性。M

#### 一、HTTPS 的核心流程

1. **SSL/TLS 握手阶段**  
   客户端发起连接请求，服务器返回证书，客户端验证证书合法性后，双方协商加密算法与会话密钥。此过程采用非对称加密交换密钥，再以对称加密进行后续通信，兼顾安全与性能。
2. **加密通信阶段**  
   握手完成后，后续 HTTP 报文（请求与响应）均通过 TLS 通道加密，只有通信双方能解密。

#### 二、HTTPS 与 HTTP 的主要差异

|  |  |  |
| --- | --- | --- |
| **特性** | **HTTP** | **HTTPS** |
| 安全性 | 明文传输，容易被监听与篡改 | 使用 TLS 加密，防止窃听与中间人攻击 |
| 端口 | 默认 80 | 默认 443 |
| 证书验证 | 无 | 需安装由 CA 签发的 SSL/TLS 证书 |
| 浏览器提示 | 显示 “不安全” | 显示“锁”图标与 HTTPS 前缀 |
| 性能 | 无加密开销，略快 | 握手有开销，但现代 TLS+HTTP/2 性能几乎相当，且支持多路复用 |
| SEO & 信任 | 不利于搜索排名，削弱用户信任 | Google 倾向 HTTPS，提升排名与用户信任度 |

总结： HTTPS 相比 HTTP，增加了 **加密（Encryption）**、**认证（Authentication）**、**完整性保护（Integrity）** 三大安全机制，使数据传输更可信、安全。今天，无论是电商支付、个人信息提交，还是普通网站浏览，HTTPS 已是网络通信的标准选择。

即使在 SQL 中使用了索引，如果查询仍然很慢，常见原因如下：

#### 1. 索引未被正确匹配使用

- 查询条件顺序不对或使用了函数/表达式（如 `DATE()`、`LOWER()`）会让索引失效。
- 组合索引若使用顺序不符合创建时的顺序，也无法命中。M
- 使用 EXPLAIN 后若看到 `ALL` 或 `index` 类型，通常说明索引没被实际利用。

#### 2. 索引选择性差

- 返回数据行太多，导致使用索引比全表扫描更慢 。
- 可以通过建高选择性索引或重建索引提升效率。

#### 3. 覆盖索引缺失

- 未使用覆盖索引，查询仍需回表，增加磁盘 I/O 开销。S
- 建议包含查询中所有字段的组合索引，实现 Index Only Scan，避免回表。

#### 4. 排序或聚合未覆盖

- ORDER BY、GROUP BY 没有索引支持，会生成临时表并进行 filesort 操作，严重拖慢查询。
- 通过在组合索引尾部加入排序列可改善这一问题。

#### 5. 统计信息过期或不准确

- 优化器基于统计信息生成执行计划，若统计信息陈旧，可能选择错误路径。
- 可通过 `ANALYZE TABLE` 强制刷新统计。

#### 6. 索引维护与硬件限制

- 数据写入频繁时，索引维护开销高；过多索引会拖慢写性能。B
- I/O 瓶颈、磁盘故障、内存不足等硬件问题也会影响查询速度 。

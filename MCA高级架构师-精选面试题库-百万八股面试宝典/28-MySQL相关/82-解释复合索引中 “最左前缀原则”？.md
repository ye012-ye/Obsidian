**最左前缀原则** 指的是：*多列索引只有从最左侧列开始、连续匹配时才可被优化器利用*。

- 对索引 `(A, B, C)` 而言，只有以下查询能完全利用索引：

- `WHERE A = ?`
- `WHERE A = ? AND B = ?`
- `WHERE A = ? AND B = ? AND C = ?`  
  略写或顺序调换（如 `B = ? AND A = ?`）仍可使用前缀。

- 若跳过中间列或仅匹配 `B` 或 `C`，优化器将放弃该索引，改为全表扫描或使用其他索引。
- **范围条件限制**：当遇到范围操作（`>`、`<`）时，只能匹配到该列（如 `B > ?` 后 `C` 不再使用索引）

​

为了最大化复合索引的效果，可以做如下操作：

- 将最常做过滤/JOIN 的列放在最左侧；
- 遵循索引列排序高选择性到低选择性顺序；
- 常用 `EXPLAIN`，检查是否命中左前缀；
- 若需支持不从左边开始的查询，可建立额外单列索引。

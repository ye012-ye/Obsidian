使用 `WHERE 1=1` 是一种动态拼条件的便捷写法，可避免拼接时遗漏 `WHERE` 或 `AND`。不过，人们担心它是否会对性能有副作用。下面从优化器角度分析：M

1. **恒等式会被优化器移除**  
   在 MySQL 5.7 及以上版本，查询优化器会在解析阶段进行“常量折叠”（constant folding），识别出 `1=1` 永真表达式并自动去除，不会影响执行计划。  
   S
2. **无需担心性能开销**  
   多数数据库（包括 MySQL、SQL Server）会在编译时优化掉恒等条件，不会在运行时产生额外代价，因此对查询性能几乎没有影响 。  
   B
3. **何时可优化代码**

- 如果系统使用 MyBatis 这种框架，可以通过 `<where>` 动态标签拼接条件，避免显式写 `1=1`，代码更优雅。
- 在 MySQL 5.7 以下版本，优化能力稍弱，使用 `<where>` 标签仍有助于简化逻辑拼接。

4. **总结建议**：

- **MySQL ≥5.7**：`WHERE 1=1` 由优化器自动移除，不影响性能，可继续使用或改为 `<where>` 标签。
- **MySQL < 5.7**：建议使用 MyBatis `<where>` 或手动拼接条件。
- **其他 DB**：大多数也具备类似常量折叠机制，对性能影响微乎其微。

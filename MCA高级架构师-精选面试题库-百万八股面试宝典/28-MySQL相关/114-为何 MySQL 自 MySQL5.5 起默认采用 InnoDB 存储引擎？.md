MySQL 从 5.5.5（约 2010 年）开始，将默认存储引擎设为 InnoDB，其核心理由是**在可靠性、一致性与高并发性能之间找到理想平衡**。M

#### 1. 完整的事务支持（ACID+崩溃恢复）

InnoDB 完全符合 ACID 属性，支持事务提交、回滚和自动崩溃恢复，即使系统异常重启，也能恢复未提交的事务，确保数据一致性。

#### 2. 行级锁与 MVCC 高并发

支持行级锁（而非 MyISAM 的表级锁），避免写操作干扰其他行，加上 MVCC，多用户并发读写性能显著提升。

#### 3. 聚簇索引与缓冲池优化

S数据按主键组织，查询效率高。InnoDB 的 Buffer Pool 缓存表和索引数据，减少磁盘 I/O，对大数据量环境尤其重要。

#### 4. 支持外键约束

具备 referential integrity 功能，确保表关联的数据一致性，便于设计事务性关系型系统 。

#### 5. 丰富功能：压缩、加密、在线 DDL

支持数据压缩、加密、在线结构修改等特性，适应现代应用需求 。B

#### 6. 实践证明稳定可靠

大量生产环境（如电商、金融、OLTP 系统）验证其性能与稳定性优异，对大多数 Web 应用都是首选。

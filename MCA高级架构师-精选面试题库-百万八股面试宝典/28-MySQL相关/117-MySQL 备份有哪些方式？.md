MySQL 数据库备份方式可从三个维度划分，以下是条理清晰的分类及常用工具总结：

---

### 一、按数据库运行状态划分M

- **热备（Hot Backup）**：数据库在线读写不中断。适用于对服务可用性要求高的场景。Percona XtraBackup 外 + `--single-transaction` 形式的 mysqldump 可实现此类备份。
- **温备（Warm Backup）**：仅支持读操作，写被阻塞，通常通过 `FLUSH TABLES WITH READ LOCK` 实现一致性。
- **冷备（Cold Backup）**：需停库再备，简单直接，但会导致服务中断。

---

### 二、按备份层面划分

- **逻辑备份（Logical Backup）**：通过 SQL 脚本导出建表和数据，如 `mysqldump`、`SELECT ... INTO OUTFILE`。优点：跨版本、可读可修改；缺点：备份/恢复慢，某些数据可能丢失精度。
- **物理备份（Physical Backup）**：复制底层数据文件，如 InnoDB 表空间、日志等，使用工具如 XtraBackup。这类备份速度快，恢复迅速但不可读、依赖引擎。

---

### 三、S按完整程度划分

- **完全备份（Full Backup）**：备份完整数据，恢复简单但成本高。
- **差异备份（Differential Backup）**：备份自上次全备以来发生变化的数据，恢复效率较高。
- **增量备份（Incremental Backup）**：备份自上次任意备份以来变化部分，节省空间但恢复复杂。

---

### 四、常用工具对比

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| 工具 | 类型 | 热/温/冷 | 优点 | 适用场景 |
| `mysqldump` | 逻辑备份 | 热/温（加选项） | 易读、易编辑、跨存储引擎兼容 | 小中库逻辑导出 |
| `XtraBackup` | 物理备份 | 热 | 备份快、支持增量、恢复迅速 | 大库高可用备份 |
| `cp/tar` | 物理备份 | 冷 | 简单高效，不依赖工具 | 小库或测试备份 |
| LVM snapshot | 物理快照 | 接近热 | 零停机、高性能 | 文件系统支持快照 |
| `mysqlhotcopy` | 逻辑备份 | 冷 | 简单实用但仅支持 MyISAM | 小规模 MyISAM |

---

### 五、典型备份策略B

- 小库、逻辑备份：使用 `mysqldump` 进行全量+增量备份。
- 大库、大表、在线需求：使用 `XtraBackup` 做全量 + 定期增量备份。
- 业务可停机窗口：配合 `cp/tar` 或 LVM 快照实现低成本备份。

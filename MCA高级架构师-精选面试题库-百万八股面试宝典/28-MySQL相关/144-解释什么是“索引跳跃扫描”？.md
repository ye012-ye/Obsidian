在 MySQL 8.0.13 版本中，优化器引入了“索引跳跃扫描”（Skip Scan）策略，旨在提升在特定条件下对复合索引的利用效率。尽管 MySQL 官方文档中未明确标注为“Skip Scan”，但该功能的实现方式与“Loose Index Scan”类似，主要用于多列索引的查询优化。

### Skip Scan 的工作原理

当查询条件未覆盖复合索引的最左前缀列时，传统的索引扫描可能无法有效利用索引，导致性能下降。Skip Scan 通过以下方式优化查询：

1. **按复合索引的前缀列进行分区**：将索引数据按前缀列的不同值进行分区，例如，如果索引为 `(a, b, c)`，则按 `a` 的不同值进行分区。
2. **对子查询条件进行范围扫描**：在每个分区中，对剩余列的条件进行范围扫描，例如，在 `a = 1` 的分区中，执行 `SELECT * FROM t WHERE b = 2 AND c > 10`。
3. **跳过不相关的分区**：仅扫描满足条件的分区，跳过其他不相关的分区，从而减少扫描的数据量。

### 使用场景与示例

假设有一个包含字段 `a`、`b` 和 `c` 的表 `msb_t`，并创建了复合索引 `(a, b, c)`。如果执行以下查询：

```sql
SELECT * FROM msb_t WHERE b = 2 AND c > 10;
```

由于查询条件未覆盖索引的最左前缀列 `a`，传统的索引扫描可能无法有效利用索引。此时，优化器可能采用 Skip Scan 策略：

- **步骤 1**：扫描索引中 `a` 的不同值，例如 `a = 1`、`a = 2` 等。
- **步骤 2**：在每个 `a` 的分区中，执行范围扫描，例如在 `a = 1` 的分区中，执行 `SELECT * FROM t WHERE b = 2 AND c > 10`。

这种方式避免了全表扫描，提高了查询效率。

### 使用 `EXPLAIN` 查看执行计划

可以通过 `EXPLAIN` 语句查看是否使用了 Skip Scan 策略：

```sql
EXPLAIN SELECT * FROM msb_t WHERE b = 2 AND c > 10;
```

在 `Extra` 列中，如果显示 `Using index for skip scan`，则表示使用了 Skip Scan 策略。

### 控制 Skip Scan 的启用与禁用

可以通过设置 `optimizer_switch` 系统变量来控制 Skip Scan 的启用与禁用：

- **启用 Skip Scan**：

```sql
SET optimizer_switch = 'skip_scan=on';
```

- **禁用 Skip Scan**：

```sql
SET optimizer_switch = 'skip_scan=off';
```

此外，还可以在查询中使用优化器提示来强制使用或禁止 Skip Scan：

- **强制使用 Skip Scan**：

```sql
SELECT /*+ SKIP_SCAN(t) */ * FROM msb_t WHERE b = 2 AND c > 10;
```

- **禁止使用 Skip Scan**：

```sql
SELECT /*+ NO_SKIP_SCAN(t) */ * FROM msb_t WHERE b = 2 AND c > 10;
```

### 注意事项

- **适用条件**：Skip Scan 适用于多列索引，且查询条件未覆盖最左前缀列。
- **性能影响**：在数据分布均匀且前缀列基数较低的情况下，Skip Scan 可以显著提高查询性能。
- **版本要求**：Skip Scan 功能自 MySQL 8.0.13 版本开始支持。

通过合理利用 Skip Scan 策略，可以在特定查询场景下提高 MySQL 的查询性能，减少不必要的数据扫描。

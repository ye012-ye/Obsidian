虽说低选择性（low selectivity）的字段常被建议慎用索引，但在特定条件下，它们仍可能发挥作用。以下是精炼、贴合场景的几种情况：

### 1. 极端少见值场景

当某个值在低区分度字段中极其罕见时，索引索引能显著提速查询。  
**示例**：`gender` 字段，仅极少部分为 `'Other'` 部分查询：

```sql
SELECT * FROM people WHERE gender = 'Other';
```

此时在 `gender` 上建立索引，可快速定位少量行，避免全表扫查。M

---

### 2. 联合索引中作为覆盖列

当低区分度字段与高选择性字段组合并构成覆盖索引时，可优化查询，避免回表。  
**示例**：

```sql
CREATE INDEX idx_status_created ON orders(status, created_at);
SELECT user_id, status FROM orders
 WHERE status = 'PENDING' AND created_at > '2025-01-01';
```

该索引既过滤了状态，也可以覆盖查询列，提升效率。S

---

### 3. 排序 + LIMIT 场景

对带有低区分度字段的排序查询，加上合适索引也可提速。  
**示例**：

```sql
SELECT * FROM items ORDER BY featured_flag, id LIMIT 1;
```

若创建复合索引 `(featured_flag, id)`，可避免全表排序。B

---

### 4. 数据仓库 & 位图索引适用环境

在只读、报表型系统中，低区分度常被用于位图索引。此类索引对多字段组合过滤效果好，维护开销低。尤其在数据仓库场合非常适用。

B 树与 B+ 树是数据库系统中广泛应用的多路平衡查找树结构，虽然两者都用于加速磁盘中的数据访问，但它们在节点内容、查找机制以及应用特性上有明显差别：

M

#### 1. **节点数据存储差异**

- **B 树：** 所有节点（无论是否为叶子节点）都可以存储键值对，即键和对应的数据内容。这样，在查找过程中可能在非叶子节点上就直接命中目标值。
- **B+ 树：** 非叶子节点只存储索引键，不存储实际数据；只有叶子节点才保存完整的键值记录。这意味着所有有效数据都集中在叶子层。

**这种设计导致 B+ 树的非叶子节点更“轻”，可以容纳更多索引键，从而降低树的高度，间接提高检索效率。**

#### 2. **查找路径不同**

- 在 **B 树** 中，目标数据可能存在于任意层级的节点上，因此查找路径可能在中间层级就终止。
- 在 **B+ 树** 中，所有查询操作都会落到叶子节点，即便找到了索引键，也必须继续深入到叶子层才能拿到对应数据。

S

#### 3. **范围访问与顺序读取效率**

B+ 树的叶子节点通常通过**链指针**连接成一个有序链表，便于在不回溯上层的前提下，顺序遍历连续数据。因此它在执行范围检索（如 `BETWEEN`、`>=`）时，效率明显优于 B 树。相比之下，B 树缺乏这种链式结构，范围访问通常需要多次树层回溯。

B

#### 4. **在数据库中的使用偏好**

由于数据集中在叶子层、支持顺序遍历且更易于控制树高，**B+ 树通常是数据库索引的首选结构**，尤其在 OLAP、批量查询、范围检索等场景下具有明显优势。而 B 树虽然理论上支持更快的点查找，但在实际数据库系统中较少使用。

​

两者对比如下：

|  |  |  |
| --- | --- | --- |
| 比较维度 | B 树 | B+ 树 |
| **节点存储** | 所有节点存储数据 | 仅叶子节点存储数据 |
| **查找路径** | 可在中间节点命中数据 | 所有查找必须到叶子节点 |
| **范围检索** | 效率一般，无链表支持 | 高效，叶子节点有序链表 |
| **应用场景** | 更适合随机访问（理论上） | 更适合范围查询与顺序遍历 |

B 树结构更适用于小型、随机数据访问需求，而 B+ 树在数据库等大规模存储系统中，更能兼顾效率与结构优化，因此被广泛采用作为底层索引机制。

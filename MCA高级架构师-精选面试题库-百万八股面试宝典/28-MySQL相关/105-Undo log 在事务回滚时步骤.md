在 InnoDB 中，Undo log（撤销日志）通过存储数据操作的“前镜像”执行事务回滚，确保原子性与一致性，具体流程如下：M

1. **修改前写入旧值**  
   每当事务执行 `INSERT`、`UPDATE` 或 `DELETE` 操作时，系统会将操作前的数据状态存入 Undo log 中，此机制既支持 MVCC，又为回滚做准备。S
2. **操作期间维护 Undo 链**  
   Undo records 形成链表，每条记录都指向前一个版本（通过 `ROLL_PTR`），构成历史版本路径，便于事务按需回退或读取一致快照。
3. **事务回滚时逆序执行 Undo**  
   在回滚阶段（用户 `ROLLBACK`、死锁处理或崩溃恢复），系统从最新 Undo 记录开始，逆序遍历，依次执行反向操作：

- 对应 `INSERT` 的 Undo 会删除该行
- 对应 `UPDATE` 的 Undo 会恢复旧值
- 对应 `DELETE` 的 Undo 会重新插入旧行

4. **只回滚当前事务内容**  
   回滚过程仅针对该事务的 Undo log 记录，跳过其他事务提交的数据，防止影响其他事务 。B
5. **清理 Undo 记录**  
   回滚完成后，相应的 Undo log 被标记为“已使用”，等待后台 purge 线程清除并释放空间

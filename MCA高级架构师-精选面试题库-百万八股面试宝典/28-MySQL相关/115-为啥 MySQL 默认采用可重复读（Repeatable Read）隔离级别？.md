MySQL 默认将 InnoDB 的隔离级别设置为 Repeatable Read，是因为它在性能和事务一致性之间提供了良好的权衡。M

---

### 核心原因

1. **MVCC 保证事务内视图一致**  
   每个事务使用一个快照（snapshot）首次读取数据，之后所有读取都基于同一快照呈现，避免了“不可重复读”问题。S

​

2. **Next‑Key（Gap）锁解决幻读**
3. **MVCC 加锁策略提升并发性**

普通 SELECT 使用 MVCC 快照，不加锁；而涉及修改时，仅对目标行及相邻间隙加锁，避免不必要阻塞。

​

4. **性能胜过可串行化（SERIALIZABLE）**

Serializable 虽能提供更强一致性，但限制并发性能显著。相比之下，RR 既能避免脏读、不规范读和幻读， 又支持读写并发，性能更优。

​

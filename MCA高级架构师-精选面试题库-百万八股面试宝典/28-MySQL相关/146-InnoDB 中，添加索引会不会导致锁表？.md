在 InnoDB 存储引擎中，是否会锁表取决于所使用的 MySQL 版本、操作类型以及配置参数。

### 1. MySQL 5.6 之前的版本

在 MySQL 5.6 之前，使用 `ALTER TABLE` 添加索引时，通常会对整个表加锁，导致在索引创建过程中，其他查询和数据操作（如 `INSERT`、`UPDATE`、`DELETE`）无法进行。这种锁定方式可能对生产环境中的高并发应用造成影响。M

### 2. MySQL 5.6.7 引入的 Online DDL

从 MySQL 5.6.7 开始，引入了 Online DDL（在线数据定义语言）功能，允许在添加索引时，表仍然可以进行读取和写入操作，从而减少对业务的影响。

- **ALGORITHM=INPLACE**：在原地修改表结构，避免创建临时表，从而提高性能。
- **LOCK=NONE**：在执行 DDL 操作时，不对表加锁，允许所有的 DML 操作并发执行。

然而，需要注意的是，即使使用 Online DDL，仍可能会对表的某些部分进行短暂的锁定，特别是在高并发情况下，可能会出现元数据锁（Metadata Lock，简称 MDL）等待现象，导致其他操作被阻塞。S

### 3. MySQL 8.0 及更高版本的改进

在 MySQL 8.0 及更高版本中，Online DDL 功能得到了进一步增强，支持更多类型的结构变更操作，且对性能的影响进一步降低。B

数据库执行 SQL 的流程可以分为服务层（Server Layer）与存储引擎层（Storage Engine Layer）两大块，整体步骤如下：

#### 1. **连接与接收（Connector）**

客户端通过连接器发起 TCP 连接、验证权限，然后将 SQL 语句发送到服务。M

#### 2. **查询缓存（Query Cache，旧版 MySQL）**

若开启且执行的是 SELECT 语句，MySQL 会检查查询缓存；命中时直接返回缓存结果并终结，不进入后续阶段。

#### 3. **解析与预处理（Parser + Preprocessor）**

一次或“硬解析”：

- **语法分析**：确保语句结构合法，并生成语法树。
- **语义检查**：验证表、列、用户权限是否有效。S
- 转换为内部可操作结构供优化器使用。

#### 4. **优化（Optimizer）**

优化器对语法树做转换（如常数折叠、JOIN 重排等），并且基于数据统计信息生成多个执行计划，选择代价最低的那个。

#### 5. **执行（Executor）**

执行器接收最佳计划，具体执行包括索引查找、表扫描、JOIN、聚合、排序等操作，期间调用存储引擎 API 访问实际数据。

#### 6. **存储引擎调用（Storage Engine）**

如 InnoDB 或 MyISAM，通过其接口执行真正的读写操作 —— 从磁盘加载数据页到 Buffer Pool，再操作后可能写回磁盘。B

#### 7. **结果返回给客户端**

执行器将最终结果逐行或批量通过连接器发送给客户端；若查询缓存开启并可用，还会将结果写入缓存 。

实现精准分页的核心在于：**避免 offset 扫描全表，并保证排序正确、查询高效**。常见策略如下：M

### 1. Cursor（游标）分页

每页记录最后一条作为游标（通常为全局唯一主键或时间戳），下一页通过 `WHERE key > last_key ORDER BY key LIMIT N` 查询。这种方式高效精准，不受页码深度影响，广泛应用于 API 和日志分页场景。

### 2. 全局排序 + Scatter–Gather

将分页请求路由至所有分片，使用 SQL 重写为 `LIMIT offset+limit OFFSET 0`，每个分片返回前 offset+limit 记录，由中间件进行归并排序后取真分页区间。这适合简单实现，但性能会随着页码加深、分片数量增加而显著下降，且占用网络、内存与 CPU 较高。S

### 3. 精确分页（双轮次或猜测加校正）

- **双轮查询**：先对每个分片拉取前 M 条（M = offset + limit），得到每片局部 max/min，然后再重新定位每片真实 offset 并补齐。
- **分布式偏移估算**：基于数据分布比例，估算每分片实际取值条数，减少冗余提取量，再归并精确分页。  
  这种方式兼顾效率与精准，适合中深度分页使用。

### 4. 禁用跳页 + 游标结合

禁止 arbitrary 页码跳转，仅支持顺序翻页模式。每页仅 `WHERE id > last_id ORDER…` 查询固定页大小数据，减少全局合并与扫描，为许多应用提供最佳实用性与性能。B

Hash 存储适合高基数、只做等值查找的场景，如用户 Session 快速查验、缓存键值对、内存中频繁读写的小结构数据。不要在需要排序、范围查、低选择性或磁盘持久化的场景滥用 Hash。 以下三种情况适合：

- **适用于等值查找**：Hash 表通过哈希函数计算键的桶位置，实现平均 O(1) 的查找、插入与删除，适用于 `WHERE key = value` 类型的查询，例如检查用户 ID 是否存在或快速查找缓存数据。
- **唯一性高的键**：当字段基数高、值分布均匀时，Hash 索引查找效率更优；但不支持范围操作（`>`, `BETWEEN` 等），也无法用于排序。
- **内存表或缓存结构**：在 Postgres 内存表或类似设计中，Hash 索引非常高效，适合频繁的等值查询场景。

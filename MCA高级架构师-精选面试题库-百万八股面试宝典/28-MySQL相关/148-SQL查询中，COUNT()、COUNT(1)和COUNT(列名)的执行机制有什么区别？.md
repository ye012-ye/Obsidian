1. `COUNT(*)`

- **功能**：统计所有行数，包括NULL值。
- **实现**：现代数据库（如MySQL InnoDB）会直接读取表的元信息或主键索引，避免全表扫描。
- **性能**：通常最优，因为引擎会做特殊优化。

2. `COUNT(1)`

- **功能**：与`COUNT(*)`等效，统计所有行。
- **实现**：对每行返回常数1，不检查实际数据。
- **性能**：多数数据库会将其优化为与`COUNT(*)`相同的执行计划。

3. `COUNT(列名)`

- **功能**：统计该列**非NULL**的行数。
- **实现**：必须扫描列值，过滤NULL条目。
- **性能**：若列包含大量NULL或未建索引，性能较差。

M 关键差异：`COUNT(列名)`需要检查数据内容S，而`COUNT(*)`和`COUNT(1)`仅统计行数。B

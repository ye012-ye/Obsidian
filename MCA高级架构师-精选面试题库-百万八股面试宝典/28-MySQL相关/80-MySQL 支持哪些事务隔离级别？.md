MySQL（InnoDB）共支持四种事务隔离级别：M

1. **读未提交（READ UNCOMMITTED）**

- 最低隔离级别，可读取其他事务未提交的数据。
- 会发生脏读、不可重复读、幻读。

2. **读已提交（READ COMMITTED）**

- 只读取已提交的数据，避免脏读。
- 但同一事务中多次读取可能变化（不可重复读），也可能出现幻读。

3. **可重复读（REPEATABLE READ，默认）**

- 事务内对同一行的多次读取结果一致，避免脏读和不可重复读。
- 但新插入行可能被检测到，仍可能发生幻读（InnoDB 使用 MVCC 快照）。

4. **串行化（SERIALIZABLE）**

- 最高隔离级别，将事务按顺序执行。
- 不会发生脏读、不可重复读、幻读，但性能最低并发差。S

**建议用法：**  
在大多数业务中，默认的 **可重复读** 已能满足一致性要求；  
若强一致性优先，可升到 **串行化**；  
性能可优先时，可选 **读已提交** 或 **读未提交**（仅限对一致性要求低的场景）。 B

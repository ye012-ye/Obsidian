在 Kubernetes 中，调度器（kube-scheduler）是控制平面组件之一，负责将新创建的 Pod 分配到合适的节点上。其核心作用是根据预定义的策略和算法，确保 Pod 在集群中得到最佳的资源分配。

M

调度器的工作流程包括以下几个关键步骤：

1. **监听 API Server**：调度器持续监听 API Server，发现新创建且尚未被调度的 Pod。
2. **过滤阶段（Filtering）**：调度器根据预设的预选策略（Predicates）筛选出满足 Pod 运行需求的节点列表。例如，节点的资源是否足够、是否存在污点（taints）等。
3. **优选阶段（Scoring）**：在满足预选条件的节点中，调度器根据优选策略（Priorities）为每个节点打分，选择最优节点进行调度。优选策略可能考虑节点的资源可用性、公平性、通信频繁程度等因素。
4. **绑定阶段（Binding）**：调度器将选定的节点与 Pod 绑定，更新 Pod 的调度信息，完成调度过程。

S

调度器还负责确保调度的公平性和资源的充分利用。在实际操作中，用户往往希望 Pod 的调度策略是可控的，以处理大量复杂的实际问题。因此，平台允许多个调度器并行工作，同时支持自定义调度器。

B

总的来说，Kubernetes 中的调度器是一个关键组件，它通过合理、充分地调度 Pod 到最优的工作节点上，提高了整个集群的可用性和性能。

**Kubernetes（K8s）简介**  
K8s 是 Kubernetes 的缩写，是一个由 Cloud Native Computing Foundation（CNCF）维护的开源集群编排系统。它可实现容器化应用的自动部署、弹性扩缩、滚动更新和自愈管理。它融合了 Google 多年的生产经验与社区最佳实践 。

M

**它解决了哪些主要问题？**  
Kubernetes 从容器大规模运行的角度出发，主要解决如下挑战：容忍单个容器或节点故障自动恢复，智能调度容器避免负载热点，自动扩容以应对突增流量，以及服务发现与负载均衡无需人工干预。通过将部署状态声明为“期望状态”，K8s 能持续调整实际状态以匹配目标，极大减少维护负担。

S

**核心架构组成**  
一个 K8s 集群由两大部分构成：控制平面（Control Plane）与工作节点（Worker Nodes）。

- 控制平面包括 API Server、Scheduler、Controller Manager 和 etcd 存储。这些组件共同决策集群资源调度、状态恢复以及集群状态持久化存储。API Server 是集群唯一入口，接收用户与系统请求；Scheduler 根据资源与策略选择最佳节点；Controller Manager 负责扩缩容、故障恢复及滚动升级；etcd 负责持久保存所有资源的期望状态与当前状态 。
- 工作节点上运行 kubelet、kube-proxy 以及容器运行时（如 containerd 或 CRI‑O）。kubelet 定期向控制平面汇报节点健康，并确保 Pod 在节点上创建、重启或销毁；kube-proxy 实现服务内部发现与网络负载均衡；容器运行时则负责具体容器的生命周期管理 。

B

**Pod 是什么及其为何重要？**  
Pod 是 Kubernetes 的最小工作负载单元，通常包含一个或多个协同工作的容器，它们共享网络与存储。Pod 之间可以通过 `localhost` 通信，并作为调度单元被控制平面分配到节点上。Pod 的设计使得应用组件可以高效协作，简化多容器场景部署 。

**总结**:K8s 是一种声明式、自动化的容器编排系统，它通过对集群控制和容器生命周期的管理，解放开发者运维双手，保障应用稳定运行。其核心架构将控制 Plane 与数据 Plane 清晰拆分，通过 API Server、Scheduler、Controller 和 kubelet/kube‑proxy等组件协作，实现高可用、高弹性、自动化管理的目标。

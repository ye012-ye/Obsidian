Dubbo 是一个由 Apache 提供的高性能 RPC 服务开发框架，最初由阿里巴巴开源，用于构建和治理微服务系统。它主要负责以下关键能力：

1. **远程方法调用（RPC）：**  
   Dubbo 支持接口驱动，通过定义服务接口并在提供方实现，消费者可以像调用本地方法一样发起调用，内部基于高效的协议完成通信。M
2. **服务注册与发现：**  
   服务提供者启动时将元信息注册到注册中心（如 ZooKeeper、Nacos），消费者则动态订阅可用服务，实现自动发现与调用。
3. **负载均衡与高可用容错：**  
   内置多种负载均衡策略，消费者自动从注册中心获取多个提供者节点并均衡请求。此外支持熔断、重试等机制，保障系统可靠性。S
4. **流量治理：**  
   提供路由、限流、降级、灰度发布等功能，开发者可通过 Dubbo Admin 控制台或配置中心实时调整流量策略，支持运维治理。
5. **协议与多语言支持：**  
   Dubbo 不依赖单一协议，支持 TCP、HTTP/2、REST、gRPC（triple）、Thrift 等，并可在同端口同时暴露多协议服务。从 Dubbo3 起还提供 Java、Go、Rust、Node.js 等 SDK 支持。B
6. **观测与治理能力：**  
   内置监控埋点，能够收集调用次数、耗时等指标，并可集成 Admin 控制台、Tracing、Metrics、审计等组件，实现服务治理与性能可视化。

在 PostgreSQL 中，共有四种隔离级别，用于平衡性能与事务间可见性的需求。以下从低到高依次说明：M

### 1. Read Uncommitted（读未提交）

这是理论上最低的隔离级别，允许读取未提交事务的数据（脏读），很容易导致数据不一致。PostgreSQL 实际上会将其降级为 Read Committed，也就是不会真正暴露脏读，所以安全性更高。S

### 2. Read Committed（读已提交）

这是 PostgreSQL 默认隔离级别。每个查询都会读取当前已提交的数据，避免脏读。事务中多次读取可能看到不同的数据版本，因此允许不可重复读和幻读。它通过 MVCC 快照机制实现较高性能，是大多数 OLTP 场景的首选 M

### 3. Repeatable Read（可重复读）

该隔离级别在事务开始时建立快照，之后所有读取操作都基于该快照执行，因此不会看到后续事务的提交内容，避免了脏读与不可重复读。PostgreSQL 的 Repeatable Read 实现比标准更强，通常不产生幻读，但可能存在序列化异常（write skew）。S

### 4. Serializable（串行化）

最高隔离级别，提供强一致性保障。除了拥有 Repeatable Read 的全部特性，还会检测并阻止可能导致不一致的并发写冲突。采用可序列化快照隔离（SSI）机制，在检测到冲突时会中止某些事务并返回错误，要求重试。该级别开销较大，但适合关键金融或强一致性场景。B

### 对比

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **隔离级别** | **脏读** | **不可重复读** | **幻读** | **实现机制** |
| Read Uncommitted | 支持 | 支持 | 支持 | 等同 Read Committed |
| Read Committed | 不支持 | 支持 | 支持 | 每查询快照 |
| Repeatable Read | 不支持 | 不支持 | 不支持 | 固定快照 + MVCC |
| Serializable | 不支持 | 不支持 | 不支持 | SSI 写冲突检测 |

**注**：PostgreSQL 的 Repeatable Read 实际禁止幻读；ANSI 标准允许，但该实现更稳固。

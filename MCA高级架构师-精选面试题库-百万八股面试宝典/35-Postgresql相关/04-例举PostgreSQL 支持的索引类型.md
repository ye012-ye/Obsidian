PostgreSQL 原生支持 **六种主要索引类型**，分别针对不同数据类型和查询需求进行优化。下面对比它们的核心特性与典型应用场景：

M

### 1. **B‑Tree**

- **适用查询**：等值与范围查找（如 `=`, `<`, `>`, `BETWEEN`）。
- **特征**：平衡树结构，默认索引类型，支持排序与唯一性约束。
- **优势**：通用、高效，尤其适用于主键、频繁查询字段。
- **劣势**：随机插入（如 UUID）可能导致节点膨胀、写放大。

### 2. **Hash**

- **适用查询**：纯等值查询（`=`）。
- **特征**：基于哈希表，查找速度快但不支持范围。
- **优势**：等值场景中略快于 B‑Tree。
- **劣势**：不 WAL 日志，不支持 crash recovery，应用受限。

### 3. **GiST（Generalized Search Tree）**

- **适用查询**：空间、全文、近似查询等复杂结构。
- **特征**：可扩展树结构，支持地理数据、全文等。
- **优势**：支持 k‑NN、范围重叠查询，如 PostGIS、全文检索。
- **劣势**：可能返回假阳性，需回表验证；写性能一般。

S

### 4. **SP‑GiST（空间分区 GiST）**

- **适用查询**：非均衡结构，如 quad-tree、k-d tree 等。
- **特征**：优化非平衡数据、地理坐标、前缀等索引。
- **优势**：适合分区型数据结构、高效支持空间查询。

### 5. **GIN（Generalized Inverted Index）**

- **适用查询**：多值字段（如数组、JSONB）、全文检索。
- **特征**：倒排索引，索引项较多，存储结构复杂。
- **优势**：支持包含查询（`@>`, `&&`），是 JSONB、全文检索的首选。
- **劣势**：索引体积大，写入/更新开销高。

### 6. **BRIN（Block Range Index）**

- **适用查询**：超大表中的数据范围检索，如时间序列。
- **特征**：为每个磁盘块记录 min/max，无需行级索引。
- **优势**：索引体积小，速度快，适合顺序数据。
- **劣势**：过滤精度低，受数据物理排序影响大。

B

### ​**对比总结**

|  |  |  |  |  |
| --- | --- | --- | --- | --- |
| **索引类型** | **查询类型** | **优势** | **劣势** | **常见场景示例** |
| B‑Tree | 等值 & 范围 | 通用、高速、支持排序 | 大随机数据导致膨胀 | 主键、时间字段 |
| Hash | 纯等值 | 查找快 | 不持久化、不支持范围查询 | 高频等值查询 |
| GiST | 范围、空间、相似性 | 灵活、支持复杂数据类型 | 假阳性、回表性能开销 | PostGIS、全文 |
| SP‑GiST | 非平衡空间分布 | 分区数据、前缀索引优化 | 使用场景较少 | 前缀搜索、地理数据 |
| GIN | 多值、全文检索 | 支持包含、数组、JSON 操作 | 写入慢，占空间 | JSONB、全文搜索 |
| BRIN | 范围扫描（大表） | 占用小、适合有序数据 | 精度低，依赖物理顺序 | 日志、时序数据 |

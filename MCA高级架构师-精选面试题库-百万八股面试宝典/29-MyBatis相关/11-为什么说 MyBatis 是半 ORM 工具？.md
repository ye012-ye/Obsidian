MyBatis 被称作**半 ORM（Object-Relational Mapping）**框架，原因在于它介于全手写 SQL（JDBC）和全自动 ORM（如 Hibernate）之间，既提供了映射功能，又保留了 SQL 控制权。

M

#### ​控制权与定位

- **编写 SQL 语句**：不同于 Hibernate 全自动生成，MyBatis 要求开发者手写 SQL，用于插入、查询、更新和删除，这意味着你对查询逻辑完全掌控，性能可调、高度定制。
- **手动定义映射**：通过 XML 或注解将查询结果映射到 Java POJO，但并不自动推断数据库结构，仅映射 SQL 与对象关系。

S

#### ​为什么是“半” ORM？

- **自动化程度介于 ORM 与 JDBC 之间**：MyBatis 自动处理参数绑定、结果集映射、动态 SQL、延迟加载等，但不会自动生成 SQL 或进行复杂实体关联管理（如事务管理、关联级联、对象状态跟踪等）。
- **映射粒度更细**：它专注于 Java 方法到 SQL 的映射，而不是 Java 类到数据库表的映射，保留了对 SQL 语句的可见性和优化能力。执行效率更高、调优更灵活，特别适合复杂或性能敏感场景。
- **无侵入领域对象**：POJO 无需嵌入 ORM 框架逻辑，可独立于持久层存在，保持领域模型纯粹。

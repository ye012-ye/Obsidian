M

#### ​优势

MyBatis 提供了对 SQL 的精细控制，你可以手写优化查询、使用存储过程和数据库特性，非常适合复杂或性能敏感场景。这种明确可见的 SQL 控制方式，使得调优和定位问题时更加直接有效。

它对 JDBC 操作进行了封装，消除了大量模板代码，如连接开关、结果集处理等，显著减少开发量并提升代码可读性。根据统计，MyBatis 能省去50% 以上的样板代码，提升开发效率。

MyBatis 的配置方式清晰简洁，通过 XML 或注解分离 SQL 与代码逻辑，也方便数据库管理员或 DBA 独立管理 SQL。它支持动态 SQL、ResultMap、插件体系、数据缓存，功能不依赖数据库厂商，可与 Spring 等框架无缝集成。

S

#### ​不足

由于必须手写 SQL，特别是在多表关联和复杂业务场景下，开发人员需要具备扎实的 SQL 能力；维护大量 SQL 可能成为一项负担。

SQL 语句高度依赖具体数据库语法，导致 MyBatis 项目在切换数据库品种时需要重写或调整语句，从而降低迁移的易用性。

MyBatis 的映射和 SQL 本质上为字符串形式，缺乏编译时的类型检查，可能导致运行时错误；如果没有配置生成器，维护映射工作量较大。

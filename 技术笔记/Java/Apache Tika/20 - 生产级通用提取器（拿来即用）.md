# 20 · 生产级通用提取器（拿来即用）

> [!info] 所属系列
> ← [[19 - 大神进阶 - 自定义解析器开发]]　|　返回 [[00 - Apache Tika 教程总览]]

> [!abstract] 这是什么
> 前 19 篇把每个零件拆开讲透了。**这一篇把它们焊成一个零件**：一个**复制进项目就能用**的、**通用**（吃任何格式）、**生产级**（隔离 / 限额 / 超时 / 不可信文件防护 / 结构化结果 / 不崩）的 `TikaExtractor` 组件。
> 适用：RAG / 搜索引擎喂料、知识库、文档治理、批量文本抽取。基于 **Tika 3.2.3 + JDK 17**。

---

## 0. 生产级的 9 条底线

> [!check] 这个组件帮你兜住的事
> 1. **线程安全单例**：`AutoDetectParser` 全局复用，不为每次请求重扫 SPI（见 [[15 - 性能调优与最佳实践#7 重复构造 vs 复用]]）
> 2. **写入上限**：`writeLimit` 截断超大文本，截断不报错只标记 `truncated`
> 3. **嵌入数上限**：`maxEmbeddedResources` 挡 Zip Bomb / 嵌套炸弹
> 4. **解析超时**：每个文件独立 `Future` + `timeout`，卡死不拖垮线程池
> 5. **MIME 白名单**：先用 `Detector` 验真，不信客户端给的类型
> 6. **加密文档**：可选 `PasswordProvider`，没密码不抛栈，返回 `ENCRYPTED`
> 7. **`TikaInputStream`**：大文件自动落临时盘 + 自动清理
> 8. **结构化不可变结果**：`ExtractResult`（record + 不可变集合），永不返回半截状态
> 9. **批处理友好**：坏文件返回 `FAILED` 状态而**不抛异常**，循环不中断

> [!warning] 一个诚实的边界
> 应用层超时能让线程**返回**，但不一定能让 Tika 内部循环**停下**（见 [[17 - 异常处理与故障排查#7 解析超时]]）。要对**完全不可信**的文件做到硬隔离，必须上 **`ForkParser` 子进程**或 **tika-server `--spawnChild`**，见本文 [[#6. 不可信文件 → ForkParser 子进程隔离|第 6 节]] 与 [[15 - 性能调优与最佳实践#3 ForkParser 子进程隔离]]。

## 2. 结果与配置（不可变）

先定结果与配置的形状。全部 `record` + 不可变集合 —— 拿到手的结果不会被别处偷改。

```java
package com.example.tika;

import java.util.Map;

/** 单个文档（或单个嵌入文档）的抽取结果。不可变。 */
public record ExtractResult(
        Status status,
        String text,
        String contentType,
        Map<String, String> metadata,
        boolean truncated,
        String message) {

    public ExtractResult {
        // 收口：永不暴露可变集合 / null
        metadata = metadata == null ? Map.of() : Map.copyOf(metadata);
        text = text == null ? "" : text;
    }

    public boolean ok() {
        return status == Status.OK;
    }

    public enum Status {
        OK,          // 成功（text 可信；truncated=true 时为截断版）
        ENCRYPTED,   // 加密且没给对密码
        TIMEOUT,     // 超过 parseTimeout
        SKIPPED_MIME,// 不在 MIME 白名单
        FAILED       // 文件损坏 / 不支持 / IO 失败（message 里有原因）
    }
}
```

```java
package com.example.tika;

import org.apache.tika.parser.PasswordProvider;

import java.time.Duration;
import java.util.Set;

/** 提取器配置。用 builder 装配，build 出来即不可变。 */
public record ExtractorConfig(
        int writeLimit,              // 单文档字符上限，-1 = 不限
        int maxEmbeddedResources,    // 最多解多少个嵌入文档
        Duration parseTimeout,       // 单文件解析超时
        Set<String> allowedMimeTypes,// MIME 白名单（baseType），空 = 不限
        PasswordProvider passwordProvider) { // 加密文档取密码，可为 null

    public ExtractorConfig {
        if (parseTimeout == null || parseTimeout.isZero() || parseTimeout.isNegative()) {
            throw new IllegalArgumentException("parseTimeout 必须为正");
        }
        allowedMimeTypes = allowedMimeTypes == null ? Set.of() : Set.copyOf(allowedMimeTypes);
    }

    public static Builder builder() {
        return new Builder();
    }

    public static final class Builder {
        private int writeLimit = 10_000_000;            // 1000 万字符
        private int maxEmbeddedResources = 200;
        private Duration parseTimeout = Duration.ofSeconds(60);
        private Set<String> allowedMimeTypes = Set.of();
        private PasswordProvider passwordProvider = null;

        public Builder writeLimit(int v)            { this.writeLimit = v; return this; }
        public Builder maxEmbeddedResources(int v)  { this.maxEmbeddedResources = v; return this; }
        public Builder parseTimeout(Duration v)     { this.parseTimeout = v; return this; }
        public Builder allowedMimeTypes(Set<String> v) { this.allowedMimeTypes = v; return this; }
        public Builder passwordProvider(PasswordProvider v) { this.passwordProvider = v; return this; }

        public ExtractorConfig build() {
            return new ExtractorConfig(writeLimit, maxEmbeddedResources,
                    parseTimeout, allowedMimeTypes, passwordProvider);
        }
    }
}
```

## 3. 核心组件 `TikaExtractor`

一个类搞定。线程安全，可作为单例 / Spring Bean 长期持有，用完 `close()`。

```java
package com.example.tika;

import org.apache.tika.config.TikaConfig;
import org.apache.tika.detect.Detector;
import org.apache.tika.exception.EncryptedDocumentException;
import org.apache.tika.exception.WriteLimitReachedException;
import org.apache.tika.io.TikaInputStream;
import org.apache.tika.metadata.Metadata;
import org.apache.tika.metadata.TikaCoreProperties;
import org.apache.tika.mime.MediaType;
import org.apache.tika.parser.AutoDetectParser;
import org.apache.tika.parser.ParseContext;
import org.apache.tika.parser.Parser;
import org.apache.tika.parser.PasswordProvider;
import org.apache.tika.sax.BodyContentHandler;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

import java.io.IOException;
import java.io.InputStream;
import java.nio.file.Path;
import java.util.LinkedHashMap;
import java.util.Map;
import java.util.Objects;
import java.util.concurrent.ExecutionException;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.Future;
import java.util.concurrent.TimeUnit;
import java.util.concurrent.TimeoutException;

import static com.example.tika.ExtractResult.Status.*;

/**
 * 生产级通用文本/元数据提取器。
 * <p>线程安全：{@link AutoDetectParser} 与本类的方法可被多线程并发调用。
 * <p>用完调用 {@link #close()} 关闭内部超时线程池。
 */
public final class TikaExtractor implements AutoCloseable {

    private static final Logger log = LoggerFactory.getLogger(TikaExtractor.class);

    private final Parser parser = new AutoDetectParser();      // 线程安全，全局复用
    private final Detector detector = TikaConfig.getDefaultConfig().getDetector();
    private final ExtractorConfig config;
    private final ExecutorService timeoutPool;

    public TikaExtractor(ExtractorConfig config) {
        this.config = Objects.requireNonNull(config, "config");
        this.timeoutPool = Executors.newCachedThreadPool(r -> {
            Thread t = new Thread(r, "tika-parse");
            t.setDaemon(true);   // 别拦住 JVM 退出
            return t;
        });
    }

    public static TikaExtractor withDefaults() {
        return new TikaExtractor(ExtractorConfig.builder().build());
    }

    /** 从文件抽取。坏文件不抛异常，返回 FAILED 状态——批处理友好。 */
    public ExtractResult extract(Path file) {
        try (TikaInputStream tis = TikaInputStream.get(file)) {   // 文件流，解析器可重读
            return doExtract(tis, file.getFileName().toString());
        } catch (IOException e) {
            log.warn("打开文件失败: {}", file, e);
            return new ExtractResult(FAILED, null, null, Map.of(), false, e.toString());
        }
    }

    /** 从任意输入流抽取（name 仅用于日志/元数据，可为文件名）。 */
    public ExtractResult extract(InputStream in, String name) {
        try (TikaInputStream tis = TikaInputStream.get(in)) {     // 必要时落临时盘并自动清理
            return doExtract(tis, name);
        } catch (IOException e) {
            log.warn("读取流失败: {}", name, e);
            return new ExtractResult(FAILED, null, null, Map.of(), false, e.toString());
        }
    }

    // —— 真正干活的地方 ——
    private ExtractResult doExtract(TikaInputStream tis, String name) throws IOException {
        Metadata metadata = new Metadata();
        metadata.set(TikaCoreProperties.RESOURCE_NAME_KEY, name);

        // 1) 先验真 MIME（不信客户端），Detector 用 mark/reset，不消费流
        MediaType detected = detector.detect(tis, metadata);
        String baseType = detected.getBaseType().toString();
        if (!config.allowedMimeTypes().isEmpty()
                && !config.allowedMimeTypes().contains(baseType)) {
            return new ExtractResult(SKIPPED_MIME, "", baseType, Map.of(), false,
                    "MIME 不在白名单: " + baseType);
        }
        metadata.set(Metadata.CONTENT_TYPE, detected.toString()); // 复用检测结果，省一次

        // 2) BodyContentHandler 会自动把"嵌入文档的正文"也拼进来（因 ctx 里设了 Parser）
        BodyContentHandler handler = new BodyContentHandler(config.writeLimit());
        ParseContext ctx = buildContext();

        // 3) 丢进独立线程跑，套超时
        Future<?> future = timeoutPool.submit(() -> {
            parser.parse(tis, handler, metadata, ctx);
            return null;
        });

        boolean truncated = false;
        try {
            future.get(config.parseTimeout().toMillis(), TimeUnit.MILLISECONDS);
        } catch (TimeoutException e) {
            future.cancel(true);
            log.info("解析超时: {}", name);
            return new ExtractResult(TIMEOUT, handler.toString(), baseType,
                    toMap(metadata), true, "超过 " + config.parseTimeout());
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            future.cancel(true);
            return new ExtractResult(FAILED, null, baseType, toMap(metadata), false, "被中断");
        } catch (ExecutionException e) {
            Throwable cause = e.getCause();
            if (isWriteLimitReached(cause)) {
                truncated = true;   // 截断不是错：handler 里是截断版正文，metadata 仍可信
            } else if (cause instanceof EncryptedDocumentException) {
                return new ExtractResult(ENCRYPTED, "", baseType, toMap(metadata), false, "加密文档");
            } else {
                log.warn("解析失败: {}", name, cause);   // 损坏/不支持/POI/PDFBox 异常
                return new ExtractResult(FAILED, "", baseType, toMap(metadata), false,
                        String.valueOf(cause));
            }
        }
        return new ExtractResult(OK, handler.toString(), baseType, toMap(metadata), truncated, null);
    }

    private ParseContext buildContext() {
        ParseContext ctx = new ParseContext();
        ctx.set(Parser.class, parser);   // 关键：嵌入文档（附件/压缩包/内嵌office）才会被递归解析
        PasswordProvider pp = config.passwordProvider();
        if (pp != null) {
            ctx.set(PasswordProvider.class, pp);
        }
        return ctx;
    }

    private static Map<String, String> toMap(Metadata m) {
        Map<String, String> out = new LinkedHashMap<>();
        for (String name : m.names()) {
            out.put(name, m.get(name));
        }
        return out;
    }

    /** 写入上限既可能直接抛，也可能被包成 SAXException 的 cause——两种都认。 */
    private static boolean isWriteLimitReached(Throwable t) {
        while (t != null) {
            if (t instanceof WriteLimitReachedException) {
                return true;
            }
            t = t.getCause();
        }
        return false;
    }

    @Override
    public void close() {
        timeoutPool.shutdownNow();
    }
}
```

> [!note] 为什么 `BodyContentHandler` 就够"通用"
> `AutoDetectParser` 只要 `ParseContext` 里设了 `Parser.class`，就会**自动钻进嵌入文档**（邮件附件、ZIP 条目、Office 内嵌文件）把它们的正文一并拼进同一个 handler。所以"给我这文件里所有的字"这个最常见需求，一个 `BodyContentHandler` 全包。要**按文档分开拿元数据**才需要下一节的递归模式。

## 4. 用法

```java
// 启动时建一次，全局复用；用完 close（Spring 见第 7 节）
try (TikaExtractor extractor = new TikaExtractor(
        ExtractorConfig.builder()
            .writeLimit(5_000_000)
            .parseTimeout(Duration.ofSeconds(30))
            .allowedMimeTypes(Set.of(
                "application/pdf",
                "application/vnd.openxmlformats-officedocument.wordprocessingml.document",
                "text/plain", "text/html"))
            .build())) {

    ExtractResult r = extractor.extract(Path.of("report.pdf"));
    if (r.ok()) {
        System.out.println("类型: " + r.contentType());
        System.out.println("作者: " + r.metadata().get("dc:creator"));
        System.out.println("字数: " + r.text().length() + (r.truncated() ? "(截断)" : ""));
    } else {
        System.out.println("跳过/失败: " + r.status() + " - " + r.message());
    }
}
```

### 批处理（一坏不影响全局）

```java
ExecutorService pool = Executors.newFixedThreadPool(
        Runtime.getRuntime().availableProcessors());
try (TikaExtractor extractor = TikaExtractor.withDefaults()) {
    List<Future<ExtractResult>> futures = files.stream()
            .map(f -> pool.submit(() -> extractor.extract(f)))
            .toList();
    for (Future<ExtractResult> f : futures) {
        ExtractResult r = f.get();          // extract 本身不抛，这里只会拿到状态码
        index(r);                           // 喂给 ES/Lucene，见 [[18 - 高级实战 - 全文检索集成]]
    }
} finally {
    pool.shutdown();
}
```

> 线程数 ≈ CPU 核数；开 OCR 时设 `核数 - 1`，留点给 IO（见 [[15 - 性能调优与最佳实践#5 并行批处理]]）。真·大规模直接上 `tika-pipes`。

## 5. 进阶：按文档分开拿元数据（递归模式）

需求是"容器 + 每个嵌入文档**各自**的正文与元数据"（典型：邮件 + 各附件分别入库）时，用 `RecursiveParserWrapper`，它把结果摊成一个 `List<Metadata>`，每个元素的正文在 `TikaCoreProperties.TIKA_CONTENT` 里。

```java
public record DocSegment(String name, String path, String contentType, String text) {}

/** 容器 + 每个嵌入文档各一个 DocSegment。list[0] 通常是容器自身。 */
public List<DocSegment> extractRecursive(Path file) {
    Parser recursive = new RecursiveParserWrapper(parser);
    var factory = new BasicContentHandlerFactory(
            BasicContentHandlerFactory.HANDLER_TYPE.TEXT, config.writeLimit());
    var handler = new RecursiveParserWrapperHandler(factory, config.maxEmbeddedResources());

    Metadata metadata = new Metadata();
    metadata.set(TikaCoreProperties.RESOURCE_NAME_KEY, file.getFileName().toString());

    ParseContext ctx = new ParseContext();
    ctx.set(Parser.class, recursive);
    if (config.passwordProvider() != null) {
        ctx.set(PasswordProvider.class, config.passwordProvider());
    }

    try (TikaInputStream tis = TikaInputStream.get(file)) {
        recursive.parse(tis, handler, metadata, ctx);
    } catch (Exception e) {
        log.warn("递归解析失败: {}", file, e);   // 已解出的部分仍在 metadataList 里
    }

    return handler.getMetadataList().stream()
            .map(m -> new DocSegment(
                    m.get(TikaCoreProperties.RESOURCE_NAME_KEY),
                    m.get(TikaCoreProperties.EMBEDDED_RESOURCE_PATH),
                    m.get(Metadata.CONTENT_TYPE),
                    Optional.ofNullable(m.get(TikaCoreProperties.TIKA_CONTENT)).orElse("")))
            .toList();
}
```

> 需要的额外 import：`org.apache.tika.parser.RecursiveParserWrapper`、`org.apache.tika.sax.BasicContentHandlerFactory`、`org.apache.tika.sax.RecursiveParserWrapperHandler`、`java.util.Optional`。底层细节见 [[12 - 嵌入式文档与递归解析]]。

> [!bug] 递归模式 + 超时/ForkParser 不要硬凑
> `RecursiveParserWrapperHandler` 把元数据列表攒在**本地**，跨 `ForkParser` 子进程边界拿不回来。要既要按文档分元数据、又要硬进程隔离，正解是 **tika-server `/rmeta` 端点**（见 [[13 - tika-server REST API]]），别在进程内自己拼。

## 6. 不可信文件 → ForkParser 子进程隔离

第 3 节的应用层超时**挡不住** OOM 和不响应中断的死循环。处理"陌生人上传的文件"，把解析关进子进程，主进程永远安全。这里换成 `ForkParser` + `BodyContentHandler`（容器正文已含嵌入文本）：

```java
ForkParser fork = new ForkParser(
        Thread.currentThread().getContextClassLoader(),
        new AutoDetectParser());
fork.setJavaCommand(List.of("java", "-Xmx512m",
        "-XX:MaxMetaspaceSize=128m", "-Dfile.encoding=UTF-8"));
fork.setServerPulseMillis(5_000);
fork.setServerParseTimeoutMillis(60_000);   // 子进程内的硬超时
fork.setServerWaitTimeoutMillis(60_000);

BodyContentHandler handler = new BodyContentHandler(writeLimit);
try (TikaInputStream tis = TikaInputStream.get(untrustedFile)) {
    fork.parse(tis, handler, new Metadata(), new ParseContext());
} finally {
    fork.close();   // 关子进程，别泄漏
}
```

> [!danger] 生产首选：tika-server 模式
> 自己管 `ForkParser` 生命周期容易踩坑。**真上生产，最稳的是业务无状态调 `tika-server` + `--spawnChild`，水平扩 N 副本**——隔离、限流、扩容全交给它。详见 [[15 - 性能调优与最佳实践#12 推荐部署形态]] 和 [[13 - tika-server REST API]]。

## 7. Spring Boot 接入

把 `TikaExtractor` 做成单例 Bean，`destroyMethod` 收尾即可。完整版（文件上传、流式、REST）见 [[16 - Spring Boot 集成]]。

```java
@Configuration
public class TikaConfiguration {

    @Bean(destroyMethod = "close")
    public TikaExtractor tikaExtractor() {
        return new TikaExtractor(ExtractorConfig.builder()
                .writeLimit(10_000_000)
                .parseTimeout(Duration.ofSeconds(60))
                .maxEmbeddedResources(200)
                .build());
    }
}
```

```java
@RestController
@RequestMapping("/api/extract")
public class ExtractController {

    private final TikaExtractor extractor;

    public ExtractController(TikaExtractor extractor) {
        this.extractor = extractor;
    }

    @PostMapping
    public ResponseEntity<ExtractResult> extract(@RequestParam MultipartFile file) throws IOException {
        ExtractResult r = extractor.extract(file.getInputStream(), file.getOriginalFilename());
        // 用状态码区分，加密/不支持不当作 500
        return switch (r.status()) {
            case OK -> ResponseEntity.ok(r);
            case ENCRYPTED, SKIPPED_MIME -> ResponseEntity.unprocessableEntity().body(r);
            case TIMEOUT -> ResponseEntity.status(HttpStatus.REQUEST_TIMEOUT).body(r);
            case FAILED -> ResponseEntity.badRequest().body(r);
        };
    }
}
```

## 8. 配套：硬化的 `tika-config.xml`

把限额和"关掉危险解析器"固化进配置，比散落在代码里更稳（语法详见 [[11 - tika-config.xml 配置]]）。

```xml
<?xml version="1.0" encoding="UTF-8"?>
<properties>
  <parsers>
    <parser class="org.apache.tika.parser.DefaultParser">
      <!-- 关掉执行类/数据库类解析器，降攻击面（见 [[15]] 第 8 节） -->
      <parser-exclude class="org.apache.tika.parser.executable.ExecutableParser"/>
      <parser-exclude class="org.apache.tika.parser.sqlite3.SQLite3Parser"/>
    </parser>
  </parsers>
  <autoDetectParserConfig>
    <writeLimit>10000000</writeLimit>
    <maxEmbeddedResources>200</maxEmbeddedResources>
    <throwOnWriteLimitReached>false</throwOnWriteLimitReached>
  </autoDetectParserConfig>
</properties>
```

加载它（替换组件里那行 `TikaConfig.getDefaultConfig()`）：

```java
TikaConfig tikaConfig = new TikaConfig(Path.of("tika-config.xml"));
Parser parser = new AutoDetectParser(tikaConfig);
Detector detector = tikaConfig.getDetector();
```

## 9. 上线前对照表

> [!check] 抄作业
> - [ ] 用 **tika-bom** 锁版本，没有手写 POI/PDFBox 版本（防 `NoSuchMethodError`）
> - [ ] `AutoDetectParser` **单例复用**，没在请求里 new
> - [ ] `writeLimit` / `maxEmbeddedResources` 都设了上限
> - [ ] 每个文件套了 `parseTimeout`
> - [ ] 不可信来源走了 **ForkParser / tika-server `--spawnChild`** 隔离
> - [ ] MIME 走 `Detector` 验真，没信客户端 `Content-Type`
> - [ ] 网关层限了文件大小（Nginx `client_max_body_size`）

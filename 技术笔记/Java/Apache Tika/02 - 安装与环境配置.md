# 02 · 安装与环境配置

> [!info] 上一篇 / 下一篇
> ← [[01 - Tika 简介与核心概念]]　|　→ [[03 - 命令行工具 tika-app]]

## 1. 环境要求

| 项 | 要求 |
|---|---|
| JDK | **11+**（Tika 3.x 强制） |
| 内存 | 默认 512MB 起步，大 PDF/Office 建议 1GB+ |
| OS | 跨平台（Linux 推荐，Windows/macOS 都行） |
| 可选 | Tesseract OCR、FFmpeg（音视频元数据）、ExifTool |

## 2. Maven 依赖（推荐方式）

> [!tip] 推荐组合
> **`tika-core` + `tika-parsers-standard-package`**。
> 前者是 API，后者是所有"主流安全"的解析器打成一个 jar，不用一个个挑。

```xml
<properties>
    <tika.version>3.2.3</tika.version>
</properties>

<dependencies>
    <!-- 必须：核心接口与检测器 -->
    <dependency>
        <groupId>org.apache.tika</groupId>
        <artifactId>tika-core</artifactId>
        <version>${tika.version}</version>
    </dependency>

    <!-- 必须：所有主流格式的解析器 -->
    <dependency>
        <groupId>org.apache.tika</groupId>
        <artifactId>tika-parsers-standard-package</artifactId>
        <version>${tika.version}</version>
    </dependency>

    <!-- 可选：语言检测（Optimaize 实现） -->
    <dependency>
        <groupId>org.apache.tika</groupId>
        <artifactId>tika-langdetect-optimaize</artifactId>
        <version>${tika.version}</version>
    </dependency>
</dependencies>
```

## 3. Gradle 依赖

```kotlin
val tikaVersion = "3.2.3"

dependencies {
    implementation("org.apache.tika:tika-core:$tikaVersion")
    implementation("org.apache.tika:tika-parsers-standard-package:$tikaVersion")
    implementation("org.apache.tika:tika-langdetect-optimaize:$tikaVersion")
}
```

## 4. 想要更精细控制？按解析器分模块引

如果你只处理 PDF + Office，不想拉一个胖 jar，可以挑着用：

```xml
<!-- PDF -->
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-parser-pdf-module</artifactId>
    <version>${tika.version}</version>
</dependency>

<!-- 微软 Office 老格式（.doc / .xls / .ppt） -->
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-parser-microsoft-module</artifactId>
    <version>${tika.version}</version>
</dependency>

<!-- 微软 Office 新格式（.docx / .xlsx / .pptx）→ OOXML -->
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-parser-ooxml-module</artifactId>
    <version>${tika.version}</version>
</dependency>

<!-- HTML -->
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-parser-html-module</artifactId>
    <version>${tika.version}</version>
</dependency>

<!-- 图片元数据 (EXIF/IPTC) -->
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-parser-image-module</artifactId>
    <version>${tika.version}</version>
</dependency>
```

完整模块清单：https://central.sonatype.com/search?q=g:org.apache.tika

## 5. 单 jar 方式（CLI 用户）

去官网下载页：https://tika.apache.org/download.html

- `tika-app-3.2.3.jar` — 含 CLI + GUI + 所有解析器（≈ 100 MB）
- `tika-server-standard-3.2.3.jar` — 含 REST 服务（≈ 80 MB）

```bash
# 立刻可用
java -jar tika-app-3.2.3.jar --help
java -jar tika-app-3.2.3.jar --text my.pdf
```

详见 [[03 - 命令行工具 tika-app]]。

## 6. Docker 方式（最快上手 tika-server）

官方镜像：

```bash
# 标准版（含主流解析器，不含 OCR）
docker run -d --name tika -p 9998:9998 apache/tika:3.2.3.0

# 全功能版（含 Tesseract OCR）
docker run -d --name tika -p 9998:9998 apache/tika:3.2.3.0-full

# 测试
curl http://localhost:9998/tika
# 返回：This is Tika Server (Apache Tika 3.2.3). Please PUT
```

完整 REST 用法见 [[13 - tika-server REST API]]。

## 7. 验证安装（最小代码）

新建 `TikaSmoke.java`：

```java
import org.apache.tika.Tika;
import java.io.File;

public class TikaSmoke {
    public static void main(String[] args) throws Exception {
        Tika tika = new Tika();
        String content = tika.parseToString(new File(args[0]));
        System.out.println(content.substring(0, Math.min(500, content.length())));
    }
}
```

```bash
mvn compile exec:java -Dexec.mainClass=TikaSmoke -Dexec.args="some.pdf"
```

能看到文本就成功了。

## 8. 可选：装 Tesseract（要做 OCR 才需要）

```bash
# Ubuntu/Debian
sudo apt-get install tesseract-ocr tesseract-ocr-chi-sim tesseract-ocr-eng

# macOS
brew install tesseract tesseract-lang

# Windows
# 下载 https://github.com/UB-Mannheim/tesseract/wiki 安装包

# 验证
tesseract --version
tesseract --list-langs
```

Tika 会在 PATH 里自动找 `tesseract` 命令。完整 OCR 配置见 [[10 - OCR 集成 Tesseract]]。

## 9. 常见安装坑

> [!warning] SLF4J 警告
> Tika 用 SLF4J。如果你的项目没绑定具体实现（logback/log4j），会看到 `SLF4J: No SLF4J providers were found`。加一个绑定即可：
> ```xml
> <dependency>
>   <groupId>ch.qos.logback</groupId>
>   <artifactId>logback-classic</artifactId>
>   <version>1.5.6</version>
> </dependency>
> ```

> [!warning] 依赖冲突
> Tika 拉了 POI、PDFBox 等大量传递依赖。**和 Spring Boot 项目混用时容易和你已有的 POI 版本冲突**。优先级：让 Tika 用它自己测试过的版本，不要硬升 POI。

> [!warning] jar 包尺寸
> `tika-parsers-standard-package` 加上传递依赖 ≈ 60–100 MB。Lambda / Serverless 部署要注意冷启动；考虑只引需要的模块（第 4 节）。

> [!bug] Java 模块系统（JPMS）报错
> Tika 不是模块化 jar。如果你用 `module-info.java`，把 Tika 当 automatic module 用即可，必要时 `requires org.apache.tika.core;`。

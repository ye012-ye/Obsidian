# 19 · 大神进阶：自定义解析器开发

> [!info] 上一篇 / 下一篇
> ← [[18 - 高级实战 - 全文检索集成]]　|　→ 回到 [[00 - Apache Tika 教程总览]]

公司有私有格式（`.acme`、`.proto`、内部日志）想接入 Tika，或者想换掉某个内置 Parser 的实现？这一篇讲怎么做。

## 1. 整体思路

要让 Tika "认识" 一种新格式，要做两件事：

1. **写 MIME 定义** — 告诉 Tika 这个格式长什么样（魔数 + 后缀）
2. **写 Parser 实现** — 告诉 Tika 怎么解析

然后通过 **Java SPI** 把它们注册进去。

## 2. 步骤 1：MIME 定义

新建文件 `src/main/resources/org/apache/tika/mime/custom-mimetypes.xml`：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<mime-info>
    <mime-type type="application/x-acme-doc">
        <_comment>ACME 内部文档格式</_comment>

        <magic priority="60">
            <!-- 头 5 字节是 "ACME1" -->
            <match value="ACME1" type="string" offset="0"/>
        </magic>

        <glob pattern="*.acme"/>
        <glob pattern="*.acme.gz"/>

        <sub-class-of type="application/octet-stream"/>
        <root-XML localName="acmeDoc"
                  namespaceURI="https://acme.example/ns"/>
    </mime-type>
</mime-info>
```

支持的 `<magic>` 类型：`string`、`stringignorecase`、`byte`、`big16`、`big32`、`little16`、`little32`、`regex`…

Tika 启动时会**自动从 classpath 合并所有 `org/apache/tika/mime/custom-mimetypes.xml`**。

## 3. 步骤 2：实现 Parser

最简单写法 — 继承 `AbstractParser`：

```java
package com.acme.tika;

import org.apache.tika.exception.TikaException;
import org.apache.tika.metadata.Metadata;
import org.apache.tika.metadata.TikaCoreProperties;
import org.apache.tika.mime.MediaType;
import org.apache.tika.parser.AbstractParser;
import org.apache.tika.parser.ParseContext;
import org.apache.tika.sax.XHTMLContentHandler;
import org.xml.sax.ContentHandler;
import org.xml.sax.SAXException;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.nio.charset.StandardCharsets;
import java.util.Collections;
import java.util.Set;

public class AcmeDocParser extends AbstractParser {

    private static final Set<MediaType> SUPPORTED =
        Collections.singleton(MediaType.application("x-acme-doc"));

    @Override
    public Set<MediaType> getSupportedTypes(ParseContext context) {
        return SUPPORTED;
    }

    @Override
    public void parse(InputStream stream,
                      ContentHandler handler,
                      Metadata metadata,
                      ParseContext context)
            throws IOException, SAXException, TikaException {

        // 1) 一定要设 Content-Type，下游链路依赖它
        metadata.set(Metadata.CONTENT_TYPE, "application/x-acme-doc");

        // 2) 用 XHTMLContentHandler 包一层，自动给 <html><body> 包装
        XHTMLContentHandler xhtml = new XHTMLContentHandler(handler, metadata);
        xhtml.startDocument();

        try (BufferedReader r = new BufferedReader(
                new InputStreamReader(stream, StandardCharsets.UTF_8))) {

            // 跳过头 5 字节 "ACME1"
            r.skip(5);

            String line;
            int paraIdx = 0;
            while ((line = r.readLine()) != null) {
                if (line.startsWith("@TITLE:")) {
                    metadata.set(TikaCoreProperties.TITLE, line.substring(7).trim());
                } else if (line.startsWith("@AUTHOR:")) {
                    metadata.add(TikaCoreProperties.CREATOR, line.substring(8).trim());
                } else if (!line.isBlank()) {
                    xhtml.startElement("p");
                    xhtml.characters(line);
                    xhtml.endElement("p");
                    paraIdx++;
                }
            }
            metadata.set("acme:paragraphCount", String.valueOf(paraIdx));
        }

        xhtml.endDocument();
    }
}
```

> [!tip] 关键三件事
> 1. `parse()` 里**第一时间设 `CONTENT_TYPE`** 到 metadata
> 2. 用 `XHTMLContentHandler` 自动出 `<html><body>` 模板
> 3. 段落用 `<p>` 包，章节用 `<h1>~<h6>`，**结构化输出**让下游切分友好

## 4. 步骤 3：SPI 注册

新建文件 `src/main/resources/META-INF/services/org.apache.tika.parser.Parser`：

```
com.acme.tika.AcmeDocParser
```

**就一行**，类的全限定名。Tika 启动时通过 `ServiceLoader` 自动加载。

## 5. 验证

```java
public class TestAcme {
    public static void main(String[] args) throws Exception {
        Parser parser = new AutoDetectParser();
        BodyContentHandler h = new BodyContentHandler(-1);
        Metadata m = new Metadata();
        ParseContext ctx = new ParseContext();
        ctx.set(Parser.class, parser);

        try (InputStream in = Files.newInputStream(Path.of("test.acme"))) {
            parser.parse(in, h, m, ctx);
        }
        System.out.println("=== meta ===");
        for (String n : m.names()) System.out.println(n + "=" + m.get(n));
        System.out.println("=== text ===");
        System.out.println(h.toString());
    }
}
```

如果只识别成 `application/octet-stream`，**MIME 定义文件没生效**：

- 路径是否精确 `org/apache/tika/mime/custom-mimetypes.xml`
- jar 是否真的在 classpath
- 验证：`java -jar tika-app.jar --list-supported-types | grep acme`

## 6. 处理嵌入资源

如果你的格式内部有嵌入文档（例如自定义"邮件"格式有附件），要回调 `EmbeddedDocumentExtractor`：

```java
import org.apache.tika.extractor.EmbeddedDocumentExtractor;
import org.apache.tika.extractor.EmbeddedDocumentUtil;

EmbeddedDocumentExtractor edx =
    EmbeddedDocumentUtil.getEmbeddedDocumentExtractor(context);

for (Attachment a : doc.getAttachments()) {
    Metadata embMeta = new Metadata();
    embMeta.set(TikaCoreProperties.RESOURCE_NAME_KEY, a.fileName());
    embMeta.set(Metadata.CONTENT_TYPE, a.guessedMime());

    if (edx.shouldParseEmbedded(embMeta)) {
        try (InputStream inEmb = a.openStream()) {
            edx.parseEmbedded(inEmb, xhtml, embMeta, true);
        }
    }
}
```

这样附件会按规则递归解 / 落盘。详见 [[12 - 嵌入式文档与递归解析]]。

## 7. 用 `tika-config.xml` 替换内置 Parser

不想走 SPI 全局注册？只对某个 Tika 实例生效：

```xml
<properties>
    <parsers>
        <parser class="org.apache.tika.parser.DefaultParser">
            <parser-exclude class="com.acme.tika.AcmeDocParser"/>
        </parser>
        <parser class="com.acme.tika.AcmeDocParser">
            <mime>application/x-acme-doc</mime>
            <params>
                <param name="strictMode" type="bool">true</param>
            </params>
        </parser>
    </parsers>
</properties>
```

要让 `<params>` 生效，Parser 需要实现 `Initializable` 或 `ConfigurableParser` 接口（标 `@Field` 注解）：

```java
import org.apache.tika.config.Field;

public class AcmeDocParser extends AbstractParser {
    @Field private boolean strictMode = false;
    // setter / getter
}
```

## 8. 替换 / 增强一个 Detector

```java
package com.acme.tika;

import org.apache.tika.detect.Detector;
import org.apache.tika.metadata.Metadata;
import org.apache.tika.mime.MediaType;
import java.io.InputStream;

public class AcmeDetector implements Detector {
    @Override
    public MediaType detect(InputStream input, Metadata metadata) {
        // 你的检测逻辑
        return MediaType.application("x-acme-doc");
    }
}
```

SPI：`META-INF/services/org.apache.tika.detect.Detector` 写入 `com.acme.tika.AcmeDetector`。

## 9. 把自定义 Parser 打包发布

最小 Maven 项目结构：

```
acme-tika/
├── pom.xml
└── src/main/
    ├── java/com/acme/tika/AcmeDocParser.java
    └── resources/
        ├── META-INF/services/
        │   └── org.apache.tika.parser.Parser
        └── org/apache/tika/mime/
            └── custom-mimetypes.xml
```

```xml
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-core</artifactId>
    <version>3.2.3</version>
    <scope>provided</scope>
</dependency>
```

下游用户只要把你的 jar 放到 classpath，Tika 就识别新格式。

## 10. 高级模式

### 10.1 实现 `Initializable` 做启动校验

```java
import org.apache.tika.config.Initializable;
import org.apache.tika.config.InitializableProblemHandler;
import org.apache.tika.exception.TikaConfigException;

public class AcmeDocParser extends AbstractParser implements Initializable {

    @Override
    public void initialize(Map<String, Param> params) {
        // 启动时一次性加载词典、ML 模型
    }

    @Override
    public void checkInitialization(InitializableProblemHandler h) throws TikaConfigException {
        if (!Files.exists(Path.of("dict.bin"))) {
            h.handleInitializableProblem("acme-parser",
                "字典文件 dict.bin 不存在");
        }
    }
}
```

### 10.2 委派到底层第三方库

如果你的格式有现成 Java 库（如 protobuf）：

```java
@Override
public void parse(...) throws ... {
    MyDoc doc = MyDoc.parseFrom(stream);
    metadata.set(Metadata.CONTENT_TYPE, "application/x-acme-doc");
    metadata.set(TikaCoreProperties.TITLE, doc.getTitle());

    XHTMLContentHandler xhtml = new XHTMLContentHandler(handler, metadata);
    xhtml.startDocument();
    for (String section : doc.getSectionsList()) {
        xhtml.startElement("p");
        xhtml.characters(section);
        xhtml.endElement("p");
    }
    xhtml.endDocument();
}
```

## 11. 测试自己的 Parser

```java
class AcmeDocParserTest {

    private final Parser parser = new AcmeDocParser();

    @Test
    void parses_simple_doc() throws Exception {
        String content = "ACME1\n@TITLE: Hello\n@AUTHOR: alice\n这是一段。\n";
        InputStream in = new ByteArrayInputStream(content.getBytes(UTF_8));

        BodyContentHandler h = new BodyContentHandler(-1);
        Metadata m = new Metadata();
        parser.parse(in, h, m, new ParseContext());

        assertEquals("Hello", m.get(TikaCoreProperties.TITLE));
        assertEquals("alice", m.get(TikaCoreProperties.CREATOR));
        assertTrue(h.toString().contains("这是一段"));
    }

    @Test
    void registered_via_spi() {
        AutoDetectParser ap = new AutoDetectParser();
        assertTrue(ap.getParsers().values().stream()
            .anyMatch(p -> p instanceof AcmeDocParser));
    }
}
```

## 12. 上游贡献清单

如果你做的是**公共格式**，欢迎贡献给 Apache Tika：

1. 在 `tika-mimetypes.xml` 加 MIME 定义
2. 在 `tika-parsers/tika-parsers-xxx-module` 新建 Parser
3. 写 JUnit + 测试样例（放 `tika-parsers-test-resources`）
4. 提 PR https://github.com/apache/tika

社区惯例：**Parser 一律 SAX 风格 + 输出 XHTML + 不引入巨型依赖**。

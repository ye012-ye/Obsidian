# 09 · 语言检测

> [!info] 上一篇 / 下一篇
> ← [[08 - 文件类型检测 Detector]]　|　→ [[10 - OCR 集成 Tesseract]]

Tika 把语言检测抽出成独立模块，并支持多个**后端实现**：

| 实现 | 坐标 | 支持语言 | 备注 |
|---|---|---|---|
| **Optimaize**（推荐） | `tika-langdetect-optimaize` | ~70 种 | 默认选，离线 |
| **Lingo24** | `tika-langdetect-lingo24` | ~250 种 | 需要 API Key |
| **OpenNLP** | `tika-langdetect-opennlp` | 100+ | 用 OpenNLP 训练模型 |
| **MITLL TextCat** | `tika-langdetect-mitll-text` | 多 | 经典 N-gram |

## 1. 引入 Optimaize（最常用）

```xml
<dependency>
    <groupId>org.apache.tika</groupId>
    <artifactId>tika-langdetect-optimaize</artifactId>
    <version>3.2.3</version>
</dependency>
```

## 2. 简单用法

```java
import org.apache.tika.language.detect.LanguageDetector;
import org.apache.tika.language.detect.LanguageResult;

LanguageDetector ld = LanguageDetector.getDefaultLanguageDetector().loadModels();

String text = "Bonjour, comment allez-vous aujourd'hui ?";
LanguageResult r = ld.detect(text);

System.out.println(r.getLanguage());     // fr
System.out.println(r.getConfidence());   // HIGH / MEDIUM / LOW / NONE
System.out.println(r.getRawScore());     // 0.0 - 1.0
```

## 3. 流式增量检测（长文档）

文档很长时不必一次性传：

```java
LanguageDetector ld = LanguageDetector.getDefaultLanguageDetector().loadModels();
ld.reset();
try (BufferedReader r = Files.newBufferedReader(Path.of("long.txt"))) {
    String line;
    while ((line = r.readLine()) != null) {
        ld.addText(line);
        if (ld.hasEnoughText()) break;     // 已经够判断了
    }
}
LanguageResult result = ld.detect();
```

## 4. 拿到 Top N 候选

```java
List<LanguageResult> top = ld.detectAll();
top.stream()
   .limit(3)
   .forEach(lr -> System.out.printf("%s : %.2f%n", lr.getLanguage(), lr.getRawScore()));
```

## 5. 限定候选语言

只在中英之间挑：

```java
import org.apache.tika.language.detect.LanguageNames;

LanguageDetector ld = LanguageDetector.getDefaultLanguageDetector().loadModels();
ld.setPriors(Map.of("en", 0.5, "zh", 0.5));   // 先验概率
```

## 6. 结合解析 — 自动写到 Metadata

> Tika 在解析时**默认会调用 `Tika.parseToString` 拿到文本后再过一遍 LanguageDetector**（在配置启用时）。但显式做更可控：

```java
String text = new Tika().parseToString(file);   // 抽文本
LanguageResult lr = LanguageDetector.getDefaultLanguageDetector().loadModels().detect(text);
metadata.set(TikaCoreProperties.LANGUAGE, lr.getLanguage());
```

## 7. CLI 检测

```bash
java -jar tika-app-3.2.3.jar --language report.pdf
# en
```

## 8. 准确率与样本量

| 文本长度 | 准确率（Optimaize） |
|---|---|
| < 50 字符 | 不稳定，可能误判 |
| 50–200 | 中等 |
| > 200 | 高 |
| > 1000 | 接近完美 |

> [!warning] 短文本误判
> 像 `"OK"`、`"123"`、纯人名、纯链接 这种**判什么都不准**。可以先用 `hasEnoughText()` 判断够不够。

## 9. 中文判断的坑

- **简体 vs 繁体**：Optimaize 把它们都归为 `zh`。要区分简繁，用 OpenCC 或额外的字符统计。
- **混合语言段落**：会输出主导语言。要分段标注，把文档按段落切，每段单独 `detect`。

## 10. 实战：邮件路由

```java
public class EmailRouter {
    private final LanguageDetector ld =
        LanguageDetector.getDefaultLanguageDetector().loadModels();

    public String routeQueue(String body) {
        LanguageResult r = ld.detect(body);
        if (!r.isReasonablyCertain()) return "queue.unknown";
        return switch (r.getLanguage()) {
            case "zh" -> "queue.cn";
            case "ja" -> "queue.jp";
            case "ko" -> "queue.kr";
            case "en" -> "queue.en";
            default   -> "queue.other";
        };
    }
}
```

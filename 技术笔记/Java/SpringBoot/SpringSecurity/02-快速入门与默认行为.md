# 02-快速入门与默认行为

## 1. 最小依赖

Spring Boot 项目最常用的是 starter：

```xml
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
```

Gradle：

```groovy
dependencies {
    implementation "org.springframework.boot:spring-boot-starter-security"
}
```

Spring Boot 会通过 BOM 管理 Spring Security 版本，通常不需要手动写 Spring Security 版本号。

## 2. 加依赖后默认发生什么

只要 classpath 中有 Spring Security，Spring Boot 自动配置会启用基础安全能力。典型表现：

1. 所有 HTTP 请求默认都需要认证。
2. 自动生成一个用户名为 `user` 的内存用户。
3. 启动日志里打印随机密码。
4. 浏览器访问页面会看到默认登录页。
5. API 请求可能触发 HTTP Basic。
6. 默认启用 CSRF 防护。
7. 默认添加若干安全响应头。

启动日志里会出现类似：

```text
Using generated security password: 8d4d3f3a-xxxx-xxxx-xxxx-xxxxxxxx
```

这只是开发期默认密码，不应该用于生产。

## 3. 写一个最小 Controller

```java
@RestController
public class HelloController {

    @GetMapping("/public/hello")
    public String publicHello() {
        return "public";
    }

    @GetMapping("/api/hello")
    public String privateHello(Authentication authentication) {
        return "hello " + authentication.getName();
    }
}
```

如果只加依赖不写配置，`/public/hello` 和 `/api/hello` 都会要求登录。

## 4. 最小 SecurityFilterChain

现代 Spring Security 推荐显式声明 `SecurityFilterChain` Bean：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(authorize -> authorize
                .requestMatchers("/public/**").permitAll()
                .anyRequest().authenticated()
            )
            .formLogin(Customizer.withDefaults())
            .httpBasic(Customizer.withDefaults());

        return http.build();
    }
}
```

这段配置的含义：

1. `/public/**` 不需要登录。
2. 其他所有请求都必须登录。
3. 浏览器可以使用表单登录。
4. 命令行或 API 客户端可以使用 HTTP Basic。

## 5. 配置固定开发用户

开发阶段可以在配置文件里指定用户：

```yaml
spring:
  security:
    user:
      name: admin
      password: admin123
      roles:
        - ADMIN
```

注意：

1. 这只适合 Demo 或本地开发。
2. 生产环境应该来自数据库、LDAP、OAuth2 Provider 或企业统一身份系统。
3. 如果你自己声明了 `UserDetailsService` Bean，Spring Boot 的默认用户自动配置通常会退场。

## 6. 自定义内存用户

```java
@Bean
UserDetailsService userDetailsService(PasswordEncoder passwordEncoder) {
    UserDetails admin = User.builder()
        .username("admin")
        .password(passwordEncoder.encode("123456"))
        .roles("ADMIN")
        .build();

    UserDetails user = User.builder()
        .username("user")
        .password(passwordEncoder.encode("123456"))
        .roles("USER")
        .build();

    return new InMemoryUserDetailsManager(admin, user);
}

@Bean
PasswordEncoder passwordEncoder() {
    return PasswordEncoderFactories.createDelegatingPasswordEncoder();
}
```

`roles("ADMIN")` 会自动加上 `ROLE_` 前缀，最终权限是 `ROLE_ADMIN`。

## 7. 登录和登出

表单登录：

```java
http.formLogin(form -> form
    .loginPage("/login")
    .loginProcessingUrl("/login")
    .defaultSuccessUrl("/dashboard", true)
    .failureUrl("/login?error")
    .permitAll()
);
```

登出：

```java
http.logout(logout -> logout
    .logoutUrl("/logout")
    .logoutSuccessUrl("/login?logout")
    .invalidateHttpSession(true)
    .deleteCookies("JSESSIONID")
);
```

如果使用默认登录页，可以不配置 `loginPage`。

## 8. 401 和 403

| 状态码 | 含义 | 常见原因 |
|---|---|---|
| 401 Unauthorized | 未认证 | 没登录、Token 无效、Basic 凭证错误 |
| 403 Forbidden | 已认证但无权限 | 登录了，但缺少角色、权限或 CSRF Token |

排查思路：

1. 先看是否进入认证逻辑。
2. 再看 `Authentication` 是否存在。
3. 再看 `Authentication#getAuthorities()` 是否包含需要的权限。
4. 如果是 POST/PUT/DELETE 返回 403，还要检查 CSRF。

## 9. 配置顺序很重要

授权规则按声明顺序匹配。更具体的规则应该写在前面，兜底规则写在最后。

正确：

```java
.authorizeHttpRequests(authorize -> authorize
    .requestMatchers("/admin/**").hasRole("ADMIN")
    .requestMatchers("/public/**").permitAll()
    .anyRequest().authenticated()
)
```

错误：

```java
.authorizeHttpRequests(authorize -> authorize
    .anyRequest().authenticated()
    .requestMatchers("/public/**").permitAll()
)
```

`anyRequest()` 是兜底规则，放前面会让后面的规则失去意义。

## 10. 入门阶段不要做的事

1. 不要全局 `csrf.disable()` 后就不管原因。
2. 不要把密码明文写入数据库。
3. 不要把 JWT 当成“万能登录方案”。
4. 不要混用 `hasRole("ADMIN")` 和数据库里的 `ADMIN`、`ROLE_ADMIN` 而不理解前缀。
5. 不要用 `web.ignoring()` 放行业务接口，除非明确知道它会绕过整个安全过滤器链。
6. 不要用过时教程里的 `WebSecurityConfigurerAdapter` 作为新项目模板。

## 11. 本章小结

最小配置要记住：

```java
@Bean
SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    http
        .authorizeHttpRequests(authorize -> authorize
            .requestMatchers("/public/**").permitAll()
            .anyRequest().authenticated()
        )
        .formLogin(Customizer.withDefaults())
        .httpBasic(Customizer.withDefaults());

    return http.build();
}
```

这就是 Spring Security 入门的第一块地基。

# 心跳、空闲检测、重连与连接管理

> [!tip] 本章目标
> 长连接不是连上就完事。你要知道谁还活着、谁掉线了、谁需要重连。

## 为什么需要心跳

长连接会遇到：

1. 客户端断网但服务端没立刻感知。
2. NAT 或防火墙清理空闲连接。
3. 移动端切后台。
4. 服务端需要释放僵尸连接。

## IdleStateHandler

```java
ch.pipeline()
        .addLast(new IdleStateHandler(60, 30, 0))
        .addLast(new HeartbeatHandler());
```

参数常见含义：

1. 读空闲时间。
2. 写空闲时间。
3. 读写空闲时间。

## 服务端心跳 Handler

```java
public class HeartbeatHandler extends ChannelInboundHandlerAdapter {

    @Override
    public void userEventTriggered(ChannelHandlerContext ctx, Object evt) throws Exception {
        if (evt instanceof IdleStateEvent event) {
            if (event.state() == IdleState.READER_IDLE) {
                ctx.close();
                return;
            }
            if (event.state() == IdleState.WRITER_IDLE) {
                ctx.writeAndFlush(PingMessage.INSTANCE);
                return;
            }
        }
        super.userEventTriggered(ctx, evt);
    }
}
```

> [!warning] 心跳间隔不是越短越好
> 心跳太频繁会浪费流量和 CPU；太慢又不能及时发现断线。移动端、内网、公网、物联网设备要分别压测和调参。

## 连接管理

IM 系统常见结构：

```java
public class ConnectionRegistry {
    private final ConcurrentMap<Long, Channel> userChannels = new ConcurrentHashMap<>();

    public void online(Long userId, Channel channel) {
        userChannels.put(userId, channel);
    }

    public void offline(Long userId) {
        userChannels.remove(userId);
    }

    public Optional<Channel> find(Long userId) {
        return Optional.ofNullable(userChannels.get(userId));
    }
}
```

> [!danger] 多端登录要特别设计
> 一个用户可能手机、电脑、网页多端在线。不要简单用 `Map<userId, Channel>` 覆盖，除非业务明确只允许单端。

## 客户端重连

重连策略：

1. 断线后延迟重连。
2. 指数退避，避免瞬间打爆服务端。
3. 重连成功后重新认证。
4. 拉取离线消息或补偿状态。

## 本章小结

> [!success] 长连接三件套
> 心跳确认连接活性，连接表管理在线状态，重连补偿网络波动。少一件，线上体验都会抖。

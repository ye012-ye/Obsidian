# MongoDB 10 常见坑与面试题

## 常见坑

### 把 MongoDB 当关系型数据库用

表现：

- 每个对象都拆集合。
- 高频查询依赖 `$lookup`。
- 频繁跨集合事务。

建议：

- 先找聚合根。
- 经常一起读取的数据优先放一个文档。
- 允许合理冗余。

### 没有索引就上线

表现：

- 开发环境很快，生产数据一多就慢。
- 执行计划出现 `COLLSCAN`。
- CPU、磁盘 IO 飙高。

建议：

- 上线前列核心查询。
- 每个核心查询用 `explain("executionStats")`。
- 为过滤、排序、分页组合建复合索引。

### 索引太多

表现：

- 写入变慢。
- 磁盘占用大。
- 索引维护成本高。

建议：

- 删除重复索引。
- 删除长期不用索引。
- 不要每个字段都建索引。

### 文档无限增长

错误示例：

```javascript
{
  userId: "u1",
  logs: [
    ...
  ]
}
```

问题：

- 文档越来越大。
- 更新越来越慢。
- 数组查询复杂。

建议：

- 小数组可以嵌入。
- 无限增长数据拆 collection。
- 日志、流水、事件单独存。

### 深分页

```javascript
db.users.find().sort({ createdAt: -1 }).skip(100000).limit(20)
```

问题：`skip` 越大越慢。

建议：用游标式分页，配合 `createdAt + _id`。

### ObjectId 和 String 混用

错误：

```javascript
db.users.find({ _id: "665000000000000000000001" })
```

正确：

```javascript
db.users.find({ _id: ObjectId("665000000000000000000001") })
```

Spring 里也要统一 id 类型。

### 认证库写错

如果账号建在 `admin`，连接业务库要写：

```text
mongodb://root:123456@localhost:27017/appdb?authSource=admin
```

### 生产开放 27017

风险：

- 被扫描。
- 弱密码爆破。
- 数据泄露。
- 数据被删除。

建议：

- 不公网裸露。
- 开启认证。
- 安全组限制来源。
- 最小权限账号。
- 定期备份。

## 面试题

### MongoDB 和 MySQL 的区别

MongoDB 是文档型数据库，数据以 BSON 文档存储，schema 灵活，适合文档聚合、半结构化数据、高吞吐和横向扩展。MySQL 是关系型数据库，强在事务、关系建模、SQL、Join 和强一致。

### MongoDB 为什么查询快

不是所有查询天然快。MongoDB 快的前提是文档建模合理，把一次业务读取需要的数据聚合在一个文档里，减少 Join，再配合合适索引。没有索引或模型差，一样会慢。

### MongoDB 支持事务吗

支持。单文档写入天然原子，多文档事务需要 session，并依赖支持事务的部署形态。实际使用时优先通过文档建模减少跨集合事务。

### 嵌入和引用怎么选

一起读、一起写、生命周期一致、数量可控，用嵌入。独立生命周期、多处共享、数量无限增长、需要独立查询，用引用。

### 复合索引怎么设计

围绕查询模式设计。通常等值过滤字段在前，排序字段跟在后，范围字段谨慎放。设计完用 `explain("executionStats")` 验证。

### 为什么不建议深分页

`skip` 很大时，数据库仍然要跳过前面大量结果，性能越来越差。建议用游标式分页，比如 `createdAt + _id`。

### MongoDB 的 `_id` 是什么

每个文档默认主键。默认类型是 ObjectId，包含时间等信息。也可以自定义 `_id`，但要保证唯一和稳定。

### `$lookup` 能不能代替 Join

可以做关联查询，但不应该把它当成 MySQL Join 随意使用。高频 `$lookup` 往往说明文档模型需要调整，可以考虑嵌入或冗余快照。

### TTL 索引有什么用

TTL 索引用来自动删除过期文档，适合 session、验证码、临时任务、日志保留期。TTL 删除不是实时毫秒级，有后台扫描延迟。

### MongoDB 上线最重要检查什么

- 认证和权限。
- 备份恢复。
- 核心查询索引。
- 慢查询监控。
- 副本集高可用。
- 文档是否无限增长。
- 是否有深分页。
- 连接池和超时是否配置。

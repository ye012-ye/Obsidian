# ES 是什么与搜索思维

> [!info] 一句话理解
> Elasticsearch 是一个基于 Lucene 的分布式搜索与分析引擎，擅长近实时搜索、全文检索、日志分析、聚合统计和向量搜索。

## 先别急着写代码

如果你把 ES 当成“另一个数据库”，后面会非常痛苦。正确心智是：

| 问题 | MySQL 更像 | Elasticsearch 更像 |
|---|---|---|
| 主要目标 | 准确存储交易事实 | 快速找到相关内容 |
| 查询方式 | 精确条件、事务一致性 | 相关性、过滤、排序、聚合 |
| 数据模型 | 表、行、列、关系 | 索引、文档、字段 |
| 强项 | ACID、关联查询 | 全文搜索、日志、分析 |
| 常见误用 | 无 | 当主库、做复杂事务 |

> [!warning] 搜索不是查询
> 查询像问“身份证号是 123 的人是谁”；搜索像问“有没有关于 Java ES 教学、适合零基础、还要有趣的内容”。后者需要分词、召回、评分、排序。

## 图书馆比喻

想象你在图书馆找书：

1. 图书馆：一个 ES 集群。
2. 书架：index。
3. 一本书：document。
4. 书里的属性：field。
5. 目录卡片：倒排索引。
6. 图书管理员拆关键词：analyzer。
7. “最像你想要的书”排前面：score。

> [!example] 倒排索引的小故事
> 正排：第 1 本书包含 `Java、搜索、ES`。  
> 倒排：`Java -> 第1本、第3本、第9本`，`ES -> 第1本、第8本`。  
> 搜索快，是因为 ES 先找“词在哪些文档里”，而不是从第一篇文档扫到最后一篇。

## ES 适合什么

适合：

1. 商品搜索、文章搜索、站内搜索。
2. 日志检索、监控分析、链路排查。
3. 多条件筛选 + 排序 + 聚合。
4. 自动补全、搜索建议、高亮。
5. 海量数据近实时查询。

不适合：

1. 银行转账这类强事务主库。
2. 多表复杂 join 的核心系统。
3. 频繁单条强一致更新。
4. 小数据量简单查询却引入一整套搜索集群。

> [!danger] 生产红线
> ES 可以存数据，但不要轻易把 ES 当唯一事实来源。常见架构是 MySQL 做主库，ES 做搜索读模型，通过消息队列或同步任务保持最终一致。

## Java 后端看 ES 的主线

Java 后端学习 ES，不是背一堆 API，而是吃透四件事：

1. 怎么建索引：字段类型、分词器、分片、别名。
2. 怎么写数据：单条写入、批量写入、失败重试。
3. 怎么查数据：Query DSL、过滤、排序、分页、聚合。
4. 怎么上线：容量、性能、监控、重建索引、灰度切换。

## 本章小结

> [!success] 记住这句话
> MySQL 回答“哪条数据满足条件”，ES 回答“哪些内容最相关”。从这句话开始，你的 ES 学习会少走一半弯路。

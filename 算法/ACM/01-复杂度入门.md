# 01 · 时间复杂度入门

> 这是所有算法的"度量衡"。拿到题第一件事：**看数据范围推算复杂度**。

---

## 1. 什么是时间复杂度

**时间复杂度 ≈ 程序执行步数随输入 n 增长的速度**，用大 O 记号。

```
n=10       n=100       n=1000      n=10000
 │          │           │           │
 ▼          ▼           ▼           ▼
O(1)     1           1          1          1
O(log n) ~3          ~7         ~10        ~14
O(n)     10          100        1000       10000
O(n log n) ~33       ~664       ~10000    ~133000
O(n²)    100         10000      10⁶        10⁸      ❌
O(2ⁿ)    1024        爆炸       爆炸       爆炸      ❌
```

**记住这张表：**

```
n \ 复杂度  O(n)  O(n log n)   O(n²)     O(n³)
────────────────────────────────────────────────
10⁸         ✅      ✅           ❌        ❌
10⁷         ✅      ✅           ❌        ❌
10⁶         ✅      ✅           ❌        ❌
10⁵         ✅      ✅           ⚠(极限)   ❌
10⁴         ✅      ✅           ✅        ❌
10³         ✅      ✅           ✅        ⚠(极限)
500         ✅      ✅           ✅        ✅
```

## 2. 常见复杂度的样子（一眼识别）

### O(1) —— 常数
```java
int x = a[0] + a[1];           // 与 n 无关
```

### O(log n) —— 每次砍一半
```java
while (n > 0) n /= 2;          // 每次 n 减半
// 例子：二分查找、求 2 的幂
```

### O(n) —— 扫一遍
```java
for (int i = 0; i < n; i++) ...
```

### O(n log n) —— 排序、分治
```java
Arrays.sort(a);                // Java 的排序是 n log n
```

### O(n²) —— 双层循环
```java
for (int i = 0; i < n; i++)
    for (int j = 0; j < n; j++) ...
```

### O(2ⁿ) —— 枚举子集
```java
for (int s = 0; s < (1 << n); s++) ...  // n ≤ 20 才能跑
```

## 3. 推复杂度的"一秒看出来"技巧

### 技巧 1：**循环嵌套层数**
嵌套 k 层 → O(n^k)（如果每层都扫 n）。

```java
for (int i = 0; i < n; i++)             // 一层
    for (int j = i; j < n; j++)         // 嵌套
        for (int k = j; k < n; k++)     // 嵌套
            ...                         // O(n³)
```

### 技巧 2：**每次减半或翻倍 → log**
```java
while (n > 1) n /= 2;          // O(log n)
for (int i = 1; i <= n; i *= 2) ...     // O(log n)
```

### 技巧 3：**双指针 / 滑窗是 O(n)**（即使有嵌套 while）
```java
int l = 0;
for (int r = 0; r < n; r++) {
    while (l < r && 条件) l++;   // l 总共只会 ++ 至多 n 次
}
// 总体 O(n)，不是 O(n²)！
```

## 4. 空间复杂度

同理，**数组/容器用了多少元素就是多少空间**。校赛开 `int[]`/`long[]` 到 10⁶ 没问题，**10⁸ 会 MLE**。

Java 中估算：
- `int[]` → 每个 4 字节
- `long[]` → 每个 8 字节
- `boolean[]` → 每个 1 字节
- `Integer[]`、`HashMap` → 每个 ≥ 16 字节（别用大的！）

**10⁶ 个 int ≈ 4MB**，安全；超过 10⁷ 开数组就要谨慎。

---

## 5. 扩展进阶：为什么有些"两层循环"还是 O(n)

新手最容易误判：看到 `for + while` 就写 O(n²)，但若某个指针/元素**只会单向移动或只会进出一次**，总次数仍是 O(n)。

```
双指针 / 单调栈 / 单调队列的共性：

时间线:   0 1 2 3 4 5 ... n-1
右指针 r: → → → → → →       只向右走 n 次
左指针 l: →   → →   → →      也只向右走，最多 n 次
栈元素 i: push .... pop      每个元素最多 1 次入栈 + 1 次出栈

总操作数 ≤ 2n / 3n / 4n
所以复杂度仍是 O(n)，这叫**均摊分析**。
```

以后看到这类题，别只数"循环层数"，而要数：
- 每个指针最多走几次
- 每个元素最多进结构几次、出结构几次

这也是为什么滑窗、单调栈、单调队列经常能把暴力的 O(n²) 压到 O(n)。

---

## 6. 踩坑提醒 ⚠️

1. **Java 慢**：CF 评测里 Java 给 C++ 时限的 2 倍，但校赛常常不给——所以 O(n²) 刚好 10⁵ 可能 TLE。
2. **常数巨大的 O(n)** 可能跑不过常数小的 O(n log n)。
3. **HashMap 比数组慢 10 倍**：能用数组别用 HashMap。
4. **String 拼接用 StringBuilder**：`s += c` 是 O(n²)，StringBuilder 才是 O(n)。

## 7. 练习：判断这些代码的复杂度

```java
// A
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= i; j++) ...
```
> 答案：1 + 2 + ... + n = n(n+1)/2 → **O(n²)**

```java
// B
for (int i = 1; i <= n; i *= 2)
    for (int j = 0; j < n; j++) ...
```
> 答案：外层 log n 次，内层 n 次 → **O(n log n)**

```java
// C
int l = 0, sum = 0, ans = 0;
for (int r = 0; r < n; r++) {
    sum += a[r];
    while (sum > K) sum -= a[l++];
    ans = Math.max(ans, r - l + 1);
}
```
> 答案：虽然有 while 嵌套，但 l 只增不减，总 ≤ n 次 → **O(n)**（双指针典型！）

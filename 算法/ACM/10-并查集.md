# 10 · 并查集（Union-Find / DSU）

> 连通性问题的"万能钥匙"。代码短、威力大。

---

## 1. 要解决什么问题

**典型问题**：
- "A 和 B 是不是在同一组？"
- "把 A 和 B 合并到同一组。"
- "现在有几个组？"

比如：朋友圈、岛屿合并、图的连通块、Kruskal 最小生成树……

---

## 2. 基本思想

**每个组用一棵"树"表示**，每个节点记录自己的父亲；同组的节点都能沿父亲指针找到同一个"根"。

```
初始：每人自己一棵树
   1   2   3   4   5

union(1, 2)：让 2 的根指向 1
   1       3   4   5
   │
   2

union(3, 4)：
   1    3    5
   │    │
   2    4

union(1, 3)：
         1         5
        / \
       2   3
           │
           4

query(2, 4)：
  find(2) → 1   find(4) → 3 → 1   相同根 → 同组 ✓
```

---

## 3. 最基础代码

```java
int[] fa = new int[n + 1];
for (int i = 1; i <= n; i++) fa[i] = i;     // 自己是自己的父

int find(int x) {
    if (fa[x] == x) return x;
    return find(fa[x]);
}

void union(int x, int y) {
    int fx = find(x), fy = find(y);
    if (fx != fy) fa[fx] = fy;
}

boolean same(int x, int y) {
    return find(x) == find(y);
}
```

**复杂度**：单次 find 最坏 O(n)（退化成链）。需要优化。

---

## 4. 优化 1：路径压缩

每次 find 时，把沿途节点全部直接挂到根上。

```
find(4)：  1            1
          / \          /|\
         2   3        2 3 4
             │
             4
```

代码就一行改动：
```java
int find(int x) {
    if (fa[x] != x) fa[x] = find(fa[x]);  // 让自己直接挂到根
    return fa[x];
}
```

---

## 5. 优化 2：按秩合并 / 按大小合并

合并时，把"小树"挂到"大树"下，避免树变高。

```java
int[] size = new int[n + 1];
Arrays.fill(size, 1);

void union(int x, int y) {
    int fx = find(x), fy = find(y);
    if (fx == fy) return;
    if (size[fx] < size[fy]) { int t = fx; fx = fy; fy = t; }
    fa[fy] = fx;
    size[fx] += size[fy];
}
```

**两者结合后，单次操作几乎 O(1)**（严格 O(α(n))，α 是反阿克曼函数）。

## 6. 完整模板（背下来！）

```java
class DSU {
    int[] fa, size;
    DSU(int n) {
        fa = new int[n + 1];
        size = new int[n + 1];
        for (int i = 0; i <= n; i++) { fa[i] = i; size[i] = 1; }
    }
    int find(int x) {
        if (fa[x] != x) fa[x] = find(fa[x]);
        return fa[x];
    }
    boolean union(int x, int y) {
        int fx = find(x), fy = find(y);
        if (fx == fy) return false;
        if (size[fx] < size[fy]) { int t = fx; fx = fy; fy = t; }
        fa[fy] = fx;
        size[fx] += size[fy];
        return true;
    }
    boolean same(int x, int y) { return find(x) == find(y); }
}
```

使用：
```java
DSU dsu = new DSU(n);
dsu.union(1, 2);
dsu.same(1, 3);  // 查询
int groups = 0;
for (int i = 1; i <= n; i++) if (dsu.find(i) == i) groups++;  // 连通块数
```

## 7. 经典应用

### 7.1 朋友圈 / 连通块数（LC 547）

```java
int findCircleNum(int[][] M) {
    int n = M.length;
    DSU d = new DSU(n);
    for (int i = 0; i < n; i++)
        for (int j = i+1; j < n; j++)
            if (M[i][j] == 1) d.union(i, j);
    int cnt = 0;
    for (int i = 0; i < n; i++) if (d.find(i) == i) cnt++;
    return cnt;
}
```

### 7.2 Kruskal 最小生成树（极常见）

1. 所有边按权重升序
2. 逐条考虑，若两端不同组则加入 + union
3. 加够 n-1 条边为止

```java
// edges: int[][] {u, v, w}
Arrays.sort(edges, (a, b) -> a[2] - b[2]);
DSU d = new DSU(n);
long mstWeight = 0; int cnt = 0;
for (int[] e : edges) {
    if (d.union(e[0], e[1])) {
        mstWeight += e[2];
        if (++cnt == n - 1) break;
    }
}
```

### 7.3 动态连通性 / 图的实时维护

只"加边"是并查集拿手戏；"删边"难（需要离线处理或可撤销并查集）。

---

## 8. 判断"何时用并查集"

关键词：
- "是否在同一组 / 连通"
- "最少合并几次才能让所有点连通"
- "加一条边会不会形成环"
- "朋友圈 / 岛屿合并 / 图连通块数"
- "最小生成树"

---

## 9. 扩展进阶：带权并查集

普通并查集只维护"是不是同组"；带权并查集还能维护：
- 到根的距离
- 模 2 / 模 3 关系
- "谁吃谁" 这类相对关系

### 例：维护差值 `value[x] - value[y]`

设 `dist[x]` 表示 `x` 到父节点的权值差。
路径压缩时，顺手把差值也累加上去。

```
x --(+2)--> fa[x] --(+3)--> root
那么 x 到 root 的总差值 = 5
```

find 时：
```java
int find(int x) {
    if (fa[x] == x) return x;
    int p = fa[x];
    fa[x] = find(fa[x]);
    dist[x] += dist[p];
    return fa[x];
}
```

这类模型常见于：
- NOI / 洛谷 "食物链"
- 维护奇偶性、敌友关系
- 带差值约束的连通问题

如果题目不只是问"连不连"，还问"连着以后两者差多少 / 关系是什么"，就该想到带权并查集。

---

## 10. 踩坑提醒 ⚠️

1. **忘了初始化 fa[i] = i** → 所有节点父都是 0，乱套。
2. **递归 find 栈溢出**：n=10⁶ 的链式 union 可能炸栈，改成迭代版：
   ```java
   int find(int x) {
       int r = x;
       while (fa[r] != r) r = fa[r];
       while (fa[x] != r) { int nx = fa[x]; fa[x] = r; x = nx; }
       return r;
   }
   ```
3. **union 前要 find**：不能直接 `fa[x] = y`，必须 `fa[find(x)] = find(y)`。
4. **节点从 0 还是 1 开始**：数组大小别开错。

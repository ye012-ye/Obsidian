# 16 · 字符串算法：KMP、Trie、字符串哈希

> 字符串题最怕"看起来像模拟，实际上是板子题"。这一章补三个校赛高频工具。

---

## 1. KMP：高效字符串匹配

### 1.1 它解决什么问题

在文本串 `s` 中查找模式串 `p` 是否出现。

暴力匹配最坏会反复回退，`O(nm)`。
KMP 利用模式串自身信息，把它优化到 `O(n+m)`。

---

## 2. next / pi 数组在干什么

`pi[i]` 表示：

> `p[0..i]` 这个前缀中，最长的"相等前后缀"长度。

例如：
```
p = "ababaca"

前缀 "ababa" 的最长相等前后缀是 "aba"
长度 = 3
```

这意味着匹配失败时，不必从头来，可以跳到一个合理位置继续。

---

## 3. KMP 图解

```
文本 s:    a b a b a b a c a
模式 p:    a b a b a c a

前面匹配到：
           a b a b a
           a b a b a

下一位失败：
文本是 b，模式期待 c

暴力：模式串整体右移，从头比
KMP：利用 pi，直接知道前面有一段还能复用
```

本质上：
> 模式串自己告诉你，失败后能保留多少匹配成果。

---

## 4. KMP 模板

```java
int[] prefixFunction(String p) {
    int n = p.length();
    int[] pi = new int[n];
    for (int i = 1; i < n; i++) {
        int j = pi[i - 1];
        while (j > 0 && p.charAt(i) != p.charAt(j)) j = pi[j - 1];
        if (p.charAt(i) == p.charAt(j)) j++;
        pi[i] = j;
    }
    return pi;
}

List<Integer> kmpSearch(String s, String p) {
    int[] pi = prefixFunction(p);
    List<Integer> ans = new ArrayList<>();
    for (int i = 0, j = 0; i < s.length(); i++) {
        while (j > 0 && s.charAt(i) != p.charAt(j)) j = pi[j - 1];
        if (s.charAt(i) == p.charAt(j)) j++;
        if (j == p.length()) {
            ans.add(i - p.length() + 1);
            j = pi[j - 1];
        }
    }
    return ans;
}
```

## 5. Trie：前缀树

适合处理：
- 单词集合
- 前缀查询
- 字典类问题

```
插入 "cat", "car", "dog"

root
 ├─ c
 │   └─ a
 │       ├─ t
 │       └─ r
 └─ d
     └─ o
         └─ g
```

### 5.1 模板

```java
class TrieNode {
    TrieNode[] next = new TrieNode[26];
    boolean end;
}

class Trie {
    TrieNode root = new TrieNode();

    void insert(String s) {
        TrieNode cur = root;
        for (char c : s.toCharArray()) {
            int x = c - 'a';
            if (cur.next[x] == null) cur.next[x] = new TrieNode();
            cur = cur.next[x];
        }
        cur.end = true;
    }

    boolean search(String s) {
        TrieNode cur = root;
        for (char c : s.toCharArray()) {
            int x = c - 'a';
            if (cur.next[x] == null) return false;
            cur = cur.next[x];
        }
        return cur.end;
    }
}
```

## 6. 字符串哈希

### 6.1 用来干嘛

快速比较：
- 两个子串是否相同
- 某个串是否出现过
- 回文 / 去重 / 二分答案配哈希

思路：
把字符串看成一个多项式。

```
s = "abc"
hash = a * P^2 + b * P + c
```

预处理前缀哈希后，任意子串哈希可 `O(1)` 取出。

### 6.2 模板思路

```java
H[i] = H[i-1] * P + s[i]
pow[i] = P^i

hash(l..r) = H[r] - H[l-1] * pow[r-l+1]
```

---

## 7. 三者怎么选

```
查模式串出现位置          -> KMP
查单词前缀 / 字典树结构    -> Trie
快速比较很多子串是否相等   -> 字符串哈希
```

---

## 8. 踩坑提醒 ⚠️

1. **KMP 的 while 回退**别写错，核心是 `j = pi[j-1]`。
2. Trie 若字符集不是 26 个小写字母，结构要改。
3. **字符串哈希有碰撞风险**，比赛里最好用双哈希或 `long` 自然溢出哈希。
4. 子串哈希下标很容易错，建议统一 1-indexed。
5. KMP 是匹配板子，不适合所有字符串题，别见字符串就套。

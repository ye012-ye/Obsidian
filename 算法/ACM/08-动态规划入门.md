# 08 · 动态规划入门

> DP 是 ACM 的半壁江山。本章从"什么是 DP"讲起，带你入门背包、LIS、LCS、区间 DP。

---

## 1. 什么是 DP

**DP = 把大问题拆成重复子问题 + 记下子问题答案避免重复计算。**

三要素：
1. **状态**：`dp[i]` 或 `dp[i][j]` 代表什么（**这是最重要的！**）
2. **转移**：`dp[i]` 怎么从前面的状态推出来
3. **初值** & **答案**：边界怎么定，最后怎么取

---

## 2. 入门例：斐波那契

```
f(0) = 0
f(1) = 1
f(n) = f(n-1) + f(n-2)
```

直接递归 → O(2ⁿ)，因为重复计算很多：
```
           f(5)
         /      \
      f(4)      f(3)
     /   \      /  \
   f(3)  f(2) f(2) f(1)
   /  \  ...  ...
  f(2) f(1)
  ...
   ↑ 同一子问题算了无数次
```

DP 就是**把 f(2), f(3)... 记在数组里**：
```java
int[] dp = new int[n + 1];
dp[0] = 0; dp[1] = 1;
for (int i = 2; i <= n; i++) dp[i] = dp[i-1] + dp[i-2];
```
O(n) 了。

**这就是 DP 的本质：空间换时间，避免重复计算。**

## 3. 线性 DP

### 3.1 爬楼梯（LC 70）
每次爬 1 或 2 级，到 n 级楼梯的方法数。

**状态**：`dp[i]` = 爬到第 i 级的方法数
**转移**：`dp[i] = dp[i-1] + dp[i-2]`（从 i-1 爬一级，或从 i-2 爬两级）
**初值**：`dp[0]=1, dp[1]=1`

### 3.2 最大子数组和（LC 53，Kadane 算法）

**状态**：`dp[i]` = 以 a[i] 结尾的最大子数组和
**转移**：
```
dp[i] = max(a[i], dp[i-1] + a[i])
     （要么从自己开始，要么接着前面）
```
**答案**：`max(dp[i])`

```java
long dp = 0, ans = Long.MIN_VALUE;
for (int x : a) {
    dp = Math.max(x, dp + x);
    ans = Math.max(ans, dp);
}
```

图示：
```
a = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

dp: [-2, 1, -2, 4,  3, 5, 6,  1, 5]
                    ▲
                    答案 = 6（子数组 [4,-1,2,1]）
```

## 4. 0-1 背包（超重要！）

### 4.1 题面

n 件物品，第 i 件重 w[i]、价值 v[i]。背包容量 W。每件物品**最多选一次**。求最大价值。

### 4.2 状态设计

**`dp[i][j]` = 只考虑前 i 件物品、背包容量 j 时的最大价值。**

### 4.3 转移方程

对第 i 件物品，两个选择：
- **不拿**：`dp[i][j] = dp[i-1][j]`
- **拿**（前提 j ≥ w[i]）：`dp[i][j] = dp[i-1][j - w[i]] + v[i]`

取较大者：
```
dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])
```

### 4.4 ASCII 表格演示

```
n=3, W=4
w = [2, 1, 3]
v = [3, 2, 4]

dp[i][j]:   j=0   j=1   j=2   j=3   j=4
i=0 初始     0     0     0     0     0
i=1 (w=2,v=3) 0    0     3     3     3
i=2 (w=1,v=2) 0    2     3     5     5
i=3 (w=3,v=4) 0    2     3     5     6
                                     ▲
                                     答案 = 6（拿第 2、3 件）
```

### 4.5 代码

```java
int[][] dp = new int[n + 1][W + 1];
for (int i = 1; i <= n; i++) {
    for (int j = 0; j <= W; j++) {
        dp[i][j] = dp[i-1][j];
        if (j >= w[i-1])
            dp[i][j] = Math.max(dp[i][j], dp[i-1][j - w[i-1]] + v[i-1]);
    }
}
System.out.println(dp[n][W]);
```

### 4.6 空间优化：滚动到一维

观察：`dp[i][j]` 只依赖 `dp[i-1][j]` 和 `dp[i-1][j - w[i]]`。可用一维数组！
但 **j 必须倒序枚举**（防止同一件物品被用两次）。

```java
int[] dp = new int[W + 1];
for (int i = 0; i < n; i++) {
    for (int j = W; j >= w[i]; j--) {     // 倒序！
        dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
    }
}
```

## 5. 完全背包

每件物品可以拿**无限次**。

唯一区别：**j 正序枚举**（这次允许重复拿）。
```java
for (int i = 0; i < n; i++) {
    for (int j = w[i]; j <= W; j++) {     // 正序！
        dp[j] = Math.max(dp[j], dp[j - w[i]] + v[i]);
    }
}
```

**口诀**：0-1 倒着刷，完全正着刷。

## 6. 最长递增子序列（LIS，LC 300）

### 6.1 O(n²) DP

**状态**：`dp[i]` = 以 a[i] 结尾的最长递增子序列长度。
**转移**：`dp[i] = max(dp[j] + 1)`，其中 j < i 且 a[j] < a[i]；若没有这样的 j，则 dp[i]=1。

```java
int[] dp = new int[n];
Arrays.fill(dp, 1);
for (int i = 0; i < n; i++)
    for (int j = 0; j < i; j++)
        if (a[j] < a[i]) dp[i] = Math.max(dp[i], dp[j] + 1);
int ans = Arrays.stream(dp).max().getAsInt();
```

### 6.2 O(n log n) 神解法

维护一个"最小末尾值"数组 `tails`，每次二分找位置替换：

```java
int[] tails = new int[n];
int len = 0;
for (int x : a) {
    int pos = lowerBound(tails, 0, len, x);  // 找第一个 >= x 的位置
    tails[pos] = x;
    if (pos == len) len++;
}
return len;
```

图示（a = [10, 9, 2, 5, 3, 7, 101, 18]）：
```
处理 10：tails=[10]        len=1
处理  9：替换 10→9，tails=[9]      len=1
处理  2：替换 9 →2，tails=[2]      len=1
处理  5：追加，tails=[2,5]         len=2
处理  3：替换 5 →3，tails=[2,3]    len=2
处理  7：追加，tails=[2,3,7]       len=3
处理 101：追加，tails=[2,3,7,101]  len=4
处理 18：替换 101→18，tails=[2,3,7,18] len=4

答案 = 4
```
> `tails` 不是答案本身（它不一定是真实的最长序列），但它的**长度**就是 LIS。

## 7. 最长公共子序列（LCS，LC 1143）

两串 s, t，求**公共子序列**的最长长度。

**状态**：`dp[i][j]` = s 前 i 个字符和 t 前 j 个字符的 LCS 长度。
**转移**：
```
若 s[i] == t[j]：dp[i][j] = dp[i-1][j-1] + 1
否则：           dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

### 7.1 表格图解（s="abcde", t="ace"）

```
       ""  a  c  e
  ""    0  0  0  0
  a     0  1  1  1
  b     0  1  1  1
  c     0  1  2  2
  d     0  1  2  2
  e     0  1  2  3  ← 答案 = 3 ("ace")
```

```java
int[][] dp = new int[n + 1][m + 1];
for (int i = 1; i <= n; i++)
    for (int j = 1; j <= m; j++) {
        if (s.charAt(i - 1) == t.charAt(j - 1))
            dp[i][j] = dp[i-1][j-1] + 1;
        else
            dp[i][j] = Math.max(dp[i-1][j], dp[i][j-1]);
    }
return dp[n][m];
```

## 8. 区间 DP（入门）

**模型**：`dp[l][r]` 表示区间 [l, r] 的最优解，转移通过**枚举中间断点** k。

### 8.1 例：石子合并

n 堆石子，每次合并相邻两堆代价为石子总和，求合并成一堆的最小代价。

**状态**：`dp[l][r]` = 合并 [l..r] 的最小代价
**转移**：
```
dp[l][r] = min over k in [l, r-1] of:
           dp[l][k] + dp[k+1][r] + sum(l..r)
```

**枚举顺序**：必须**先小区间，后大区间**，所以外层是区间长度：
```java
long[] s = 前缀和;
int[][] dp = new int[n+2][n+2];
for (int len = 2; len <= n; len++) {
    for (int l = 1; l + len - 1 <= n; l++) {
        int r = l + len - 1;
        dp[l][r] = Integer.MAX_VALUE;
        for (int k = l; k < r; k++) {
            dp[l][r] = Math.min(dp[l][r],
                       dp[l][k] + dp[k+1][r] + (int)(s[r] - s[l-1]));
        }
    }
}
```

复杂度 O(n³)，n ≤ 300 可过。

## 9. DP 的思考套路

遇到题目想 DP：
1. **状态定义**：`dp[...]` 代表什么？
   - 常见维度：**前 i 个**、**位置 i**、**区间 [l, r]**、**分组数 j**、**是否选过**
2. **转移**：从哪里来？
3. **边界**：`dp[0]`, `dp[i][0]` 等。
4. **答案**：`dp[n]`、`max(dp)` 还是其他？
5. **枚举顺序**：要保证转移时依赖的状态已经算好。

**DP 的"困"在于想状态——多做题培养感觉。**

## 10. 扩展进阶：状态设计路线图

很多人不是不会写转移，而是不知道 `dp[...]` 该怎么定义。可以先按题型套这张图：

```
题目在问什么？
├─ 以某个位置结尾/开头最优？ → 线性 DP：dp[i]
├─ 前 i 个物品、容量 j？     → 背包 DP：dp[i][j] / dp[j]
├─ 两个序列怎么匹配？        → 双序列 DP：dp[i][j]
├─ 一个区间如何合并？        → 区间 DP：dp[l][r]
├─ 树上选或不选？            → 树形 DP：dp[u][state]
└─ 状态很小、要枚举集合？     → 状压 DP：dp[mask]
```

再问自己两句：
1. "最后一步发生了什么？"
2. "如果最后一步确定，前面剩下的是不是同类子问题？"

### 例：树形 DP 的味道

以"打家劫舍 III"为例：
- `dp[u][0]` = 不选 u 时，u 子树最大收益
- `dp[u][1]` = 选 u 时，u 子树最大收益

```
        u
      /   \
   child1 child2

选 u    → 子节点都不能选
不选 u  → 子节点可选可不选，取更优
```

看到"树上选点 + 相邻有限制"时，就该往树形 DP 想。

## 11. 踩坑提醒 ⚠️

1. **开 long**：DP 结果常常很大（如背包价值和）。
2. **下标 0-indexed 还是 1-indexed**：混用会边界错。建议 1-indexed，用 `dp[0][*]=0` 哨兵。
3. **空间优化别抢**：先写清二维对的 DP，再滚动数组优化。
4. **输入可能需要离散化**：值域大但个数少时。
5. **初始化**：求最小用 INF，求最大用 -INF，求方案数用 0/1。别漏。

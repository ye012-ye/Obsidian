# 09 · 贪心入门

> 贪心：每一步都选当下最优的。简单但"证明"难——如何相信它对？

---

## 1. 贪心的本质

**贪心算法**：每一步做"当前看起来最好"的选择，希望最终全局最优。

**它和 DP 的区别**：
- DP：试遍所有可能，选最优
- 贪心：只走一条"看起来最优"的路径

**什么时候贪心对？** 当问题有"贪心选择性质"—— 当前最优 + 子问题最优 = 全局最优。

## 2. 入门例：分发饼干（LC 455）

n 个孩子胃口 g，m 块饼干 s，每个孩子最多 1 块饼干。最多满足几个？

**贪心**：两者都排序，**最小的饼干先给胃口最小的孩子**。

```
g = [1, 2, 3]   （孩子胃口）
s = [1, 1, 2, 3] （饼干大小）

排序后：
g: 1 2 3
s: 1 1 2 3

双指针：
  i=0, j=0: s=1 >= g=1 ✓, i++, j++, count=1
  i=1, j=1: s=1 <  g=2 ✗, j++
  i=1, j=2: s=2 >= g=2 ✓, i++, j++, count=2
  i=2, j=3: s=3 >= g=3 ✓, i++, j++, count=3

答案 = 3
```

```java
int findContentChildren(int[] g, int[] s) {
    Arrays.sort(g); Arrays.sort(s);
    int i = 0, j = 0;
    while (i < g.length && j < s.length) {
        if (s[j] >= g[i]) i++;
        j++;
    }
    return i;
}
```

## 3. 经典：区间调度

### 3.1 问题：会议室安排

n 个会议 [s_i, e_i]，问最多能安排几个不冲突的。

**贪心策略**：**按结束时间 e 升序**，依次选，每次选当前能选且 e 最小的。

### 3.2 为什么是"按 e 升序"？

直觉：**结束得越早，留给后面的时间越多**。

```
会议:   [1,3] [2,4] [3,5] [4,7] [6,8]

按 e 升序：
[1,3] → 选，当前 end = 3
[2,4] → s=2 < 3，冲突，跳过
[3,5] → s=3 = 3，冲突（若要求严格后续则跳过）
[4,7] → s=4 > 3，选，end = 7
[6,8] → s=6 < 7，冲突

答案 = 2 （[1,3] 和 [4,7]）
```

### 3.3 代码

```java
int maxMeetings(int[][] meet) {
    Arrays.sort(meet, (a, b) -> a[1] - b[1]);
    int cnt = 0, end = Integer.MIN_VALUE;
    for (int[] x : meet) {
        if (x[0] >= end) { cnt++; end = x[1]; }
    }
    return cnt;
}
```

### 3.4 为什么不是"按开始时间 s 升序"？
反例：
```
[0, 100]  开始最早，按 s 选它 → 只能选 1 个
[1, 2]
[3, 4]
[5, 6]
```
正确贪心（按 e）能选 [1,2], [3,4], [5,6] 三个。

## 4. 经典：合并果子 / 哈夫曼

n 堆果子，每次可合并两堆，代价 = 两堆之和。求最小总代价。

**贪心**：每次**合并最小的两堆**。

**工具**：小根堆。

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();
for (int x : a) pq.offer(x);
long ans = 0;
while (pq.size() > 1) {
    int x = pq.poll(), y = pq.poll();
    ans += x + y;
    pq.offer(x + y);
}
return ans;
```

**图解**：
```
[1, 2, 3, 9]

第 1 次：合并 1+2 = 3，代价 3
         堆：[3, 3, 9]

第 2 次：合并 3+3 = 6，代价 6
         堆：[6, 9]

第 3 次：合并 6+9 = 15，代价 15
         堆：[15]

总代价 = 3 + 6 + 15 = 24
```

## 5. 经典：跳跃游戏（LC 55）

给 a[]，a[i] 表示从位置 i 最多能跳几步。能否从 0 跳到 n-1？

**贪心**：维护**当前能跳到的最远位置** maxReach。

```java
int maxReach = 0;
for (int i = 0; i < n; i++) {
    if (i > maxReach) return false;      // 卡住了
    maxReach = Math.max(maxReach, i + a[i]);
}
return true;
```

图示：
```
a = [2, 3, 1, 1, 4]
      ↑           
i=0: 在 0，maxReach = max(0, 0+2) = 2
i=1: 在 1 ≤ 2，maxReach = max(2, 1+3) = 4
i=2: 在 2 ≤ 4，maxReach = max(4, 2+1) = 4
i=3: 在 3 ≤ 4，maxReach = max(4, 3+1) = 4
i=4: 在 4 ≤ 4，到达终点 ✓
```

## 6. 贪心的"证明"方法

这是最难的部分——新手常犯的错是"想当然地贪心"。建议两种手段：

### 6.1 **举反例** 证否
如果你想到一种贪心策略，先**手算几个小数据**。如果有反例，立刻换思路。

### 6.2 **交换论证**（严谨证明）
假设最优解不包含你的贪心选择，证明交换一下不会更差。

举例（会议调度）：
- 贪心选了 e 最早的会议 m₁
- 假设最优解没选 m₁，选了 m'
- 因为 e(m₁) ≤ e(m')，把 m' 换成 m₁ 后，最优解依然合法且数量不变 → 矛盾。

## 7. 常见贪心模型速查表

| 问题类型 | 贪心策略 |
|---------|---------|
| 区间不冲突最多数量 | 按 e 升序 |
| 区间全覆盖最少数量 | 按 s 升序 + 每次跳到能覆盖右端最远的 |
| 合并最小代价 | 堆 + 每次合最小两个（哈夫曼） |
| 分饼干 / 分组 | 双排序 + 双指针 |
| 跳跃 | 维护 maxReach |
| 加油站 | 总和 ≥ 0 时答案存在 |
| 任务调度（每个任务截止） | 按截止时间排序 + 反悔堆 |

## 8. 反悔贪心（进阶）

有时"当下贪"错了怎么办？**允许反悔**：先拿再扔。

例：任务调度。n 个任务，每个 `(截止时间 d_i, 收益 p_i)`，每个任务 1 单位时间。最大化收益。

**思路**：按 d 升序，能选就选进堆；若当前时间 > 堆大小，弹出堆中收益最小的（反悔）。

```java
Arrays.sort(tasks, (a, b) -> a[0] - b[0]);   // 按 d
PriorityQueue<Integer> pq = new PriorityQueue<>();  // 小根堆，按 p
for (int[] t : tasks) {
    pq.offer(t[1]);
    if (pq.size() > t[0]) pq.poll();
}
long ans = 0;
while (!pq.isEmpty()) ans += pq.poll();
```

## 9. 扩展进阶：区间覆盖型贪心

除了"选最多个不冲突区间"，另一大类高频题是：

> 用尽量少的区间，覆盖目标区间 `[L, R]`

经典策略：
1. 先按左端点升序排序
2. 扫描所有左端点 `<= 当前覆盖右端` 的区间
3. 在这些区间里，选一个**右端最远**的
4. 覆盖范围一口气跳过去

```
目标覆盖 [1, 10]

当前已覆盖到 1
能接上的区间：
[1,4] [1,5] [2,6]   → 选右端最远的 [2,6]

当前已覆盖到 6
能接上的区间：
[4,7] [5,9] [6,10]  → 选 [6,10]

两段搞定
```

为什么不是看见能用的就立刻选？
因为**当前能接上的区间里，右端越远，留给后面的空间越大**。

这类题常见于：
- 最少线段覆盖
- 视频拼接（LC 1024）
- 灌溉 / 雷达覆盖 / 路灯覆盖

## 10. 踩坑提醒 ⚠️

1. **贪心不总是对**，遇到稍复杂的题先考虑 DP / 搜索。
2. 排序的 key 选错就全错（**按 e 还是按 s？**）。
3. 证明比思路更难——**举反例** 是验证快速的方法。
4. Java 排序要注意：`Arrays.sort(int[])` 是随机快排（对抗数据可能慢），`Integer[]`/`ArrayList` 用归并。
5. 浮点数比较别用 `==`，用 `Math.abs(a-b) < eps`。

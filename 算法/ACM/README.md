## 🎯 校赛目标复杂度参考

| 数据范围 n | 允许的复杂度 | 典型算法 |
|---|---|---|
| n ≤ 10         | O(n!) O(2ⁿ)     | 全排列、状态压缩 |
| n ≤ 20         | O(2ⁿ)           | 枚举子集 |
| n ≤ 500        | O(n³)           | 三重循环、Floyd |
| n ≤ 5000       | O(n²)           | 两重循环、简单 DP |
| n ≤ 10⁵        | O(n log n)      | 排序、二分、堆、线段树 |
| n ≤ 10⁶ ~ 10⁷ | O(n)            | 双指针、前缀和、单调队列 |
| n ≤ 10⁹        | O(log n) O(√n) | 二分、数论、矩阵快速幂 |

> 牛客/CF 一般 1 秒约能跑 **10⁸** 次简单操作，Java 要打 **3~5 折**，按 **2~3×10⁷** 估。

## 🛠 使用方式

1. 先按顺序过 `01 ~ 10`，把基础篇做成肌肉记忆。
2. `11` 在比赛前一周反复看，它不是知识点章，而是临场手册。
3. 基础稳了以后，再补 `12 ~ 18`，这部分通常就是"能不能冲前排"的差距。
4. 在 `D:\java\code\leetcode\src\main\java\com\ye\leetcode\acm\` 下开对应文件夹练题。

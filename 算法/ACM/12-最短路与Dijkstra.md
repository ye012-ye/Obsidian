# 12 · 最短路与 Dijkstra

> 只要题目出现"最少代价"、"最短时间"、"最少花费"，大概率就是最短路。

---

## 1. 什么是最短路

图上每条边都有代价，要求从起点 `s` 到终点 `t` 的最小总代价。

```
     2
1 ─────► 2
│        │
│5       │1
▼        ▼
3 ─────► 4
    1

从 1 到 4：
1 -> 2 -> 4  代价 = 2 + 1 = 3
1 -> 3 -> 4  代价 = 5 + 1 = 6

最短路 = 3
```

---

## 2. 图怎么存

带权图一般用邻接表：

```java
List<List<int[]>> g = new ArrayList<>();
for (int i = 0; i <= n; i++) g.add(new ArrayList<>());

// 有向边 u -> v，权重 w
g.get(u).add(new int[]{v, w});

// 无向边要加两次
g.get(u).add(new int[]{v, w});
g.get(v).add(new int[]{u, w});
```

每个 `int[]` 存 `{to, w}`。

## 3. Dijkstra 的核心思想

**前提：边权非负。**

思路：
1. 维护 `dist[x]` = 起点到 x 的当前最短距离
2. 每次从还没确定的点里，取出 `dist` 最小的那个
3. 用它去更新邻居

为什么对？
> 因为所有边权都非负，当前最小的那个点不可能再被"绕一圈"变得更小。

```
已确定区      未确定区
[ 起点 ... 最近点 ] | [ 其他点 ]

每次把"离起点最近"的那个未确定点拉进来
```

这是一种非常典型的**贪心**。

## 4. 图解：堆优化 Dijkstra

```
起点 1

1 --2--> 2 --1--> 4
 \       |
  5      3
   \     v
    ---> 3 --1--> 4
```

初始化：
```
dist[1]=0，其余=INF
堆 = [(0,1)]
```

弹出 `(0,1)`：
```
更新 2：dist[2]=2
更新 3：dist[3]=5
堆 = [(2,2), (5,3)]
```

弹出 `(2,2)`：
```
走 2 -> 4：dist[4]=3
走 2 -> 3：dist[3]=min(5, 2+3)=5
堆 = [(3,4), (5,3)]
```

弹出 `(3,4)`：
```
4 已是当前最短
答案 = 3
```

## 5. 堆优化 Dijkstra 模板

时间复杂度：`O((n+m) log n)`，校赛最常用。

```java
long[] dijkstra(int n, List<List<int[]>> g, int s) {
    long INF = (long) 4e18;
    long[] dist = new long[n + 1];
    Arrays.fill(dist, INF);
    dist[s] = 0;

    PriorityQueue<long[]> pq = new PriorityQueue<>(Comparator.comparingLong(a -> a[0]));
    pq.offer(new long[]{0, s});   // {dist, node}

    while (!pq.isEmpty()) {
        long[] cur = pq.poll();
        long dis = cur[0];
        int u = (int) cur[1];
        if (dis != dist[u]) continue;   // 旧状态，跳过

        for (int[] e : g.get(u)) {
            int v = e[0], w = e[1];
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.offer(new long[]{dist[v], v});
            }
        }
    }
    return dist;
}
```

## 6. 什么时候不是 Dijkstra

### 6.1 边权只有 0/1

用 **0/1 BFS** 更快，`O(n+m)`。

### 6.2 有负边

**Dijkstra 不能用。**

反例：
```
1 -> 2  权重 2
1 -> 3  权重 5
3 -> 2  权重 -10

真实最短路：1 -> 3 -> 2 = -5
但 Dijkstra 会先认为 2 的答案是 2，错了
```

这时通常考虑：
- Bellman-Ford
- SPFA（会被卡，谨慎）
- 若是 DAG，则可以拓扑序 DP

### 6.3 点数很小、要求任意两点最短路

可用 Floyd，复杂度 `O(n^3)`，适合 `n <= 500`。

## 7. 最短路建模常见信号

看到这些词就要警觉：
- 最少花费
- 最短时间
- 最少操作次数
- 从 A 到 B 的最优路径
- 地图、道路、航班、传送门
- 状态之间转移，每次转移有代价

很多题表面不是图，其实状态图早就藏在里面了。

---

## 8. 踩坑提醒 ⚠️

1. **Dijkstra 只适用于非负边**。
2. **距离用 long**：边权和路径长度都可能爆 `int`。
3. **堆里会有旧状态**：一定要写 `if (dis != dist[u]) continue;`
4. **无向边要加两次**，有向边只加一次。
5. **起点不可达终点**时，`dist[t]` 还是 INF，要单独判。

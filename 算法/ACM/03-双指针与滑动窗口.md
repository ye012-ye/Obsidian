# 03 · 双指针与滑动窗口

> 把 O(n²) 的枚举变成 O(n) 的利器。

---

## 1. 两种双指针

### 1.1 同向双指针（滑动窗口）

两个指针都往右走，形成一个可变长的"窗口"。
适合"子数组满足某条件"的题目。

```
l=0, r=0, 开始：
               ┌─窗口─┐
  a =  [2  1   3   4   1   2   5]
       ↑↑
       l r

r 右移，扩大窗口：
               ┌─窗口─┐
  a =  [2  1   3   4   1   2   5]
       ↑       ↑
       l       r

不满足条件了，l 右移收缩：
               ┌窗口┐
  a =  [2  1   3   4   1   2   5]
           ↑   ↑
           l   r
```

### 1.2 相向双指针

两个指针一个从左、一个从右，往中间走。
适合"有序数组里找两数和"。

```
  a = [1  2  4  7  11  15]
       ↑              ↑
       l              r

  sum = a[l] + a[r] = 1 + 15 = 16
  太大 → r 左移：

  a = [1  2  4  7  11  15]
       ↑          ↑
       l          r

  sum = 1 + 11 = 12
  太小 → l 右移：
  ...
```

## 2. 滑动窗口的万能模板

**场景**：找"最长子数组满足条件"或"最短子数组满足条件"。

```java
int l = 0;
int ans = 0;
// 窗口内状态（如 sum、字符计数等）
int sum = 0;

for (int r = 0; r < n; r++) {
    // 1) r 进入窗口，更新状态
    sum += a[r];

    // 2) 若违反条件，l 右移直到满足
    while (sum > K) {
        sum -= a[l];
        l++;
    }

    // 3) 此时窗口 [l, r] 合法，更新答案
    ans = Math.max(ans, r - l + 1);
}
```

**为什么是 O(n)？** r 走了 n 步，l 也最多走 n 步，总 2n 步。

## 3. 经典例题图解

### 例：最长连续子数组和 ≤ K

```
数组： a = [2, 1, 3, 4, 1, 2]      K = 6

r=0: sum=2  窗口 [2]          len=1
r=1: sum=3  窗口 [2,1]        len=2
r=2: sum=6  窗口 [2,1,3]      len=3 ← 目前最长
r=3: sum=10 >6，l 右移：
     sum=8  [1,3,4]
     sum=7  [3,4]
     sum=6  [3,4]? 等等
     
     实际过程：
     sum=10 → 减 a[0]=2 → sum=8, l=1
     sum=8  → 减 a[1]=1 → sum=7, l=2
     sum=7  → 减 a[2]=3 → sum=4, l=3
     现在窗口 [4]，合法。len=1
r=4: sum=5  [4,1]             len=2
r=5: sum=7  >6，l 右移：
     sum=3 [1,2], l=4         len=2

答案 = 3（即 [2,1,3] 或其他长度为 3 的段）
```

## 4. 相向双指针模板

**场景**：有序数组找两数之和 = target。

```java
int l = 0, r = n - 1;
while (l < r) {
    int sum = a[l] + a[r];
    if (sum == target) return new int[]{l, r};
    else if (sum < target) l++;
    else r--;
}
```

**关键前提**：数组**有序**！否则得先排序。

## 5. 滑窗判定"何时能用"

以下关键词基本都是滑窗信号：
- "最长 / 最短 连续子数组 ..."
- "包含所有字符的最小窗口" （变种）
- "和 ≤ K / ≥ K 的最长段"
- "区间内不同元素 ≤ K" → 配合 HashMap 计数
- "乘积 < K 的子数组数目"（LeetCode 713）

**共同特征**：
> 窗口扩大时条件"变严"，窗口缩小时"变松"（**单调性**）。

没这个单调性，就别用滑窗（想想为什么：l 不能退）。

## 6. 进阶：带 HashMap 的滑窗

### 例：最长无重复子串（LeetCode 3）

```java
int l = 0, ans = 0;
Map<Character, Integer> cnt = new HashMap<>();
for (int r = 0; r < s.length(); r++) {
    char c = s.charAt(r);
    cnt.merge(c, 1, Integer::sum);
    while (cnt.get(c) > 1) {              // 有重复，收缩
        char lc = s.charAt(l++);
        cnt.merge(lc, -1, Integer::sum);
    }
    ans = Math.max(ans, r - l + 1);
}
```

图示：
```
s = "abcabcbb"

r=0 'a': [a]           len=1
r=1 'b': [ab]          len=2
r=2 'c': [abc]         len=3 ←
r=3 'a': [abca] 有重复
         → 缩 l 到 1：[bca]
         → 此时 'a' 只剩 1，合法
                       len=3
r=4 'b': [bcab] 重复
         → 缩到 [cab]
                       len=3
...
```

## 7. 扩展进阶：恰好 K 个不同元素怎么做

很多题不是问"至多 K 个"，而是问"恰好 K 个"。这类题常用一个非常高频的套路：

> 恰好 K = 至多 K - 至多 (K-1)

为什么？
```
所有子数组按"不同元素个数"分层：

至多 3 个 = {1个, 2个, 3个}
至多 2 个 = {1个, 2个}

两者一减，正好只剩 {3个}
```

### 例：恰好 K 个不同整数的子数组个数（LC 992）

先写函数 `countAtMost(K)`：
- 维护一个"至多 K 个不同数"的合法窗口
- 固定右端点 r 后，以 r 结尾的合法子数组一共有 `r - l + 1` 个

```
窗口 [l..r] 合法
以 r 结尾的子数组：
[l..r], [l+1..r], [l+2..r], ... [r..r]

一共 r-l+1 个
```

所以这类题本质还是滑窗，只是答案更新从：
- 求最长时：`ans = max(ans, r - l + 1)`
- 求个数时：`ans += r - l + 1`

掌握这个套路后，很多"恰好 K"的题都能一眼转成两次"至多 K"。

## 8. 踩坑提醒 ⚠️

1. **只有单调性才能用滑窗**：数组有负数时，"和 ≥ K 的最短"就不能直接滑窗。
2. **while 里先更新状态再 l++**：别把更新和递增写错顺序。
3. **ans 更新位置**：要在 **while 退出后**（窗口合法）再更新，不然答案错。
4. **相向双指针需要排序**：顺序错了会漏答案。
5. **边界**：`while (l < r)` 还是 `l <= r`？找到重复/不重复要分清。

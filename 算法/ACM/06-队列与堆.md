# 06 · 队列与堆

> 队列用于 BFS；单调队列是滑窗最值神器；堆是"动态找最小/最大"的标准工具。

---

## 1. 队列（Queue）基础

### 1.1 原理：先进先出（FIFO）

```
入队 1, 2, 3：
   队头 →  1  2  3  ← 队尾
   
出队 → 拿到 1
          2  3
```

### 1.2 Java 写法

```java
ArrayDeque<Integer> q = new ArrayDeque<>();
q.offer(x);          // 入队（尾部）
int head = q.peek(); // 看队头
int x = q.poll();    // 出队
```

### 1.3 最主要用途：BFS

见 **07-DFS与BFS.md**。

## 2. 单调队列（Monotonic Deque）

### 2.1 经典问题：滑动窗口最大值（LC 239）

**题面**：数组 a，窗口大小 k，求每个窗口的最大值。

**暴力**：每个窗口扫一遍 → O(nk)，n=10⁶ 超时。
**单调队列**：O(n)。

### 2.2 原理

维护一个**单调递减**的双端队列（存下标）：
- 新元素从队尾进入，入队前先把队尾比它小的都弹掉（因为有了更大更新的，旧的那些永远不可能是最大值了）
- 队头元素如果超出窗口范围，弹掉
- 此刻队头就是窗口最大值

### 2.3 ASCII 图解

```
a = [1, 3, -1, -3, 5, 3, 6, 7]    k = 3

i=0, a=1:  弹没什么弹，入队。  队(值): [1]
                                窗口未满

i=1, a=3:  1 < 3，弹 1。入队 3。 队: [3]
                                窗口未满

i=2, a=-1: -1 < 3，直接入队。   队: [3, -1]
                                窗口满，最大值 = 队头 3 ✓

i=3, a=-3: -3 < -1，入队。      队: [3, -1, -3]
           队头下标 1 (值3)，还在 [1..3] 窗口内。
                                最大值 = 3 ✓

i=4, a=5:  弹 -3, -1, 3（都 < 5）。入队 5。 队: [5]
                                最大值 = 5 ✓

i=5, a=3:  3 < 5，入队。         队: [5, 3]
                                最大值 = 5 ✓

i=6, a=6:  弹 3, 5（5 的下标 4 还在窗口 [4..6] 内！别管值，这里也要弹）
                                入队 6。 队: [6]
                                最大值 = 6 ✓

i=7, a=7:  弹 6。入队 7。        队: [7]
                                最大值 = 7 ✓

结果 [3, 3, 5, 5, 6, 7] ✓
```

### 2.4 模板

```java
int[] maxInWindow(int[] a, int k) {
    int n = a.length;
    int[] ans = new int[n - k + 1];
    ArrayDeque<Integer> dq = new ArrayDeque<>();   // 存下标，a[dq] 单调递减
    for (int r = 0; r < n; r++) {
        // 1) 弹队尾比当前小的
        while (!dq.isEmpty() && a[dq.peekLast()] <= a[r]) dq.pollLast();
        // 2) 入队
        dq.offerLast(r);
        // 3) 弹队头超出窗口的
        while (dq.peekFirst() <= r - k) dq.pollFirst();
        // 4) 记录答案
        if (r >= k - 1) ans[r - k + 1] = a[dq.peekFirst()];
    }
    return ans;
}
```

> 滑窗**最小值**：把 `<=` 改成 `>=`，队列改成单调递增。

### 2.5 判定"何时用单调队列"

关键词：
- "固定大小窗口的最大/最小值"
- "长度 ≤ k 的区间和 / 极值"
- "DP 状态转移中需要快速求 max/min 且转移点连续"

## 3. 堆 / 优先队列（Priority Queue）

### 3.1 概念

**堆**本质是一棵"完全二叉树"，满足**父节点 ≤ 子节点**（小根堆）。

```
        1           ← 根（最小）
       / \
      3   5
     / \ / \
    8  4 7  9
```

Java 的 `PriorityQueue` **默认小根堆**。

### 3.2 常用操作（都是 O(log n)）

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();    // 小根堆
pq.offer(5);    // 入堆
int min = pq.peek();   // 看堆顶（最小）
int x = pq.poll();     // 弹出堆顶
```

**大根堆**：
```java
PriorityQueue<Integer> maxHeap = new PriorityQueue<>((a, b) -> b - a);
// 或（推荐，防溢出）：
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
```

### 3.3 图解：入堆过程（上浮）

```
堆: [1, 3, 5, 8, 4]，现在 offer(2)

     1              1                1
    / \            / \              / \
   3   5   →      3   5     →      2   5
  / \ /          / \ /\            / \ /\
 8  4           8  4 2             8  4 3
                  ↑上浮              （2 上浮，3 下移）
```

### 3.4 经典应用

**a. Top-K 问题**
找数组中前 K 小的数：维护大小为 K 的**大根堆**，新数比堆顶小则换入。
```java
PriorityQueue<Integer> pq = new PriorityQueue<>(Comparator.reverseOrder());
for (int x : a) {
    pq.offer(x);
    if (pq.size() > K) pq.poll();
}
// 堆里就是最小的 K 个
```

**b. 合并 K 个有序链表**：把每个链表头放进小根堆。

**c. Dijkstra**：配合 `pq` 实现堆优化最短路（见 10 章）。

**d. 带权区间调度**：反悔贪心模板里常见。

### 3.5 踩坑 ⚠️

1. Java 的 `PriorityQueue` **不是线程安全**（校赛不管这个）。
2. **遍历 PQ 不保证有序**！要有序只能 poll 全弹出来。
3. 比较器用 `(a,b) -> a - b` 有 **int 溢出风险**，大数用 `Integer.compare(a, b)` 或 `Long.compare`。
4. 存自定义对象要写好 `Comparator`。

## 4. 队列 / 栈 / 堆 对比

```
结构   特点          进出方式             取用方式
队列   FIFO          尾进头出             只能取队头
栈     LIFO          顶进顶出             只能取栈顶
堆     有序树        offer 随便，poll 最小 只能取堆顶（最值）
单调队列  双端单调    两头都能进出         可看最值也可看两端
```

## 5. 扩展进阶：对顶堆维护中位数

如果题目变成：
- 数据流不断加入数字
- 随时询问当前中位数

经典做法是 **两个堆**：
- 左边一个大根堆：维护较小的一半
- 右边一个小根堆：维护较大的一半

```
较小一半（大根堆）      较大一半（小根堆）
        [5]   |   [8]
      [3, 4]  |   [9, 10]

要求：
1. 左边所有数 <= 右边所有数
2. 两边元素个数差不超过 1
```

插入新数后：
1. 先放进合适的一边
2. 若两边大小失衡，就把堆顶挪过去

这样：
- 总数奇数时，中位数 = 元素更多那边的堆顶
- 总数偶数时，中位数 = 两个堆顶的平均值

这就是 `LC 295 数据流的中位数` 的标准解法，也是"堆不只是取最小值"的典型扩展。

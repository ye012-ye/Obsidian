# 14 · 树状数组（Fenwick Tree）

> 树状数组是"可修改前缀和"神器。它解决的是：数组会变，前缀和也得快速查。

---

## 1. 它解决什么问题

有一个数组 `a`：
- 支持单点加：`a[i] += v`
- 支持区间和：查询 `[1..x]` 或 `[l..r]` 的和

如果每次修改后都重算前缀和，太慢。

树状数组可以做到：
- 单点修改：`O(log n)`
- 前缀和查询：`O(log n)`

---

## 2. lowbit 是什么

树状数组的灵魂函数：

```java
int lowbit(int x) { return x & -x; }
```

意义：取出 x 二进制里**最低位的 1**。

```
x = 12 = 1100
-x      ...
x & -x = 0100 = 4
```

---

## 3. 每个节点管哪一段

令 `tree[i]` 管一段连续区间，长度就是 `lowbit(i)`。

```
i = 1, lowbit=1  -> 管 [1,1]
i = 2, lowbit=2  -> 管 [1,2]
i = 3, lowbit=1  -> 管 [3,3]
i = 4, lowbit=4  -> 管 [1,4]
i = 5, lowbit=1  -> 管 [5,5]
i = 6, lowbit=2  -> 管 [5,6]
i = 8, lowbit=8  -> 管 [1,8]
```

所以它像一棵"隐形树"：
```
8
├── 4
│   ├── 2
│   │   ├── 1
│   │   └── 2
│   └── 4
└── 8
```

你不用真的建树，只要用数组和 `lowbit` 跳就行。

---

## 4. 两个核心操作

### 4.1 单点加 `add(i, v)`

```java
void add(int i, long v) {
    while (i <= n) {
        tree[i] += v;
        i += i & -i;
    }
}
```

含义：位置 i 的值变了，所有"管到它"的节点都要更新。

### 4.2 前缀和 `sum(x)`

```java
long sum(int x) {
    long ans = 0;
    while (x > 0) {
        ans += tree[x];
        x -= x & -x;
    }
    return ans;
}
```

区间和：
```java
long rangeSum(int l, int r) {
    return sum(r) - sum(l - 1);
}
```

---

## 5. 图解：为什么查询是对的

求 `sum(13)`：

```
13 = 1101

取 tree[13]，它管 [13,13]
跳到 12

取 tree[12]，它管 [9,12]
跳到 8

取 tree[8]，它管 [1,8]
跳到 0

拼起来正好是：
[13,13] + [9,12] + [1,8] = [1,13]
```

这就是为什么能在 `O(log n)` 时间求前缀和。

## 6. 完整模板

```java
class BIT {
    int n;
    long[] tree;

    BIT(int n) {
        this.n = n;
        tree = new long[n + 1];
    }

    void add(int i, long v) {
        while (i <= n) {
            tree[i] += v;
            i += i & -i;
        }
    }

    long sum(int i) {
        long ans = 0;
        while (i > 0) {
            ans += tree[i];
            i -= i & -i;
        }
        return ans;
    }

    long rangeSum(int l, int r) {
        return sum(r) - sum(l - 1);
    }
}
```

## 7. 常见应用

### 7.1 动态前缀和

最裸的树状数组题。

### 7.2 逆序对

思路：
1. 先离散化
2. 从右往左扫
3. 查询当前数前面有多少更小的

### 7.3 区间加、单点查

对差分数组开树状数组。

```java
// 区间 [l, r] + v
add(l, v);
add(r + 1, -v);
```

查某一点的值就是前缀和。

---

## 8. 树状数组 vs 线段树

```
树状数组：代码短、常数小、适合前缀和类
线段树：更通用，可做区间最值、区间修改、懒标记
```

如果题目只是：
- 单点改
- 前缀和 / 区间和

优先树状数组，写得快很多。

---

## 9. 踩坑提醒 ⚠️

1. **树状数组通常从 1 开始编号**，别从 0 开。
2. **lowbit 写成 `x & -x`**，别手误。
3. **数据范围大用 long**。
4. 做逆序对时，值域大要先**离散化**。
5. 树状数组更擅长"可加可减的前缀信息"，别拿它硬做区间最小值。

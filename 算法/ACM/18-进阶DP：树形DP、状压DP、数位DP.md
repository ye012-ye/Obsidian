# 18 · 进阶 DP：树形 DP、状压 DP、数位 DP

> 普通 DP 会了以后，校赛里真正拉开差距的，往往是这几类"状态不再是线性的 DP"。

---

## 1. 树形 DP

### 1.1 什么时候想到

题目是树，并且：
- 每个点要选 / 不选
- 父子之间有约束
- 需要统计子树最优解

### 1.2 经典模型：相邻点不能同时选

定义：
- `dp[u][0]` = 不选 u 时，u 子树最大收益
- `dp[u][1]` = 选 u 时，u 子树最大收益

转移：

```java
dp[u][1] += dp[v][0];
dp[u][0] += Math.max(dp[v][0], dp[v][1]);
```

图上理解：

```
        u
      /   \
     v     w

若选 u：
  v、w 都不能选

若不选 u：
  v、w 各自选最优
```

### 1.3 模板框架

```java
void dfs(int u, int fa) {
    dp[u][1] = val[u];
    for (int v : g.get(u)) {
        if (v == fa) continue;
        dfs(v, u);
        dp[u][0] += Math.max(dp[v][0], dp[v][1]);
        dp[u][1] += dp[v][0];
    }
}
```

## 2. 状压 DP

### 2.1 什么时候想到

如果题目有：
- `n <= 20`
- 子集、访问状态、选过哪些点

就要想到用二进制表示集合。

### 2.2 用一个整数表示集合

```
mask = 10110
表示第 1、2、4 位被选了
```

判断第 i 位是否选过：

```java
((mask >> i) & 1) == 1
```

加入第 i 位：

```java
mask | (1 << i)
```

---

## 3. 经典例：旅行商入门版

`dp[mask][i]` 表示：

> 已经访问过的点集合是 `mask`，最后停在 i 的最小代价。

转移：

```java
dp[mask | (1 << j)][j]
    = min(dp[mask | (1 << j)][j],
          dp[mask][i] + w[i][j]);
```

虽然复杂度高，但当 `n <= 15~20` 时可行。

## 4. 数位 DP

### 4.1 什么时候想到

题目长这样：
- 统计 `0 ~ n` 中满足某性质的数有多少个
- 性质和数位有关
- 不能直接暴力枚举

比如：
- 不含数字 4 的数有多少个
- 数位和等于 k 的数有多少个
- 某些数位不能相邻出现

### 4.2 核心状态

常见 DFS 状态：

```java
dfs(pos, state, limit, started)
```

含义：
- `pos`：当前处理到第几位
- `state`：之前的状态（如前一位是多少、数字和是多少）
- `limit`：是否贴着上界
- `started`：前面是否已经开始放有效数字

这是一个"记忆化搜索版 DP"。

---

## 5. 数位 DP 图上理解

求 `0 ~ 325` 中满足条件的数：

```
第 1 位：可选 0~3
若选 3，则第 2 位只能选 0~2
若第 2 位选 2，则第 3 位只能选 0~5

这就是 limit 的含义：
前面贴着上界，后面就受约束；
前面一旦变小，后面就随便选。
```

---

## 6. 三类进阶 DP 怎么区分

```
树结构上的状态转移      -> 树形 DP
集合/访问状态很小       -> 状压 DP
统计区间内满足数位性质   -> 数位 DP
```

它们本质都还是：
- 定义状态
- 枚举转移
- 记忆结果

只是状态空间长得不一样。

---

## 7. 踩坑提醒 ⚠️

1. **树形 DP 要防止走回父节点**。
2. 状压 DP 的空间很大，先估复杂度：`2^n * n` 或 `2^n * n^2`。
3. 数位 DP 的 `limit` 和 `started` 很容易漏记。
4. `1 << n` 在 `n=31` 以上会炸，必要时用 `1L << n`。
5. 先把状态定义写清楚，再写转移，否则很容易乱。
